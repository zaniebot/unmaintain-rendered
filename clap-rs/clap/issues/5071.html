Rust Version

rustc 1.71.1 (eb26296b5 2023-08-03)

Clap Version

4.3.21

Minimal reproducible code

use clap::{Command};
fn main() {
    let mut app = clap::Command::new("clap-bug")
        .multicall(true)
        .version(env!("CARGO_PKG_VERSION"))
        .propagate_version(true)
        .subcommand(
            Command::new("clap-bug")
                .author("Author Here")
                .about("Program name")
                .arg_required_else_help(true)
                .subcommands([Command::new("list").about("list applets")])
                .subcommand_value_name("APPLET")
                .subcommand_help_heading("APPLETS")
        );

    // Parse args
    app.get_matches_mut();
}

Steps to reproduce the bug with the above code

cargo run -- -h

Actual Behaviour

author and program version are missing in the output:

Program name

Usage: clap-bug [APPLET]

APPLETS:
  list  list applets
  help  Print this message or the help of the given subcommand(s)

Options:
  -h, --help     Print help
  -V, --version  Print version

Expected Behaviour

This is the behaviour with clap 3.2.25:

clap-bug 0.1.0
Author Here
Program name

USAGE:
    clap-bug [APPLET]

OPTIONS:
    -h, --help       Print help information
    -V, --version    Print version information

APPLETS:
    help    Print this message or the help of the given subcommand(s)
    list    list applets

Additional Context

I also tried to put the author and program name in the main Command, but this does not change anything.
