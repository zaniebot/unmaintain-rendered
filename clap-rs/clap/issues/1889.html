`print_help()` and `--help` differ when using the derive macro

#1889 opened by @cole-h on 2020-05-01 03:04
Issue opened by @cole-h on 2020-05-01 03:04

NOTE: This could very well be present without the Clap derive; I didn't test that.

Make sure you completed the following tasks

  • [x] Searched the discussions
  • [x] Searched the closed issues

Code

use clap::{Clap, IntoApp};

#[derive(Clap, Debug)]
struct Test {
    app: Option<String>,
}

fn main() {
    let a = Test::parse();

    match a.app {
        Some(_) => (),
        _ => Test::into_app().print_help().unwrap(),
    }
}

Steps to reproduce the issue

  1. Run cargo run -- --help
  2. Run cargo run
  3. Notice that the two helps are different

Version

  • Rust: rustc 1.42.0 (b8cedc004 2020-03-09)
  • Clap: master @ 087fab2ab3b643b73e93f9ba7de74f36164d1a45

Actual Behavior Summary

Compare

# cargo run -- --help
claptest

USAGE:
    claptest [app]

ARGS:
    <app>

FLAGS:
    -h, --help       Prints help information
    -V, --version    Prints version information

vs

# cargo run
claptest

USAGE:
    claptest [app]

ARGS:
    <app>


FLAGS:
    -h, --help
            Prints help information

    -V, --version
            Prints version information

Expected Behavior Summary

I'd expect both to look like the first.
