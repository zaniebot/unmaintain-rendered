# `default_value_if` does not respect `global(true)`

---

_Issue opened by @adamspofford-dfinity on 2023-12-01 01:06_

- [X] I have searched the [discussions](https://github.com/clap-rs/clap/discussions)

### Rust Version

rustc 1.73.0 (cc66ad468 2023-10-03)

### Clap Version

4.4.10, master (c0a1814d)

### Minimal reproducible code

```rust
use clap::{Parser, builder::ArgPredicate};
#[derive(Parser)]
struct Cmd {
    #[arg(
      default_value_if("default", ArgPredicate::IsPresent, "default value")
    )]
    param: Option<String>,
    #[arg(long, global = true)]
    default: bool,
    #[command(subcommand)]
    subcommand: Option<Subcommand>,
}
#[derive(Parser)]
enum Subcommand {
    Subcommand,
}
fn main() {
    let cmd = Cmd::parse();
    println!("param: {}", cmd.param.unwrap_or("<none>".into()));
}
```


### Steps to reproduce the bug with the above code

```sh
cargo run -- --default
cargo run -- subcommand --default
```


### Actual Behaviour

```
param: default value
param: <none>
```

### Expected Behaviour

```
param: default value
param: default value
```
