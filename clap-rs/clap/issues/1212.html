Comment by @zx8 on 2018-03-18 13:06

The issue here is the usage of the __fish_using_command function. Here's what's going on:

$ rg --no-follow --<TAB>
function __fish_using_command
    # cmd is set to array: ["rg", "--no-line-number"]
    set cmd (commandline -opc)
    # 'count $cmd' is 2, 'count $argv' is 1  
    if [ (count $cmd) -eq (count $argv) ]
        # etc.
        return 0
    end
    # no completions
    return 1
end

If you drop this function entirely (which I understand was introduced to fix subcommand completion; kbknapp/clap-rs#687, /ping @wdv4758h), flag completions >1 work again, i.e.

complete -c rg -s N -l no-line-number -d 'Suppress line numbers.'
