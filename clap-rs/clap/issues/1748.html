arg_enum Custom Names

Closed #1748 opened by @erayerdin on 2020-03-14 23:04
Issue opened by @erayerdin on 2020-03-14 23:04

Currently, we can assign enums to possible_values with arg_enum macro. While this is good enough, it is usually considered values to be all lower case. Take the example:

arg_enum! {
    enum AuthType {
        HOTP,
        TOTP
    }
}

In this case, the possible values for an arg referencing this would be "HOTP" or "TOTP" whereas it would be more convenient to have it "hotp" and "totp".
