Help message alignment inconsistent when using .multiple(true)

#702 opened by @mattcl on 2016-10-24 10:02
@mattcl

Given the following code and clap version 2.16.0 OS is Ubuntu 14.04 TERM=xterm-256color

extern crate clap;
use clap::{App, Arg};

fn main() {
    let matches = App::new("MyApp")
        .version("1.0")
        .author("foo")
        .about("bar")
        .arg(Arg::with_name("some")
             .help("some option")
             .short("s")
             .long("some")
             .takes_value(true))
        .arg(Arg::with_name("other")
             .help("some other option")
             .short("o")
             .long("other")
             .takes_value(true))
        .arg(Arg::with_name("label")
             .help("a label")
             .short("l")
             .long("label")
             .multiple(true)
             .takes_value(true))
        .get_matches();
}

running clap-test --help yields

MyApp 1.0
foo
bar

USAGE:
    clap-test [OPTIONS]

FLAGS:
    -h, --help       Prints help information
    -V, --version    Prints version information

OPTIONS:
    -l, --label <label>    a label
    -o, --other <other>       some other option
    -s, --some <some>         some option

Note the missing padding before "a label"

Whereas removing the .multiple(true) yields

MyApp 1.0
foo
bar

USAGE:
    clap-test [OPTIONS]

FLAGS:
    -h, --help       Prints help information
    -V, --version    Prints version information

OPTIONS:
    -l, --label <label>    a label
    -o, --other <other>    some other option
    -s, --some <some>      some option
