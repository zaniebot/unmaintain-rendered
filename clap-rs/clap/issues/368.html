App name is not used in help

#368 opened by @ghost on 2015-12-24 13:55
Issue opened by @ghost on 2015-12-24 13:55

The first line of help should be the app name but it's always the binary name (whether bin_name is configured or not.)

For example, when this app is compiled to mraa

extern crate clap;

use clap::App;

fn main() {
    App::new("My-Really-Awesome-App").get_matches();

    println!("Hello, world!");
}

mraa --help prints

mraa 

USAGE:
    mraa [FLAGS]

FLAGS:
    -h, --help       Prints help information
    -V, --version    Prints version information

whereas it should print

My Really Awesome App

USAGE:
    mraa [FLAGS]

FLAGS:
    -h, --help       Prints help information
    -V, --version    Prints version information
