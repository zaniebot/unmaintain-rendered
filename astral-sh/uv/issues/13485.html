Not using git LFS by default is uv-specific behavior, no other tool I know of does this. So no other tool would require such an URL scheme.

But @gsemet is right that if uv wants to stick with this rather surprising behavior, a project that depends on a git dependency with LFS files needs some mechanism for declaring this uv-specific requirement, perhaps in something like a [tool.uv.lfs] section.
