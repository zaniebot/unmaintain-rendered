The warning is to indicate that that the command you're running does not come from the installed package. In this case, it's not a big deal because that's the intent of the ptipython meta-package. In general, it's not common for a package (in this case, ptpython) to contain a command (ptipython) that cannot actually be used. Ideally they'd provide an extra that includes ipython instead of using a meta package like this.

An invocation that does not warn would be uvx --from ptpython --with ipython ptipython

Unfortunately I can't think of a way for us to improve this without encoding knowledge of specific packages.
