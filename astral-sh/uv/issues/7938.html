Feature request: Add ability to exclude certain file/directory patterns from uv build output

#7938 opened by @pirate on 2024-10-05 06:18
Issue opened by @pirate on 2024-10-05 06:18

Currently it seems uv build includes everything found in the root directory, but it would be great to be able to exclude certain files/directories.

For example, I don't want to bundle these in my sdist/bdist wheel:

**/.DS_Store
**/*.pyc
**/__pycache__
tests

It would also be great if I could force it to include my pyproject.toml and .git/refs/HEAD from the source code root dir as I like to ship it inside the sdist (so that I can print more useful debug info including installed commit hash and more).

Should this be accomplished by adding MANIFEST.in support? Or perhaps just some pyproject.toml config options? No strong opinions on my side, up to you.

Thanks for building uv!

Comment by @kahojyun on 2024-10-05 06:33

uv is just a build frontend right now, and the file selection is handled by build-backend specified in pyproject.toml. For example, you can configure hatchling (the build-backend in the pyproject.toml generated by uv init --lib).

Comment by @zanieb on 2024-10-05 14:52

Yeah we're working on our own build backend but right now this is configurable using other tools (the default build backend is setuptools if one is not declared).
