Flag for sequential packages download/sync

Closed #16101 opened by @Khaled-Fawry on 2025-10-02 10:38
Issue opened by @Khaled-Fawry on 2025-10-02 10:38

Summary

UV Version: 0.8.22 The operating system: Debian GNU/Linux 13 (trixie) Command Ran: uv sync

Problem When I run uv sync with many packages on a bad internet connection, it constantly fails with this timeout error:

Failed to download distribution due to network timeout. Try increasing UV_HTTP_TIMEOUT (current value: 30s)

This happens because uv is downloading a ton of packages in parallel. If one download stalls or fails, all its progress is lost. I get stuck in a painful loop: try to sync → fail → lose progress → try again → fail...

Increasing UV_HTTP_TIMEOUT doesn't always work; the network is just too unstable for many parallel streams.

Suggested Fix Add a flag, maybe --sequential, to force uv to download one package at a time. This puts less strain on the connection and makes timeouts less likely.

Example

uv sync --sequential
