`uv pip install` should support reasonable variations on `pythonw.exe` e.g. `.pythonw.exe`

#5256 opened by @paveldikov on 2024-07-21 12:26
Issue opened by @paveldikov on 2024-07-21 12:26

When generating GUI script executables on Windows, uv will take python_executable and try to replace the filename with a constant "pythonw.exe".

Whereas pip is slightly more subtle in this regard. It would take the filename and replace the python component with pythonw. This allows for the executable name to be renamed (within reason) e.g. .python.exe and .pythonw.exe respectively.

Our use case here is that we generate relocatable virtual environments, and we deliberately rename the interpreter executable so as to avoid shadowing existing Python interpreters on the $PATH.

Happy to raise a PR, although I am new to Rust so you might have to bear with me futzing around a bit :-)
