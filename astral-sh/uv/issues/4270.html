Comment by @zanieb on 2024-06-12 13:45

Hi! Sorry this will be clarified in future documentation so it's not a surprise. We use the python-build-standalone builds similar to Hatch and Rye. We're collaborators over on that project so they're not random.

The problem with using the official Python builds (as far as I know) is that they are not portable e.g. whether or not the build will work on your machine depends on things outside of our control. We can definitely consider an option for this though and I'd love to hear more discussion about the trade-offs.

You may also be interested in PEP 711. cc @konstin who might be able to provide some more context on the builds.

Comment by @charliermarsh on 2024-06-12 13:47

I started writing this before Zanie replied so I'll just post my perspective.

Appreciate the question, a few pieces to this...

First, python.org doesn't provide pre-built redistributable binaries -- they provide source releases. We don't currently plan to facilitate building Python from source -- perhaps that will change in the future. You're of course welcome to use a separate tool if you want to install Python from source, and our tooling will contain to interoperate with those external toolchains. There's no requirement that you use the uv toolchain stuff to use uv and that will continue to be true going forward.

Second, we're maintainers on python-build-standalone, which compiles Python from the source releases published on python.org. It's become a very popular source of installing Python and is fairly load-bearing in the ecosystem now -- the releases are used tens of millions of times per month, and they're the Python distribution that you get if you use Bazel, Rye, Mise, etc.
