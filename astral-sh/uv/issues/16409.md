# uv can't find Python with correct version

---

_Issue opened by @typed-sigterm on 2025-10-22 12:02_

### Summary

Possibly regression of #10898.

```
pi@raspberrypi:~/hzh $ source .venv/bin/activate

(znbw) pi@raspberrypi:~/hzh $ echo $PATH
/home/pi/hzh/.venv/bin:/home/pi/.local/bin:/home/pi/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/local/games:/usr/games

(znbw) pi@raspberrypi:~/hzh $ python --version
Python 3.13.9

(znbw) pi@raspberrypi:~/hzh $ which python
/home/pi/hzh/.venv/bin/python

(znbw) pi@raspberrypi:~/hzh $ uv run main.py
error: Failed to inspect Python interpreter from search path at `/usr/bin/python`
  Caused by: Can't use Python at `/usr/bin/python`
  Caused by: Python executable does not support `-I` flag. Please use Python 3.8 or newer.
```

To reproduce:
1. Remove all `python`s and put Python 3.7 to `/usr/bin`
2. `uv init`
3. `source .venv/bin/activate`
4. `uv python install 3.13`
5. `uv run main.py`

uv recognized Python 3.7 instead Python 3.13, so it complained.

### Platform

Linux 5.10.103-v7l+ armv7l GNU/Linux

### Version

uv 0.9.5

### Python version

N/A (error: Failed to inspect Python interpreter...)
