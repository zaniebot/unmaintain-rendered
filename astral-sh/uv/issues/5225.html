Should the Python version used to generate documentation in CONTRIBUTING docs be explicitly specified?

Closed #5225 opened by @FishAlchemist on 2024-07-19 16:15
Issue opened by @FishAlchemist on 2024-07-19 16:15

Explicitly specify the Python version when creating the Python virtual environment for generating documentation. Instead of letting UV only look for the Python available in the environment, it is suggested to specify the minimum required Python version. https://github.com/astral-sh/uv/blob/12dd450a8e0f61ab503f9b93ecae2ccb049bcd06/CONTRIBUTING.md?plain=1#L126 I recommend explicitly specifying the Python version because the minimum Python version is specified when generating requirements file https://github.com/astral-sh/uv/blob/12dd450a8e0f61ab503f9b93ecae2ccb049bcd06/CONTRIBUTING.md?plain=1#L160-L163

  -p, --python-version <PYTHON_VERSION> The minimum Python version that should be supported by the resolved requirements (e.g., `3.8` or `3.8.17`)

While the requirements file generated by UV uses environment markers as described in PEP 508, and should work fine, using the same Python environment is likely to be more stable.
