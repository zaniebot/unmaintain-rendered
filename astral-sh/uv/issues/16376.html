Allow Script Installation as CLI Tools

Closed #16376 opened by @o-murphy on 2025-10-20 13:50
Issue opened by @o-murphy on 2025-10-20 13:50

Summary

Currently, installing a Python script as a global CLI tool with uv requires setting up a full project layout and build system ‚Äî even when dealing with a simple, single-file script. This adds unnecessary complexity for lightweight utilities.

Proposed Solution

Enable direct installation of standalone scripts as CLI tools using the Declaring script dependencies mechanism.

Example Usage

From a local script

uv install tool myscript.py

From a remote source (e.g. GitHub / GitHub Gist)

uv install tool https://gist.githubusercontent.com/<username>/<gist-id>/raw/<file-name>.py

Benefits

  • Simplifies distribution and reuse of small, dependency-aware scripts.
  • Reduces the need for boilerplate project setup.
  • Aligns with uv run --script behavior, providing a consistent experience for both one-off execution and persistent installation.

Notes

This feature would make uv an even more powerful replacement for tools like pipx when working with lightweight, single-file utilities.
