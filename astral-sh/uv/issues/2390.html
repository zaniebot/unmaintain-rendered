Is this by design as pip should only be called with uv?

If I run python3.11 -m venv .venv this doesn't happen. We're on version 0.1.17 of uv.

Comment by @charliermarsh on 2024-03-12 19:43

You can run uv venv --seed to include pip. We don't install pip into virtual environments by default, since uv on its own is sufficient to install, etc. (It's supposed, it's just not the default.)
