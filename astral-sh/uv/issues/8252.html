Comment by @charliermarsh on 2024-10-16 13:16

It's something specific to Lerna, related to the use of a blanket .gitignore in the cache:

// Add the .gitignore.
match fs_err::OpenOptions::new()
    .write(true)
    .create_new(true)
    .open(root.join(".gitignore"))
{
    Ok(mut file) => file.write_all(b"*")?,
    Err(err) if err.kind() == io::ErrorKind::AlreadyExists => (),
    Err(err) => return Err(err),
}

I have no idea why that's causing problems for Lerna though. I tried a bunch of other tricks like adding a non-empty .gitignore one subdirectory below, but none of them helped.
