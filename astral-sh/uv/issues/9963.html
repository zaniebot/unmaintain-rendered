That’s not really what’s happening. It’s looking for a torch version, and it picks that version, and that version happens to be lacking macOS wheels. You can read this issue if you want more detail: astral-sh/uv#9711. There’s also a linked PR that would fix this automatically.

Comment by @charliermarsh on 2024-12-17 21:42

The logging you mentioned is also a red herring. We just pick an arbitrary wheel to determine the package metadata. It has nothing to do with whether the wheel is considered valid on macOS.
