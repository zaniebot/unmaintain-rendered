UPD2: was able to install on python 3.10.12 (uv).

UPD3: Was able to install on 3.12.5. I just installed pip using uv pip install pip and then added my dependency:

pip install pystemmer
uv add fastembed-haystack

Question: are there more elegant solutions, and how can i avoid doing those workarounds and just install normally in the future?

UPD4: Got a response in pystemmer github. Apparently, when you directly specify that you want to use a gcc instead of clang as a compiler during installing this package using regular uv means, it works:

CC=gcc-11 uv pip install pystemmer # or
CC=gcc-11 uv add fastembed-haystack

Is there a way to use gcc by default (instead of clang)?
