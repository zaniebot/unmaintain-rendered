Summary

However, when executing these scripts, the same packages are being reinstalled repeatedly, bypassing the global cache that uv typically leverages. How can I ensure that the package installations via python -m pip install respect uvâ€™s global cache to avoid redundant downloads and improve efficiency?

Example

Use uv init to create a project, then use uv add --dev pip to install pip.

Run uv add torch to cache packages and uv remove torch to remove it.

Activate the venv, run python -m pip install torch, torch is downloaded again insteaded of fetched from global cache.
