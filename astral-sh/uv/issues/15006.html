Feature request: pyproject.toml formatting and validation

#15006 opened by @gorewilliams on 2025-07-31 19:35
@gorewilliams

Summary

Add support for:

  1. Formatting pyproject.toml to a consistent style (ordering, wrapping, indentation)
  2. Validating pyproject.toml against PEP 621 and tool-specific schemas/semantics

Motivation

pyproject.toml has become the single source of truth for modern Python projects (build system, dependencies, tool configs). It therefore deserves fast, accurate validation and formatting.

Related discussions

  1. Feature request for uv to support pyproject.toml formatting: uv#6308
  2. Interest in pyproject.toml validation: ruff#7839

Existing tools

  1. Formatting tool: pyproject-fmt
  2. Validation tool: validate-pyproject

Potential UX

A Ruff-style interface:

uv format
uv check --fix

Or a single, simpler command that handles both formatting and validation:

uv pretty
