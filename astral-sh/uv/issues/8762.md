`uv add mlx` uses [universal resolution](https://docs.astral.sh/uv/concepts/resolution/#universal-resolution) so it checks all macOS versions.

I don't think we'll have an automatic matching mechanism in the future. We're already using the standard variable to determine the macOS deployment target and universal resolution is the better way to go about this.
