Comment by @geofft on 2025-08-06 16:27

I think the specific reason /usr/bin shows up here is that Gentoo's Python is set up such that sudo pip install something, the commands go into /usr/bin (i.e., that sysconfig.get_path('scripts') == '/usr/bin'). On e.g. Debian/Ubuntu, it's set up to use /usr/local/bin even though Python itself is in /usr/bin. So you can reproduce this bug on Debian/Ubuntu with e.g.

sudo truncate -s 1000 /usr/local/bin/unreadable
sudo chmod 000 /usr/local/bin/unreadable
uv run --no-managed-python --with attrs python

This fails in the same way on uv 0.8.4, and is indeed fixed on 0.8.5.
