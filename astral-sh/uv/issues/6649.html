Cannot install packages or download Python in Windows with Corporate Cisco Umbrella Certificate

#6649 opened by @Syrus-at-Philips on 2024-08-26 19:25
@Syrus-at-Philips

Using the native TLS option, uv is able to do a self-update, create a virtual environment, and list available Python versions. It cannot download Python or install packages from pypi. The issue is that self-update and checking for available package versions does work, so the initial SSL queries are okay using native TLS and the corporate man-in-the-middle certificate. It's later in the process when uv tries to download, that certificate errors occur. I suspect that uv is not using native TLS during the download phase for Python distributions as well as for packages.
