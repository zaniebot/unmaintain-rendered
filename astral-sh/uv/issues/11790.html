uv can't install packages with very long version numbers

#11790 opened by @nkitsaini on 2025-02-26 04:45
Issue opened by @nkitsaini on 2025-02-26 04:45

Summary

NOTE: This has not been fixed yet, see comment below.

uv can't parse versions with long digits as it expects each segment to fit in a u64.

This does not come from a real use-case and should be pretty rare, so the issue should be very low priority. I found this while working on https://github.com/astral-sh/uv/pull/11738 and it was still not easy to find a package that worked with pip but didn't with uv.

I doubt anyone will run into this, from a quick look at pypi-data there are very few packages with such versioning. I am creating the issue mostly to make sure it is a conscious decision to not support these versions.
