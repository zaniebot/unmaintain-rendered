uv sync --no-editable does not rebuild/reinstall the project after source is modified

Closed #13685 opened by @f3flight on 2025-05-27 17:17
Issue opened by @f3flight on 2025-05-27 17:17

I ran into a problem where I need UV to install a package normally (not in editable mode), but also this env is part of a development workflow and if changes are made locally the expectation is that the installed package will get updated when uv sync --no-editable is executed again, however this doesn't happen. What's worse is that even if venv is wiped and created anew, uv will still install the old version of the built package from cache, it will not rebuild it.

Is there a way to make uv detect that sources are modified (even though project version did not change) and rebuild the package when --no-editable is requested? Imho this should be default behavior, as caching an in-development package might not be the best idea.

Let me know what you think and if there's any existing solution

Comment by @charliermarsh on 2025-05-27 17:18

You can set a cache key if you want that behavior: https://docs.astral.sh/uv/concepts/cache/
