# Make UV_GIT_LFS active by default on some url

---

_Issue opened by @gsemet on 2025-05-16 07:34_

### Summary

When installing some on our dependencies by URL, we need to set {{UV_GIT_LFS=1 uv run...}}.

Would it be possible to have this option definied somewhere arround the URL, to specify that a given git+https needs LFS, and another one does not.

Ideas:

```
uv add git+lfs+ssh://git@github.com/...
```

_Comment by @pwuertz on 2025-07-29 11:05_

Not using git LFS by default is `uv`-specific behavior, no other tool I know of does this. So no other tool would require such an URL scheme.

But @gsemet is right that if `uv` wants to stick with this rather surprising behavior, a project that depends on a git dependency with LFS files needs some mechanism for declaring this `uv`-specific requirement, perhaps in something like a `[tool.uv.lfs]` section.
