Treat path sources as package = true by default

#12015 opened by @charliermarsh on 2025-03-06 18:20
@charliermarsh

In [tool.uv.sources], path sources should be treated as packages by default (even if they don't specify a build system). It's confusing that we treat them as virtual. As of astral-sh/uv#12014, users can annotate such dependencies with package = false as an escape hatch.

See: astral-sh/uv#12014.

See: astral-sh/uv#11998.
