Allow fields that are classified as "project metadata" in `uv.toml` files in projects

Open #14908 opened by @zanieb on 2025-07-25 21:07
Issue opened by @zanieb on 2025-07-25 21:07

It's unclear if we should do this, but this issue is for discussion.

We currently categorize fields as project metadata, which are only allowed in the pyproject.toml, or configuration which is allowed in the uv.toml.

While we do not want to read project metadata settings from outside the project (e.g., from a user level uv.toml), we could allow reading project metadata fields from a uv.toml that is alongside a pyproject.toml that does not contain a [tool.uv] table.
