# If the project path contains Cyrillic characters, the project will not be imported.

---

_Issue opened by @Wintreist on 2025-04-02 10:42_

### Summary

I think that we have already reached a point in time when the name of the path in non-English language should not cause problems.

However if there are Cyrillic characters in the project path, an import error will occur:
```
Traceback (most recent call last):
  File "d:\Не рабочее\uv_cirrilic_path\tests\test_import.py", line 1, in <module>
    from uv_cyrillic_path import hello
ModuleNotFoundError: No module named 'uv_cyrillic_path'
```

I created a regular project using `uv init --lib --no-pin-python --no-readme --author-from=none`
and use `uv sync`
The virtual environment looks like this:
```
D:\НЕ РАБОЧЕЕ\UV_CIRRILIC_PATH\.VENV\LIB\SITE-PACKAGES
│   _uv_cyrillic_path.pth
│   _virtualenv.pth
│   _virtualenv.py
│
├───uv_cyrillic_path-0.1.0.dist-info
│       direct_url.json
│       INSTALLER
│       METADATA
│       RECORD
│       REQUESTED
│       uv_cache.json
│       WHEEL
```
`_uv_cyrillic_path.pth` in utf-8 encoding:

> D:\Не рабочее\uv_cirrilic_path\src

And everything looks right, but for some reason it doesn't work.

If I rename the path to English and change only the `_uv_cyrillic_path.pth` file, there will be no error.

I have tested this error with different combinations.
I checked the path of English characters only - it works
I checked the path of English characters and spaces - it works
I checked the path of Cyrillic characters and spaces (my original problem) - it doesn't work
I checked the path of Cyrillic characters only - it doesn't work


### Platform

Windows 10 x86_64

### Version

uv 0.6.11 (0632e24d1 2025-03-30)

### Python version

Python 3.12.2
