Comment by @trallard on 2024-11-19 12:03

Hey @killcoder it would not be a 1-to-1 equivalent, but you should be able to install your package in editable (similar-ish to development mode in conda) with uv pip install -e ... ref: https://docs.astral.sh/uv/pip/packages/#editable-packages

Comment by @konstin on 2024-11-19 13:54

In your pyproject.toml, you need to declare a build backend (https://docs.astral.sh/uv/concepts/projects/#build-systems). For that, the [project] name (UV_Test) should match the directory with your code (assets). The default from uv is:

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

With a build backend declared, uv sync will do an editable install of your project, and your code should import as you expect it.
