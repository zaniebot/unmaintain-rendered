uv pip compile does not include extra args in resulting requirements file.

#2565 opened by @jakesco on 2024-03-20 15:02
@jakesco

Hello!

I'm running uv version 0.1.19 on WSL.

I noticed our extra --index-url arguments are not getting put in requirements files when running uv pip compile

For our current usage of pip-tools we run the following command:

$ pip-compile pyproject.toml -o requirements.txt \
	--index-url https://example.org/root/pypi/+simple/ \
	--extra-index-url https://example.org/root/otherrepo/+simple/

This produces a requirements file like so:

#
# This file is autogenerated by pip-compile with Python 3.11
# by the following command:
#
#    pip-compile --extra-index-url=https://example.org/root/otherrepo/+simple/ --index-url=https://example.org/root/pypi/+simple/ --output-file=requirements.txt pyproject.toml
#
--index-url https://example.org/root/pypi/+simple/
--extra-index-url https://example.org/root/otherrepo/+simple/

anyio==3.7.1
    # via httpcore
...

However, when running

$ uv pip compile pyproject.toml -o requirements.txt \
	--index-url https://example.org/root/pypi/+simple/ \
	--extra-index-url https://example.org/root/otherrepo/+simple/

This produces a requirements file like so:

#
# This file is autogenerated by uv with the following command
#       uv pip compile pyproject.toml -o requirements.txt
anyio==3.7.1
    # via httpcore
...
Comment by @charliermarsh on 2024-03-20 15:06

You can pass --emit-index-url.
