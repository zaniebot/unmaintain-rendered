Comment by @hoechenberger on 2025-03-14 20:40

Typically you would specify a lower bound for Python in pyproject.toml; that is, the oldest version of Python that your package supports.

In contrast, .python-version specifies which Python version, concretely, to use during development.

Example: You want to use a development venv with Python 3.13, but indicate that your package runs with Python 3.10 and newer. Here, you would have 3.13 in .python-version, and >= 3.10 in pyproject.toml.

I hope this helps.
