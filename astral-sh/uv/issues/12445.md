IMO this is wrong and uv should pick 3.13t.

To reproduce this with `kornia-rs`, do:

```

$ just install-dev
$ uv run -v maturin develop -m Cargo.toml
```

I haven't checked if this happens in a simpler setup.

This sounds like a bug, we should allow a discovered environment to meet this request in a project context. I think it might be a bit tricky to fix, because outside of a project if you did `uv python find '>=3.8'` we would not want to select a free-threaded interpreter without opt-in.
