uv pip installs installs override that does not match requested environment markers

Closed #4976 opened by @sbidoul on 2024-07-10 21:07
Issue opened by @sbidoul on 2024-07-10 21:07

This is a variant of #4826, with uv 0.2.24

pyproject.toml:

[project]
name = "demo"
version = "1.0"
dependencies = [
   "importlib_metadata; python_version<'3.8'"
]

override.txt

importlib-metadata==4.6.4

In a python 3.10 venv, uv pip install --override override.txt . installs importlib-metadata==4.6.4, where I expect it not to, because my project does not depend on it for python >= 3.8.

This maybe raises interesting questions about the precise semantics of --override.

Comment by @charliermarsh on 2024-07-10 21:31

This is by design but we might want to change the design to be more like constraints (where we AND the markers). The current intent is that your override replaces all instances of the requirement verbatim. So you'd want the override to be importlib-metadata==4.6.4 ; python_version<'3.8'.
