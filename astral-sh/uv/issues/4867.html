`uv tool list` warns about malformed tool `interpreter-v2`

Closed #4867 opened by @oriori1703 on 2024-07-07 21:04
Issue opened by @oriori1703 on 2024-07-07 21:04

It seems that when running uv tool list it creates a directory called interpreter-v2 in uv's tool directory, i.e. /home/orip/.local/share/uv/tools/interpreter-v2.

This could be verified by running uv tool list && ls "$(uv tool dir)/interpreter-v2" . The folder seems to conain .msgpack files.

This also causes a the next time uv is run: warning: Ignoring malformed tool interpreter-v2: missing receipt
