Question/Bug: Can't specify exactly 3.14.0 as the Python version

#16175 opened by @Ahajha on 2025-10-08 05:58
Issue opened by @Ahajha on 2025-10-08 05:58

Summary

3.14.0 just released, I wanted to make a venv with the new release. Running (on Linux):

uv venv --python 3.14.0

gives

Using CPython 3.14.0rc3
Creating virtual environment at: .venv
Activate with: source .venv/bin/activate

Which is an RC. I also tried

uv venv --python ">3.14.0rc3"

but it still picks the same version.

Is there something I'm missing here, or is this a bug in the version resolution?

Platform

Ubuntu 22.04

Version

0.9.0

Python version

3.14.0

Comment by @zanieb on 2025-10-09 15:35

It's intentional that if you have previously installed an rc we won't upgrade you to a new Python version without an explicit install. We could make requesting 3.14.0 explicitly (rather than 3.14) fail to match pre-releases, though I worry that's breaking.

I've fixed the >=... bugs though.
