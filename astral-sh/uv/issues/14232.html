uv run --env-file=.env not overriding current env

#14232 opened by @Hellzed on 2025-06-24 08:36
@Hellzed
Question

Currently, when running uv run --env-file=.env, current env is not overridden.

Given the following .env file:

GREETING=hello

Running GREETING=bonjour uv run --env-file=.env will display

bonjour

Is this the expected behaviour? Doc doesn't make it explicitly cleat that the env is not overridden, and this can be surprising behaviour when comparing with similar tools (as using --env-file will not grant you a clean slate if you've previously exported env vars and forgot about it).

Platform

Linux 6.14.0-22-generic x86_64 GNU/Linux

Version

uv 0.7.14+2 (aa2448ef8 2025-06-23)
