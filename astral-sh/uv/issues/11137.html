Build backend should not expand recursive extras

#11137 opened by @charliermarsh on 2025-01-31 18:01
Issue opened by @charliermarsh on 2025-01-31 18:01

I asked about this here. There wasn't consensus either way, but most people suggested it was fine for a backend to expand these (or not). I think we should not expand them, since it doesn't match the metadata you get from reading the pyproject.toml directly and it's actually sort of a lossy transform (you can't identify a self-referential extra afterwards).
