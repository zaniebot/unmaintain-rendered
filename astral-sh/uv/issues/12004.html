The Keyring 1Password backend

This is beyond the scope of the uv project, but some wider context might be helpful.

Details

We tried to use Keyring with the 1Password third party backend mentioned on the keyring page - onepassword-keyring - but that implementation was apparently never meant for anything more than personal use by its author. It is currently dysfunctional - it never returns any credentials, but only a [use 'op item get <item ID> --reveal' to reveal] message - unless its source code is monkey-patched.

There are two public forks, one of which fixed the problem mentioned, but never published that fix as a package.

The alternative we are currently using is keyrings.onepassword; we forked that project and added some logging to a file: keyring and its backends communicate over stdout, so can't just use print(...) for logging.

As uv works with index urls, but 1Password works with vault items, we also added a rudimentary map to translate the former to the latter.

Finally, to reduce the effort to reproduce this issue to the least amount possible, we removed the need to install the 1Password application, the 1Password CLI, set up any items in any 1Password vaults or pass any environment variables.

This is just an awful hack. But maybe it helps us to fix this issue.

See https://github.com/DesmoDyne/keyrings.onepassword for details.
