# subsequent image builds download numerous dependencies

_Issue opened by @newne on 2025-05-30 02:03_

### Summary

When using uv to build docker images, even when only the version number changes in pyproject.toml (via uv version bump in CI/CD), subsequent image builds still download numerous dependencies. The current Dockerfile implementation:
```
# Stage 1: UV installation
FROM ghcr.io/astral-sh/uv:0.7.8 AS uv
# Stage 2: Build stage
FROM python:3.10.14-slim AS builder

WORKDIR /app
# Create and activate virtual environment
RUN --mount=from=uv,source=/uv,target=./uv \
    ./uv venv /opt/venv

ENV VIRTUAL_ENV=/opt/venv \
    PATH="/opt/venv/bin:$PATH" \
    LANG=C.UTF-8 \
    LC_ALL=C.UTF-8 \
    TZ=Asia/Shanghai \
    UV_HTTP_TIMEOUT=100 \
    UV_LINK_MODE=copy
#
COPY pyproject.toml .
COPY uv.lock .
# Install dependencies and cleanup in one layer
RUN --mount=type=cache,target=~/.cache/uv  \
    --mount=from=uv,source=/uv,target=./uv \
    ./uv sync --active --frozen --no-install-project --no-dev -v

# Stage 3: Runtime stage
FROM python:3.10.14-slim AS runtime

# Set runtime environment variables
ENV VIRTUAL_ENV=/opt/venv \
    PATH="/opt/venv/bin:$PATH" \
    PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    PYTHONHASHSEED=random \
    # Add security related env vars
    LANG=C.UTF-8 \
    LC_ALL=C.UTF-8 \
    TZ=Asia/Shanghai

RUN echo "" > /etc/apt/sources.list
RUN echo "deb http://mirrors.aliyun.com/debian buster main" >> /etc/apt/sources.list; \
    echo "deb http://mirrors.aliyun.com/debian-security buster/updates main" >> /etc/apt/sources.list ;\
    echo "deb http://mirrors.aliyun.com/debian buster-updates main" >> /etc/apt/sources.list ;
RUN apt-get update
RUN apt-get install libgomp1 --assume-yes

WORKDIR /app

COPY --from=builder  /opt/venv /opt/venv
COPY dist/src/ ./
COPY  docker/run.sh ./

CMD ["bash", "run.sh"]
```
