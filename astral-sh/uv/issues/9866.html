uv pip installs aarch64 package on 32-bit raspberry pi os, uv python3 -m pip installs correct version (gnueabihf / armv7)

#9866 opened by @elcojacobs on 2024-12-13 12:17
Issue opened by @elcojacobs on 2024-12-13 12:17

I have an issue installing the pydantic-core package on a raspberry pi 4 (64-bit) running a 32-bit os. The kernel is 64-bit: uname -a gives Linux pi32 6.6.51+rpt-rpi-v8 #1 SMP PREEMPT Debian 1:6.6.51-1+rpt3 (2024-10-08) aarch64 GNU/Linux But user land is 32 bit. dpkg --print-architecture gives armhf

When I my package with uv pip install --reinstall --index=https://www.piwheels.org/simple git+https://github.com/brewblox/brewblox-ctl@edge --index-strategy unsafe-best-match

I get the following package for pydantic-core:

pi@pi32:~/brewblox $ ls .venv/lib/python3.11/site-packages/pydantic_core
__init__.py  _pydantic_core.cpython-311-aarch64-linux-gnu.so  _pydantic_core.pyi  core_schema.py  py.typed

This gives an import error for pydantic:

pi@pi32:~/brewblox $ uv run python3
Python 3.11.2 (main, Sep 14 2024, 03:00:30) [GCC 12.2.0] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> import pydantic_core
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
  File "/home/pi/brewblox/.venv/lib/python3.11/site-packages/pydantic_core/__init__.py", line 6, in <module>
    from ._pydantic_core import (
ModuleNotFoundError: No module named 'pydantic_core._pydantic_core'
>>>

A workaround is to use regular pip instead of uv pip, which does install the package for the correct platform:

pi@pi32:~/brewblox $ uv pip install pip

Installed 1 package in 975ms
