Summary

Please consider publishing the new astral-tokio-tar fork on crates.io as https://crates.io/crates/astral-tokio-tar.

Example

This would allow the git dependency

https://github.com/astral-sh/uv/blob/748582ee6f3ff1007692f131fa4b59d7c100ec66/Cargo.toml#L76

to become a version-based one.

As maintainer of the uv package in Fedora, this would allow to me add a rust-astral-tokio-tar package to Fedora, rather than bundling an additional source archive in uv and patching Cargo.toml to convert the git dependency into a path dependency. That’s permitted and feasible – I am already doing it for the async_zip, pubgrub, and tl forks – but each additional dependency of that nature adds slightly but noticeably to the complexity of the package and the effort required to maintain it.

(I assume your fork might also be useful to other projects with similar requirements.)
