Can't install package with dependency group without editable mode

#1815 opened by @noaonoszko on 2024-02-21 14:55
@noaonoszko
uv version

0.1.6

Platform

Darwin

Description

This works:

pip install ".[dev]"

and this works:

uv pip install -e ".[dev]"

but this doesn't:

uv pip install ".[dev]"

I get

 uv::requirements::from_source source=.[dev]
error: Failed to parse `.[dev]`
  Caused by: Expected package name starting with an alphanumeric character, found '.'
.[dev]
^

when running with --verbose.

I'm installing my package from a pyproject.toml and the dependency groups are defined under [project.optional-dependencies].

Comment by @charliermarsh on 2024-02-21 14:59

Thanks, this is a duplicate of astral-sh/uv#313. You can use uv pip install "package-name @ .[dev]" for now, but we'll lift this restriction in the future.

Closed by @charliermarsh on 2024-02-21 14:59
Comment by @charliermarsh on 2024-03-22 21:26

This is supported as of v0.1.24. You can uv pip install .[dev] directly, without including a package name.
