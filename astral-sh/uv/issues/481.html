Move simple index queries to `CachedClient`

#481 opened by @konstin on 2023-11-21 16:17
Issue opened by @konstin on 2023-11-21 16:17

Currently simple index queries use the http_cache. We should move them to CachedClient. This allows merging different types of index response before caching, sharding by index (#480), saving a smaller response (we can filter it down to wheels, source dists and sentinel versions) and makes optimizations for much faster cache reading and writing possible.
