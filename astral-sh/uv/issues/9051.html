Comment by @FishAlchemist on 2024-11-12 11:25

I think you've got it wrong. This is because uv run by default checks if the lock file is up-to-date. The lock file is used to find cross-platform compatible dependencies that meet your requirements. https://docs.astral.sh/uv/concepts/projects/#project-lockfile

Comment by @konstin on 2024-11-12 11:53

If your project does only support linux, you can set tool.uv.environments to e.g. exclude windows.

On uv's side, we shouldn't try resolving (python_full_version == '3.11.*' and platform_system == 'Windows' and sys_platform == 'darwin'): platform_system == 'Windows' and sys_platform == 'darwin' doesn't exist, and we shouldn't consider it.
