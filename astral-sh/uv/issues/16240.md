# Mirror CPython and support ùúãthon in 3.14 venvs

---

_Issue opened by @TimPansino on 2025-10-10 20:36_

### Summary

In CPython 3.14's `venv` module there is a new alias for the `python` interpretter alongside `python3` or `python3.14` called `ùúãthon`. This is an easter egg for this minor version of CPython, and it would be neat if `uv` also added this alias.

In the spirit of the original easter egg, would probably be best to leave it out of the release notes and just quietly add it.

Original Easter Egg: https://github.com/python/cpython/pull/125035

### Example

With `venv`

```bash
python3.14 -m venv .venv
```

Or with `uv`

```bash
uv venv -p 3.14 .venv
```

Would produce this list of executables in the `.venv/bin/` directory

```bash
$ ls .venv/bin/
activate                activate.fish           pip                     python
activate_this.py        activate.nu             pip3                    python3
activate.bat            activate.ps1            pip3.14                 python3.14
activate.csh            deactivate.bat          pydoc.bat               ùúãthon
```

_Comment by @edmorley on 2025-10-11 09:12_

One downside to uv supporting this, is that uv would have to keep support for this for many more years than the upstream CPython project does, due to uv supporting historic versions of Python. (ie: uv will likely need to support Python 3.14 for many years after 3.14 reaches upstream EOL)

In contrast, the upstream CPython project was already able to remove support from `main` (where Python 3.15 is now being developed), so won't be carrying the feature in any newer branch:
https://github.com/python/cpython/pull/134037
