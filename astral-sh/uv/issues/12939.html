Comment by @zanieb on 2025-04-17 21:38

I would recommend using uv init --script <path> and uv add --script <path> <dependency> for script dependencies and uv init / uv add <dependency for multifile projects. A global, mutable environment leads to a fair number of problems and if you just specify your dependencies where you need them, you should have an easier time running things when you come back to them in the future.

If you really want a set of shared dependencies, I'd maybe do uv init ~/shared then uv add --project ~/shared httpx pandas then you can do:

❯ uv init foo
❯ cd foo
❯ uv add ~/shared

etc.
