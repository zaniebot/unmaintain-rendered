uv 0.2.32 ignores development resolutions when running `pip compile`

#5729 opened by @A5rocks on 2024-08-02 15:17
Issue opened by @A5rocks on 2024-08-02 15:17

The issue title isn't great, I'm not sure how to best describe this. However, here's a reproduction:

In a.in:

cffi

In a.txt:

# This file was autogenerated by uv via the following command:
#    uv pip compile a.in -o a.txt
cffi==1.17.0rc1
    # via -r a.in
pycparser==2.22
    # via cffi

Now, run uv pip compile a.in -o a.txt and note that a.txt was modified to become:

# This file was autogenerated by uv via the following command:
#    uv pip compile a.in -o a.txt
cffi==1.16.0
    # via -r a.in
pycparser==2.22
    # via cffi

Note that I'm not passing any sort of upgrade flag. This did not happen in uv==0.2.26. I can reproduce this on both Debian Linux and Windows. Found in https://github.com/python-trio/trio/pull/3043.
