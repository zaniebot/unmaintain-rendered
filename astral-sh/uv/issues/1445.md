# Request: Support uv venv --prompt 

---

_Issue opened by @frague59 on 2024-02-16 07:08_

Hi,
`venv` has an mainly undocumented feature :

```sh
$ python3.9 -m venv --prompt=<my prompt> ./venv
```
This command creates the virtuelenv, and append a prompt parameter ti the `pyvenv.cfg` file : 

```sh
$ cat ./venv/pyvenv.cfg
home = /usr/local/bin
include-system-site-packages = false
version = 3.9.16
prompt = '<my prompt>'
```

This option is useful when you use several virtualenvs, as the active venv prompt can be displayed by the (linux) prompt, like [liquidprompt](https://liquidprompt.readthedocs.io/en/stable/) or [starship](https://starship.rs/)

Thanks !
