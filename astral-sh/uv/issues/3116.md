# uv powershell windows only works if you are running powershell as admin

---

_Issue opened by @Chris2L on 2024-04-18 05:51_

Trying to run uv using VS code on windows.  Install uv on powershell using command 
`powershell -c "irm https://astral.sh/uv/install.ps1 | iex"`
This installs version 0.1.33

If I try to create venv running command:
`uv venv`
powershell responds with 

> uv : The term 'uv' is not recognized as the name of a cmdlet, function, script file, or operable program.

If I install and run powershell as admin, it seems to work fine while still in admin.  This should not be the way to use it as running things with admin should be reserved to extreme cases. 

Running windows 11
