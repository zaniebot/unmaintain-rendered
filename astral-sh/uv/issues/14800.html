uv pip compile ignores dependencies specified in uv.toml

#14800 opened by @jelle-openai on 2025-07-21 21:25
@jelle-openai

Summary

When pip.dependency-metadata is used in uv.toml to override a package's dependencies, uv pip compile treat the package as if it has no dependencies, instead of using the dependencies specified in uv.toml.

% cat requirements.in 
protovalidate==0.8.0
% cat my-uv.toml 
[pip]
dependency-metadata = [
    {name = "cel-python", version = "0.3.0", requires_dist = ["lark>=0.12", "jmespath>=1.0.1", "pendulum>=3.1", "pyyaml>=6"]},
]
% uv pip compile requirements.in
