# How to give "uv tool" access to system site packages similar to "pipx"?

---

_Issue opened by @akkadaya on 2025-06-01 00:41_

### Question

I'm trying to use `uv` to install Chirp software following [this guide](https://chirpmyradio.com/projects/chirp/wiki/ChirpOnLinux#2-Install-CHIRP-and-Python-dependencies)

I'm facing the issue where it can't find the module `wxPython` although it's already installed on Fedora.
`pipx` has the option to give access to system site packages, how do I do it with `uv`?

### Platform

Fedora 42 x86_64

### Version

uv 0.7.9 (Homebrew 2025-05-30)

---

_Label `question` added by @akkadaya on 2025-06-01 00:41_

---

_Comment by @zanieb on 2025-06-01 04:47_

I'd recommend including the dependency in the environment, e.g., `uv tool install --with wxPython ...`

We don't support using system site packages in tools yet. I'd generally prefer not to unless there's a use-case that cannot be solved by an isolated environment.

_Comment by @00sapo on 2025-09-17 07:57_

I don't think this is a solution. Some python packages are integrated in the OS and is much easier to give access to system site packages than building from scratch (even if it is possible). There is `--system-site-packages` for venvs, but it's not enabled for the `uv tool` CLI.

A workaround is using `--python /usr/bin/python3` and then setting `system-site-packages = true` in `pyvenv.cfg`:

```
uv install --python /usr/bin/python3 <your-package>
sed -i 's/system-site-packages = false/system-site-packages = true/' $(uv tool dir)/<your-package>/pyvenv.cfg
```
