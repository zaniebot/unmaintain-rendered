`uv pip compile` ignores `--quiet` / `-q` flag.

#2040 opened by @Wim-De-Clercq on 2024-02-28 09:24
Issue opened by @Wim-De-Clercq on 2024-02-28 09:24

Reproduce

pyproject.toml:

[project]
version = "0.0.1"
name = "my_package"
dependencies = [
    "uv==0.1.11",
]

command:

uv pip compile --quiet -o requirements.txt pyproject.toml

output:

# This file was autogenerated by uv via the following command:
#    uv pip compile --quiet -o requirements.txt pyproject.toml
uv==0.1.11

+ requirements.txt generated.

info

Behaviour: The whole output file gets printed on console Expected: No output, only a generated requirements.txt platform: linux, ubuntu 22 uv version: 0.1.11

Label bug added by @konstin on 2024-02-28 09:33
Comment by @Wim-De-Clercq on 2024-02-28 09:50

Update: after more debugging, it seems a lingering 0.1.10 install was still installed from cargo.

After removing all PATH changes to include .cargo/bin in .profile and .bashrc, this issue appears to be resolved in 0.1.11 installed via pip.

I will close this for now.
