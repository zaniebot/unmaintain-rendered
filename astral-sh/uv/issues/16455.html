uv pip install fails to build certain packages on Ubuntu 24.04 when Python is invoked via nested symlinks

#16455 opened by @desertfury on 2025-10-26 21:03
Issue opened by @desertfury on 2025-10-26 21:03

Summary

Example command to reproduce on Ubuntu 24.04

twix-python -m uv pip install bravado

Expected output

bravado (and dependent packages) build successfully

Actual output

Build fails with an error: ModuleNotFoundError: No module named 'math'

Structure of interpreter symlinks

/usr/bin/twix-python
  -> /etc/alternatives/twix-python
    -> /usr/bin/twix-python3.12
      -> ../twix/python3.12/bin/twix-python

Additional info

Detailed repro with inputs and outputs of bash command could be found here

Example on building python deb package with symlinks structure that will certainly break installing packages via uv could be found here

Notes

Problem does not reproduce if you use pip instead uv Problem does not reproduce if you use Ubuntu20.04 (both python 3.11 and python 3.12 works) Problem does not reproduce if you use 3.11 version of python on Ubuntu 24.04 Problem does not reproduce if you replace path to symlink with realpath to binary

twix-python -m uv pip install bravado  # fails
$(realpath `which twix-python`) -m uv pip install bravado # works

Platform

Ubuntu 24.04

Version

0.9.5

Python version

Python 3.12.8
