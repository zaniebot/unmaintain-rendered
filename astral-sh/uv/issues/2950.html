Avoid password in verbose output

#2950 opened by @avelychko12 on 2024-04-09 23:49
Issue opened by @avelychko12 on 2024-04-09 23:49

If you run uv pip install with --verbose it will show you password in url

export UV_PIP_EXTRA_INDEX_URL="https://oauth2accesstoken:my_secret_token@europe-python.pkg.dev/project/pypi/simple/"                                                                         

uv pip install package --system --verbose                                                                                                                                                 0 
DEBUG Starting interpreter discovery for default Python
DEBUG Probing interpreter info for: ...
DEBUG Found Python 3.11.2 for: ...
DEBUG Using Python 3.11.2 environment at ...
DEBUG Trying to lock if free: /tmp/uv-e5f656fc5acf8d1c.lock
DEBUG Using registry request timeout of 300s
DEBUG Solving with target Python version 3.11.2
DEBUG Adding direct dependency: package*
TRACE Fetching metadata for package from https://oauth2accesstoken:my_secret_token@europe-python.pkg.dev/project/pypi/simple/package/
...

pip will always show you https://oauth2accesstoken:****@europe-python.pkg.dev/project/pypi/simple/ For me, it looks like potential security issue, similar to what sqlalchemy had https://data.safetycli.com/v/51668/97c/
