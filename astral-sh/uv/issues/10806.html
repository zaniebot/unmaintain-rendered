Support for PEP-compliant `dev` extra for optional dev dependencies

Closed #10806 opened by @aaronsteers on 2025-01-21 03:44
Issue opened by @aaronsteers on 2025-01-21 03:44

There seems to be a gap in PEP 508 and PEP 621 related to how dev dependencies should be declared.

All the tools, Poetry, uv, etc. each have their own proprietary declaration syntaxes. One idea gaining traction (pending a new PEP that adds a uniform declaration syntax) is to standardize around an optional "dev" extra.

Is it acceptable to declare dependencies using an extra simply called 'dev'? Any great alternatives or suggestions? Thanks!

Label question added by @konstin on 2025-01-21 08:14
Comment by @konstin on 2025-01-21 08:14

PEP 735 has recently added dependency groups, which can be used to specify development dependencies (https://docs.astral.sh/uv/concepts/projects/dependencies/#dependency-groups).

Comment by @charliermarsh on 2025-01-21 14:06

That's right -- dependency groups are designed for this!
