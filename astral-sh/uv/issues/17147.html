Hello,

I've observed that uv tool installs tools from alpha versions of the package.

My understanding is that installing a package will ignore alpha versions if not explicitly requested. I would expect similar behavior from uv tool. This would allow for testing apps before they are ready for general consumption.

I haven't actually published any non-alpha version of this particular package. I would expect uv tool to fail in that case, with a standard error message

Comment by @konstin on 2025-12-16 14:00

This is a special rule in the default if-necessary-or-explicit strategy (https://docs.astral.sh/uv/reference/settings/#prerelease): If there a no stable releases, use the latest unstable release. This avoid uv tool install from failing when it's clear that the users want the unstable version. This rule was initially inspired by black, which was publishing only prereleases for some time, where users still wanted to install and use black anyway.

Usually the gating is that people need to request foo specifically in uv tool install foo, at least I'm not aware of specific instances when excluding alpha packages would have prevented a specific problem. Once you made the first non-dev release, people won't be upgraded to pre-releases automatically.
