Comment by @charliermarsh on 2024-08-22 12:39

If you use requests==2.0.0, then we add requests==2.0.0 to your dependencies. uv lock --upgrade won't change any dependency definitions, it will just upgrade the lockfile to the latest compatible versions. But given requests==2.0.0, 2.0.0 is the latest compatible version you could ever get (since, e.g., 2.0.1 does not equal 2.0.0).

I'd suggest uv add "requests>=2.0.0" or similar. Does that make sense?
