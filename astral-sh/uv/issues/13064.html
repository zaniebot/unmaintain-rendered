uv add adds [[tool.uv.index]] entry when UV_INDEX is set

#13064 opened by @sebastian-cradle on 2025-04-23 08:58
@sebastian-cradle
Summary

This is likely related to: astral-sh/uv#12483

Background/Context: we have a mono-repo with multiple projects and host our packages in an internal repo. In order to not have to replicate the index configuration in each pyproject.toml, we set the UV_INDEX env variable (we do this via mise).

Now when adding a package via uv add, e.g.:

uv add "my-internal-package>=0.0.1"

The add command also adds a [[tool.uv.index]] section to the pyproject.toml pointing to the index specified via the UV_INDEX env variable.

Is there a way to have the add command not do that?
