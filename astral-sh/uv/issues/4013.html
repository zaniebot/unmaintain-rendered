No warning is output even if parsing the workspace settings fails.

Closed #4013 opened by @Di-Is on 2024-06-04 12:45
Issue opened by @Di-Is on 2024-06-04 12:45

The implementation has been designed to output a warning if the parsing of the workspace configuration files (pyproject.toml/uv.toml) fails.

https://github.com/astral-sh/uv/blob/main/crates/uv-workspace/src/workspace.rs#L38-L61

However, no warnings are actually output.

pyproject.toml

[project]
name = "sample"
version = "0.0.0"
dependencies = ["ruff"]

[tool.uv.pip]
# originally string type.
index-url = 1

command result

uv pip compile pyproject.toml 

Resolved 1 package in 383ms
# This file was autogenerated by uv via the following command:
#    uv pip compile pyproject.toml
ruff==0.4.7
    # via sample (pyproject.toml)

Specifying a URL that does not exist in tool.uv.pip.index-url will cause an error, which suggests that the configuration value itself is being read.

pyproject.toml

[project]
name = "sample"
version = "0.0.0"
dependencies = ["ruff"]

[tool.uv.pip]
# originally string type.
index-url = "http://aaa"

command output

uv pip compile pyproject.toml 

⠼ Resolving dependencies...
error: Could not connect, are you offline?
  Caused by: error sending request for url (http://aaa/ruff/)
  Caused by: client error (Connect)
  Caused by: dns error: failed to lookup address information: No address associated with hostname
  Caused by: failed to lookup address information: No address associated with hostname
