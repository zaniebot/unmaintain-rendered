The problem is that

uv add paddlepaddle-gpu==3.0.0 --default-index https://www.paddlepaddle.org.cn/packages/stable/cu126/

set the paddlepaddle index as default index in `pyproject.toml, so later

uv add paddleocr

uses it too, but paddleocr is only on PyPI not on the paddlepaddle index. This is different from the pip commands, where the index is only used for the first command.

The problem is the index configuration: https://www.paddlepaddle.org.cn/packages/stable/cu126/paddleocr/ returns a 200 OK response, even though it doesn't have the package. The index returns status 200 for any random package name, so uv doesn't know that the index does not have the package. You can use uv add paddleocr --index-strategy unsafe-best-match to work around this, but usually, we don't recommend using unsafe- options.
