# git tag not preserved on compile requirements, converted to SHA

The git tag is not preserved when compiling (unless I missed a config option) -- This is also an incompatibility with `pip compile` which preserves this information.

for example
```
echo "uv @ git+https://github.com/astral-sh/uv.git@0.2.5" > r.txt

uv pip compile r.txt
```

results in:
```
# This file was autogenerated by uv via the following command:
#    uv pip compile r.txt
uv @ git+https://github.com/astral-sh/uv.git@47db418ba26dd4b720ec968b2520f2483842419d
    # via -r r.txt
```

The above should be:
```
# This file was autogenerated by uv via the following command:
#    uv pip compile r.txt
uv @ git+https://github.com/astral-sh/uv.git@0.2.5
    # via -r r.txt
```
