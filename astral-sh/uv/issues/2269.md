I upgraded my uv install inside a conda environment on Windows (from 0.1.11 to 0.1.15) to try to fix an issue with entry points. However after the upgrade, uv failed with:
```
uv pip install pyvisa-sim
 uv::requirements::from_source source=pyvisa-sim
    0.003712s DEBUG uv_interpreter::python_environment Found a virtualenv through CONDA_PREFIX at: C:\xxxxxx\miniconda3\envs\charac
    0.004238s DEBUG uv_interpreter::interpreter Probing interpreter info for: \\?\C:\xxxxxxxx\miniconda3\envs\charac
error: Failed to query python interpreter `\\?\C:\xxxxxxxx\miniconda3\envs\charac`
  Caused by: Access is denied. (os error 5)
```
I then downgraded and found that 0.1.13 is the last version that works (install, I did not check the entry_points yet).
