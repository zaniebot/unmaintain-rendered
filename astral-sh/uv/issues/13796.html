Add a benchmark for a large workspace

Open #13796 opened by @konstin on 2025-06-03 08:27
Issue opened by @konstin on 2025-06-03 08:27

To keep uv run usable in large projects, we need to keep the overhead low even if there are many projects in a workspace (see e.g. https://github.com/astral-sh/uv/pull/12096). This overhead may be parsing pyproject.tomls, parsing uv.lock, suboptimal directory traversal or quadratic freshness check. We should add a benchmark the performs warm cache uv sync and uv run in a large project such as airflow to continuously measure this overhead.
