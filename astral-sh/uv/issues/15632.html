Q1. Is it normal "right" that the non-dev dependency yt-dlp isn't in a group too?

Yes.

Q2. I ran uv add ut-dlp and not uv add --script scripts/transcript_auto_fetcher.py yt-dlp and have nothing special at the top of my transcript-auto-fetcher.py script. Is this okay like this?

Yes, scripts are mainly used outside projects.

Q3. When I run uv lock --upgrade does this upgrade all my project dependencies to the latest stable availible versions inside my virtual environment? And if so, then why don't the version numbers in my pyproject.toml file update too (as, is the a reason)? As an example:

Your looking for uv sync --upgrade.

Q4. If I wanted to update the numbers to match what's actually in my virtual environment (e.g., "yt-dlp>=2025.8.11"), what's the best way to do this? ... more importantly, is this good to do?

There's an open feature request for automating this: astral-sh/uv#6794.

For libraries, you want wide version bounds in pyproject.toml, for applications, you can have tight version bounds because you have uv.lock
