Comment by @kahojyun on 2024-10-23 06:51

Edited: This is a known limitation of uv: https://docs.astral.sh/uv/pip/compatibility/#local-version-identifiers

It looks like uv doesn't follow the python packaging version specifiers spec

If the specified version identifier is a public version identifier (no local version label), then the local version label of any candidate versions MUST be ignored when matching versions. If the specified version identifier is a local version identifier, then the local version labels of candidate versions MUST be considered when matching versions, with the public version identifier being matched as described above, and the local version label being checked for equivalence using a strict string equality comparison.

+cpu is a local version label which should be ignored when matching against pytorch==2.5.0, so installing pytorch==2.5.0 should select wheel with version 2.5.0+cpu when using https://download.pytorch.org/whl/cpu as index-url.
