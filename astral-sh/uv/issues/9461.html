Comment by @samypr100 on 2024-12-01 14:50

Generally settings like http_proxy or https_proxy should not be part of VCS as it can pose a security risk especially if it would now allow it to be defined on OSS projects. This type of setting goes better on a system / user level config like rye's config.toml.

Comment by @Mai0313 on 2024-12-03 02:28

Generally settings like http_proxy or https_proxy should not be part of VCS as it can pose a security risk especially if it would now allow it to be defined on OSS projects. This type of setting goes better on a system / user level config like rye's config.toml.

Thank you for your input. I understand the security concerns associated with including settings like http_proxy or https_proxy in VCS, especially for open source projects. However, I believe it could be beneficial to support these settings in both config.toml and pyproject.toml.

For open source projects, it can be unsafe to use these settings in pyproject.toml since it is publicly accessible. But for private projects within a company, this security issue is less relevant because the projects are not accessible externally. In environments with strict PIM policies, having this flexibility can be quite useful. Your thoughts on this would be greatly appreciated.
