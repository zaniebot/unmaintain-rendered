authentication using SSH: failed to parse url to GitLab repo

Closed #11983 opened by @MttBallu on 2025-03-05 16:54
Issue opened by @MttBallu on 2025-03-05 16:54

Summary

I am trying to add a git dependency source where the remote repo is stored on an instance of Git Lab by running : uv add git+ssh://git@<gitlab_instance>:<my_username>/<path_to_repo>.git but I get the following error :

error: Failed to parse: Caused by: invalid port number

But I get no error when using git+https://<gitlab_instance>:<my_username>/<path_to_repo>.git. I have no issue when I manually clone the repo using ssh.

Is there any extra information to provide when using ssh over https in the uv add command ?
