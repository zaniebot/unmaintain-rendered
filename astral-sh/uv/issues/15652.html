Comment by @konstin on 2025-09-03 10:20

The problem described in pyright's documentation to my best knowledge is only a problem with setuptools, other build backends default to .pth files. The advice they share is for the case of using uv a build frontend (e.g. uv sync, uv pip install -e .) with setuptools as backend (build-backend = "setuptools.build_meta"), if you're using the uv_build backend (with any frontend), or uv with a different build backend, you are not affected. See also https://github.com/astral-sh/uv/issues/1708#issuecomment-2174463476 and https://github.com/astral-sh/uv/issues/3898. I proposed some changes to the pyright docs to clarify this: https://github.com/microsoft/pyright/pull/10888
