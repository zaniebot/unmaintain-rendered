Sorry for not answering earlier, I try to explain my need.

I have a lot of scripts on a git repository, which use as a dependency some classes developed by some colleagues (not packages) that are hosted in a different repository which I use as a sub-module of mine (e.g. in a subfolder ./ext/fancy-classes with respect to my scripts contained in ./python/ directory of my repo)

To avoid setting PYTHONPATH in the environment each time, currently with python -m venv, I am adding the directories where these directories in a text file in "..venv\Lib\site-packages"

Now I discovered this wonderful tool named uv, that dynamically manages and creates the venv way better, but my "workaround" is definitely weaker with this tool.

Since it has so many amazing features, I was thinking that there could be a way to set the PYTHONPATH in the pyproject.toml file, so that with uv run all my scripts simply work.

Hope that this better explains my need.
