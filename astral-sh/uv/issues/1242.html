Make --no-index behavior consistent between pip sync and pip install

#1242 opened by @charliermarsh on 2024-02-03 16:45
@charliermarsh

If you run pip sync --no-index, the install plan will allow you to use cached distributions. But if you run pip install --no-index, it will fail (unless you're using local dependencies of --find-links, etc.), since we won't be able to find the metadata for the package you're installing.

We should probably have a consistent experience here. Arguably, --no-index shouldn't work in either case? We could also consider adding an --offline option to force reading from the cache (but still respecting indexes).
