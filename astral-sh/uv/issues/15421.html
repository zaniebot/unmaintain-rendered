Comment by @konstin on 2025-08-21 17:29

This is unfortunately a known problem with the PyQt5 package, which only publishes specific platforms in some versions (there's some previous discussion: https://github.com/astral-sh/uv/issues?q=is%3Aissue%20PyQt5, and see specifically https://github.com/astral-sh/uv/issues/11865 for the Windows case), and the workaround is to set tool.uv.required-environments to include your platform.

    "pyqt5-qt5==5.15.2",
]
