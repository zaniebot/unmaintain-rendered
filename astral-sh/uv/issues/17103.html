For ISO 8601 durations, new data type duration is available after draft 2019-09.

However, the friendly durations accepted by jiff::fmt::friendly are too complex to define as regular expressions.

Comment by @BurntSushi on 2025-12-12 17:46

I'm not terribly familiar with what having precise types in a JSON Schema gives you. The "friendly" format isn't a widely recognized standard and representing it with a succinct regex is probably not possible, so you'll likely need string somewhere.

I do believe the friendly format can be represented as a single regex, but it would be quite large. (About as big as the friendly format grammar.)
