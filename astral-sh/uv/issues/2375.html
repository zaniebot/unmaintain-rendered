URL Environment Variable Substitution ordering

#2375 opened by @matty-rose on 2024-03-12 04:08
@matty-rose

Description

Environment variable substitution only happens after matching the scheme of the provided url for a dependency in a requirements file, which means a URL like ${LIBRARY_URL} gets matched as a file and errors out. If we change the variable in the requirements file to https://${LIBRARY_URL} and remove the https:// from the variable value itself, uv works as expected.

assuming I'm looking at the right code: parse_url calls preprocess_url which matches on scheme, but since the env var isn't expanded yet, the https:// isn't there so we hit this else branch which attempts to parse as a path
