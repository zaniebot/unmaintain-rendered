Enhancement: Allow specifying --groups via environment variable UV_DEFAULT_GROUPS

#16735 opened by @turbotimon on 2025-11-14 09:29
@turbotimon
Summary

I suggest an environment variable UV_DEFAULT_GROUPS (or similar) that overwrites the default groups

Motivation

In a project with many groups, where different team members need different sets of group for development (e.g. a monorepo), it should be possible to configure the default groups individually. Therefore, it is not necessary to specify --group x --group y ... ... for every uv sync or uv run command. In this scenario, it is not possible to alter the pyproject.toml as this would change the behavior for all team members.

The UV_DEFAULT_GROUPS="x, y, z" environment variable could then be e.g. set in an [.env]

Current workaround

Best workaround I came up with, so fare, is make an alias alias uv-sync="uv sync --group x --group y --group z". Or is there already another solution?

Related

Somewhat related to #15009

Example 
export UV_DEFAULT_GROUPS="x, z"
uv sync 
uv run ...
``
