Comment by @FishAlchemist on 2024-07-06 10:51

Virtual environment cannot use the virtual environment for all cells just using source command. If you need a virtual environment because UV is installed into a virtual environment by default. You can add --system flag. e.g.

!uv pip install --system [packages]

https://github.com/astral-sh/uv/blob/main/PIP_COMPATIBILITY.md#virtual-environments-by-default

Flag information in uv 0.2.21 (ebfe6d8fc 2024-07-03)

      --system
          Install packages into the system Python.

          By default, `uv` installs into the virtual environment in the current working directory or any parent
          directory. The `--system` option instructs `uv` to instead use the first Python found in the system `PATH`.

          WARNING: `--system` is intended for use in continuous integration (CI) environments and should be used with
          caution, as it can modify the system Python installation.

          [env: UV_SYSTEM_PYTHON=]
