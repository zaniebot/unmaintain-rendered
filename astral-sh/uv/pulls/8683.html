Simplify pep440 -> version ranges conversion

#8683 opened by @konstin on 2024-10-29 20:26
@konstin

With #8669 as preparation, we can remove what used to be the PubGrubSpecifier and instead implement a direct conversion from VersionSpecifiers to Range<Version>. Thanks to astral-sh/uv#8669#discussion_r1821241638, this conversion is infallible, removing a lot of fallibility downstream.

The conversion takes an owned value, to signal that we consume the value. Previously, the function had cloned the version internally.
