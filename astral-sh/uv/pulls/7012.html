uv/tests: filter out link mode warning in one test

#7012 opened by @BurntSushi on 2024-09-04 12:44
@BurntSushi

In the lock_redact_https test specifically, it prompts a link mode warning from uv on my system. Debugging seems to suggest it is provoked by attempting to hardlink between /tmp and ~/.local. Since these are on different file systems for me (with /tmp being a ramdisk), it provokes the warning, and this turn spoils the snapshot when running tests locally.

This PR adds a test specific filter rule to fix this.
