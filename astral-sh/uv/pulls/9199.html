Remove separate test files in favor of same-file mod tests

#9199 opened by @charliermarsh on 2024-11-18 14:21
@charliermarsh

Summary

These were moved as part of a broader refactor to create a single integration test module. That "single integration test module" did indeed have a big impact on compile times, which is great! But we aren't seeing any benefit from moving these tests into their own files (despite the claim in this blog post, I see the same compilation pattern regardless of where the tests are located). Plus, we don't have many of these, and same-file tests is such a strong Rust convention.
