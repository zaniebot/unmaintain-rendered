Always install as editable when duplicates are requested

#4208 opened by @charliermarsh on 2024-06-10 18:40
Pull request opened by @charliermarsh on 2024-06-10 18:40

Summary

If the user requests a package as both editable and non-editable, the editable now "wins".

Previously, pip install -e . . would install as editable. However, pip install -e . -r requirements.txt would not if requirements.txt contained ., because we ignored editable when deduplicating and the order of iteration was just dependent on internals.

Closes https://github.com/astral-sh/uv/issues/4053.
