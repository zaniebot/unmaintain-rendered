uv-client: switch to RFC 9110 compatible format

#8752 opened by @BurntSushi on 2024-11-01 12:34
@BurntSushi

This still utilizes the RFC 2822 datetime formatter, but utilizes new methods added in jiff 0.1.14 to emit timestamps in a format strictly compatible with RFC 9110.

It seems like most HTTP servers were pretty flexible and supported RFC 2822 datetime formats, but #8747 shows at least one case where that isn't true. Given that the MDN docs prescribe RFC 9110, we defer to them.
