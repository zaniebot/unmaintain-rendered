Allow conflicting locals when forking

#5104 opened by @ibraheemdev on 2024-07-16 14:18
@ibraheemdev
Summary

Currently, the Locals type relies on there being a single local version for a given package. With marker expressions this may not be true, a similar problem to astral-sh/uv#4435. This changes the Locals type to ForkLocals, which tracks locals for a given fork. Local versions are now tracked on PubGrubRequirement before forking.

Resolves astral-sh/uv#4580.
