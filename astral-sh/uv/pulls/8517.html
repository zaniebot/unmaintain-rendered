Comment by @konstin on 2024-10-24 16:28

According to https://gregoryszorc.com/docs/python-build-standalone/main/running.html:

x86_64_v2-*

Targets 64-bit Intel/AMD CPUs approximately newer than Nehalem) (released in 2008). Binaries will have SSE3, SSE4, and other CPU instructions added after the ~initial x86-64 CPUs were launched in 2003. Binaries will crash if you attempt to run them on an older CPU not supporting the newer instructions.

Requiring at least a cpu from 2008 is safe enough. I'd like to add cpu feature detection on top of that (e.g. avx2 for x86_64_v3), but this change in itself is a good idea.
