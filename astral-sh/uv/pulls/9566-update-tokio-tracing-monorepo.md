### [`v0.1.41`](https://redirect.github.com/tokio-rs/tracing/releases/tag/tracing-0.1.41): tracing 0.1.41

[Compare Source](https://redirect.github.com/tokio-rs/tracing/compare/tracing-0.1.40...tracing-0.1.41)

\[ [crates.io][crate-0.1.41] ] | \[ [docs.rs][docs-0.1.41] ]

This release updates the `tracing-core` dependency to [v0.1.33][core-0.1.33] and
the `tracing-attributes` dependency to [v0.1.28][attrs-0.1.28].

##### Added

-   **core**: Add index API for `Field` ([#&#8203;2820])
-   **core**: Allow `&[u8]` to be recorded as event/span field ([#&#8203;2954])

##### Changed

-   Bump MSRV to 1.63 ([#&#8203;2793])
-   **core**: Use const `thread_local`s when possible ([#&#8203;2838])

##### Fixed

-   Removed core imports in macros ([#&#8203;2762])
-   **attributes**: Added missing RecordTypes for instrument ([#&#8203;2781])
-   **attributes**: Change order of async and unsafe modifier ([#&#8203;2864])
-   Fix missing field prefixes ([#&#8203;2878])
-   **attributes**: Extract match scrutinee ([#&#8203;2880])
-   Fix non-simple macro usage without message ([#&#8203;2879])
-   Fix event macros with constant field names in the first position ([#&#8203;2883])
-   Allow field path segments to be keywords ([#&#8203;2925])
-   **core**: Fix missed `register_callsite` error ([#&#8203;2938])
-   **attributes**: Support const values for `target` and `name` ([#&#8203;2941])
-   Prefix macro calls with ::core to avoid clashing with local macros ([#&#8203;3024])
