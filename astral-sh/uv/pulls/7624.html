Avoid retaining forks when requires-python range changes

#7624 opened by @charliermarsh on 2024-09-22 17:21

Summary

If the requires-python bound expands, the space covered by resolution-markers may no longer include all supported Python versions. In such cases, we need to avoid reusing the forks (but we can reuse the preferred versions).

Closes astral-sh/uv#7618.
