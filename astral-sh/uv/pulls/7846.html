Allow dependency metadata entries for direct URL requirements

#7846 opened by @charliermarsh on 2024-10-01 15:35
Pull request opened by @charliermarsh on 2024-10-01 15:35

Summary

This is part of making https://github.com/astral-sh/uv/issues/7299#issuecomment-2385286341 better. You can now use tool.uv.dependency-metadata for direct URL requirements. Unfortunately, you must include a version, since we need one to perform resolution.

For example:

[project]
name = "project"
version = "0.1.0"
requires-python = ">=3.12"
dependencies = ["flash-attn"]

[tool.uv.sources]
flash-attn = { git = "https://github.com/Dao-AILab/flash-attention", tag = "v2.6.3" }

[[tool.uv.dependency-metadata]]
name = "flash-attn"
version = "2.6.3"
requires-dist = ["torch", "einops"]
