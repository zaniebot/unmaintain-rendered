Accept multiple .env files in UV_ENV_FILE

#11665 opened by @Seven4ME on 2025-02-20 13:21
Pull request opened by @Seven4ME on 2025-02-20 13:21

According to the UV documentation, the UV_ENV_FILE environment variable should support multiple .env files, separated by spaces. However, when I tried using this feature in my repository, it didn’t work as expected.

To investigate, I checked the UV repository for relevant tests and found run_with_multiple_env_files. This test asserts the following error: No environment file found at: .env1 .env2.

This discrepancy could indicate either a mismatch between the documentation and the implementation or a bug in the code.

I decided to fix the issue in the code since the ability to pass multiple .env files is a valuable feature. If my fix isn’t appropriate, I’d be happy to make any necessary adjustments.
