# Make `--python` a global argument

---

_Pull request opened by @charliermarsh on 2024-05-28 20:44_

## Summary

This PR makes `--python` and `--system` global arguments, so that they can be passed anywhere.

Closes https://github.com/astral-sh/uv/issues/3704.

I have similar opinion on https://github.com/astral-sh/uv/pull/3864:
> Add a `--package` option that allows switching the current project in the workspace. Wherever you are in a workspace, you should be able to run with any other project as root. This is the uv equivalent of `cargo run -p`.
> 
> I don't love the `--package` name, esp. since `-p` is already taken and in general to many things start with p already.


What I can suggest is to use `--py` alias instad of `-p` short at there.
for _short_, I think we could rename the args to `python_toolchain` and use `-t` for _short_ usage.

It's breaking but we can also continue without _short_ at this point.
