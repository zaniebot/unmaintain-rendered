Prefer compatible to incompatible distributions when packages exist on multiple indexes

#8961 opened by @charliermarsh on 2024-11-09 00:27
@charliermarsh

Summary

At time of writing, markupsafe==3.0.2 exists on the PyTorch index, but there's only a single wheel:

MarkupSafe-3.0.2-cp313-cp313-manylinux_2_17_x86_64.manylinux2014_x86_64.whl

Meanwhile, there are a large number of wheels on PyPI for the same version. If the user is on Python 3.12, and we return the incompatible PyTorch wheel without considering the PyPI wheels, PubGrub will mark 3.0.2 as an incompatible version, even though there are compatible wheels on PyPI.

Closes https://github.com/astral-sh/uv/issues/8922.
