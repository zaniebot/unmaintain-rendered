Prefer updating existing .zshenv over creating a new one in tool update-shell

#16866 opened by @benberryallwood on 2025-11-26 19:41
Pull request opened by @benberryallwood on 2025-11-26 19:41

Summary

Update the logic in uv_shell::Shell.configuration_files for Zsh to match rustup's rc file detection for the case when ZDOTDIR is set, but there is an existing ~/.zshenv file.

Now instead of creating a new file at ZDOTDIR/.zshenv, it will update the existing ~/.zshenv file.

Fixes: #16865

Test Plan

Tested locally following the minimal example in #16865, as well as a couple of other scenarios to ensure that behaviour hasn't changed.
