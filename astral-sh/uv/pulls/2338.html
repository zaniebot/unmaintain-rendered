Use python from python -m uv as default

#2338 opened by @konstin on 2024-03-10 13:37
@konstin

Python tools run with python -m <tool> will use this python as their default python, including pip, virtualenv and the built-in venv. Calling Python tools this way is common, the pip docs use python -m pip exclusively, the built-in venv can only be called this way and certain project layouts require python -m pytest over pytest. This python interpreter takes precedence over the currently active (v)env.

These tools are all written in python and read sys.executable. To emulate this, we're setting UV_DEFAULT_PYTHON in the python module-launcher shim and read it in the default python discovery, prioritizing it over the active venv. User can also set UV_DEFAULT_PYTHON for their own purposes.

The test covers only half of the feature since we don't build the python package before running the tests.
