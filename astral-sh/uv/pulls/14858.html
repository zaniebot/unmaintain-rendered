Respect credentials from all defined indexes

#14858 opened by @charliermarsh on 2025-07-23 21:13
@charliermarsh
Summary

The core problem here is that allowed_indexes only includes at most one "default" index. This is problematic for tool upgrades, since the index in the receipt will be marked as default, but credentials will be omitted; if credentials are then defined in a uv.toml, we'll never look at those, since that will also be marked as default, and we only look at the first default.

Instead, we should consider all defined indexes in priority order.

Closes astral-sh/uv#14806.
