Treat already-installed base environment packages as preferences in uv run --with

#13284 opened by @charliermarsh on 2025-05-04 15:40
@charliermarsh
Summary

If a script has some requirements, and you provide --with, we currently ignore any constraints from those requirements. We might want to treat them as hard constraints in the future. For now, though, we just treat them as preferences -- so we prefer those versions, but don't require them to match and still run the --with resolution in isolation.

Closes astral-sh/uv#13173.
