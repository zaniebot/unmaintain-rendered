Preserve index URL priority order when writing to pyproject.toml

#14831 opened by @charliermarsh on 2025-07-22 18:45
@charliermarsh
Summary

A little nuanced, but... When you add multiple --index URLs on the CLI (e.g., in uv pip install), we check the first-provided index, then the second index, etc. However, when we write those URLs to the pyproject.toml in uv add, we were adding them in reverse-order. We now add them in a way that preserves the priority order.

Closes astral-sh/uv#14817.
