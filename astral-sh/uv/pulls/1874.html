Retain authentication attached to URLs when making requests to the same host

#1874 opened by @zanieb on 2024-02-22 15:37
@zanieb

Closes astral-sh/uv#1860

In astral-sh/uv#1816, we started using the URL attached to a response instead of the request URL for subsequent requests — this fixes various bugs but has the side-effect of dropping credentials from the URL. Here, we transfer credentials from the request URL to the response URL. We perform RFC compliant checks for safety.
