Always use base Python discovery logic for cached environments

#11254 opened by @charliermarsh on 2025-02-05 19:11
@charliermarsh

Summary

This is attempting to solve the same problem surfaced in #11208 and #11209. However, those PRs only worked for our own managed Pythons. In Gentoo, for example, they disable the managed Pythons, which led to failures in the test suite, because the "base Python" returned after creating a virtual environment would differ from the "base Python" that you get after querying an existing virtual environment.

The fix here is to apply our same base Python normalization and discovery logic, to non-standalone / non-managed Pythons. We continue to use sys._base_executable for such Pythons when creating the virtualenv, but when caching, we perform this second discovery step.

Closes https://github.com/astral-sh/uv/issues/11237.
