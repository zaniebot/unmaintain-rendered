Add markers to edges rather than distributions

#4166 opened by @charliermarsh on 2024-06-08 18:08
@charliermarsh

Summary

We've debated this a bit but the thing that tipped me over the edge is https://github.com/astral-sh/uv/issues/4157. As-is, there's no way to represent "a package should be installed, but the extra should only be installed conditionally based on the markers", because the markers sit on the distribution. By placing the markers on the edge, we can now represent scenarios that weren't previously representable.

Closes https://github.com/astral-sh/uv/issues/4137. Closes https://github.com/astral-sh/uv/issues/4125. Closes https://github.com/astral-sh/uv/issues/4157.

Comment by @charliermarsh on 2024-06-08 18:17

I think it's possible that we could continue to store markers on distributions if we changed the schema such that extras were once again stored as their own "distributions" (i.e., their own nodes in the graph), and then changed the routine in graph.rs to treat a None marker as "universal" (right now, None markers are discarded, effectively).
