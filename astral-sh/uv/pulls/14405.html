Ignore Python patch version for `--universal` pip compile

#14405 opened by @charliermarsh on 2025-07-02 01:59
Pull request opened by @charliermarsh on 2025-07-02 01:59

Summary

The idea here is that if a user runs uv pip compile --universal, we should ignore the patch version on the current interpreter. I think this makes sense... --universal tries to resolve for all future versions, so it seems a bit odd that we'd start at the current patch version.

Closes https://github.com/astral-sh/uv/issues/14397.
