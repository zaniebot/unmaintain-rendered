Avoid Removing Quotes From Requirement Markers

#3214 opened by @ibraheemdev on 2024-04-23 15:26
@ibraheemdev
Summary

Avoid removing quotes from markers, e.g. numpy (>=1.19) ; python_version >= "3.7" should not be rewritten. Fixes astral-sh/uv#2551.

This PR also makes fixups a bit more flexible internally for fixes that aren't simple to implement with a pure regex replacement, like this one. astral-sh/uv#1529 fixed a similar problem but the current regex is still not smart enough to avoid all markers completely (like python_version).
