Use resolver-returned wheel over alternate cached wheel

#12301 opened by @charliermarsh on 2025-03-18 22:16
@charliermarsh

Summary

I think this is reasonable to change. Right now, if you're on Python 3.11, the resolver returns multiprocess-0.70.17-py311-none-any.whl, but multiprocess-0.70.17-py310-none-any.whl is in the cache, we'll reuse multiprocess-0.70.17-py310-none-any.whl (since it is compatible with Python 3.11).

Instead, we now require the cached wheel to match the wheel returned by the resolver.

Closes https://github.com/astral-sh/uv/issues/12273.
