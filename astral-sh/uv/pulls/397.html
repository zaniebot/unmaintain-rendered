Remove __pycache__ directories when uninstalling

#397 opened by @charliermarsh on 2023-11-10 18:26
@charliermarsh

According to the packaging documentation, "uninstallers should be smart enough to remove .pyc even if it is not mentioned in RECORD". Previously, we weren't handling this case, so if you installed via Puffin, then imported a file (to trigger bytecode compilation), then uninstalled, we'd leave spare __pycache__ directories around.

Closes https://github.com/astral-sh/puffin/issues/395.
