Avoid respecting preferences from other indexes

#10782 opened by @charliermarsh on 2025-01-20 17:12
@charliermarsh
Summary

The fix I shipped in astral-sh/uv#10690 regressed an important case. If we solve a PyPI branch before a PyTorch branch, we'll end up respecting the preference, and choosing 2.2.2 instead of 2.2.2+cpu.

This PR goes back to ignoring preferences that don't map to the current index. However, to solve astral-sh/uv#10383, we need to special-case requirements.txt, which can't provide explicit indexes. So, if a preference comes from requirements.txt, we still respect it.

Closes astral-sh/uv#10772.
