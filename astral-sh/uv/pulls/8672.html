uv-resolver: simplify case analysis in ForkMap::get

#8672 opened by @BurntSushi on 2024-10-29 17:26
@BurntSushi

Previously, when doing a uv pip resolution, we would only return the first entry in the map. But there should only ever be one entry, or else we would have incompatible dependencies. So we can collapse the case with the "one universal fork" case.

(I found this while doing some refactoring of how we handle forking, and collapsing these cases simplifies some of that refactoring work.)
