Avoid attempting rename in copy fallback path

#1546 opened by @charliermarsh on 2024-02-16 22:02
Pull request opened by @charliermarsh on 2024-02-16 22:02

Summary

This could fix https://github.com/astral-sh/uv/issues/1454, but I'm not sure. I was able to replicate by forcing a bunch of error states. But, in short, if we fail to hardlink on the initial copy due to a file existing, and then fail again, we fallback to copying. But if we copy, then the tempfile doesn't exist, and so the fs_err::rename(&tempfile, &out_path)?; will fail with "File not found".

This PR just ensures that the cases are explicitly mutually exclusive: we only attempt to rename if the hardlink succeeded.
