Make hashes optional

#910 opened by @konstin on 2024-01-14 12:34
@konstin

There is no guarantee that indexes provide hashes at all or the sha256 we support specifically. PEP 503:

The URL SHOULD include a hash in the form of a URL fragment with the following syntax: #=, where is the lowercase name of the hash function (such as sha256) and is the hex encoded digest.

We instead use the url as input to generate a hash when caching.
