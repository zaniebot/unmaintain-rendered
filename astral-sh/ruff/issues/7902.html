UP036 (outdated-version-block) is off-by-one for the target version

Closed #7902 opened by @oprypin on 2023-10-10 18:59
Issue opened by @oprypin on 2023-10-10 18:59

If my target-version is py38, I expect the condition if sys.version_info >= (3, 8): to be dropped but it isn't. It gets dropped for py37, though.

This is a regression introduced in #7284 while fixing #7258.

A minimal code snippet that reproduces the bug.

import sys

if sys.version_info >= (3, 8):
    print('oh no')

expected:

print('oh no')

actual: (no lint and the file doesn't get fixed)

The current Ruff settings (any relevant sections from your pyproject.toml).

target-version = "py38"
select = ["UP036"]
