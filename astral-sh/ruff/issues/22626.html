reimplemented-starmap (FURB140): trigger on maps

#22626 opened by @GideonBear on 2026-01-16 18:00
@GideonBear
Summary

reimplemented-starmap (FURB140) doesn't report an issue for:

map(lambda x: f(*x), l)

unnecessary-map (C417) suggests turning it into:

(f(*x) for x in l)

Which is then correctly picked up by FURB140.

This means that anyone not using C417 loses out on FURB140, while it is just as valid on the first example.
