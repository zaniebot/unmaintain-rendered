GitHub, PyPI.

  • [x] NIC001: Implicitly concatenated str literals on one line
    • Already implemented via ISC001 from flake8-implicit-str-concat.
  • [x] NIC002: Implicitly concatenated str literals over multiple lines
    • Already implemented via ISC002 from flake8-implicit-str-concat when using the configuration option provided by ruff (allow-multiline = false)
  • [x] NIC101: Implicitly concatenated bytes literals on one line
    • Already implemented via ISC001 from flake8-implicit-str-concat.
  • [x] NIC102: Implicitly concatenated bytes literals over multiple lines
    • Already implemented via ISC002 from flake8-implicit-str-concat when using the configuration option provided by ruff (allow-multiline = false)

This is somewhat the inverse of flake8-implicit-str-concat. Despite this, they agree that implicit concatenation on a single line shouldn't happen. One also has a separate rule for bytes vs str where the other uses combined rules.
