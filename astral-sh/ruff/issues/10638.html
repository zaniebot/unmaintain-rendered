Warn about multiple imports of the same module

Open #10638 opened by @Beliavsky on 2024-03-27 20:46
Issue opened by @Beliavsky on 2024-03-27 20:46

I request that ruff warn of multiple imports of the same module, for example

import numpy as np
print(np.random.normal(size=3))
import numpy as np
print(np.random.normal(size=5))

Thanks very much for ruff.

Comment by @MichaReiser on 2024-03-28 13:37

Ruff's F811 already warns you about redefinition if the first definition is unused

import numpy as np
import numpy as np

print(np.random.normal(size=3))

print(np.random.normal(size=5))

I could see this as a possible extension of F401 that caches unused imports, assuming that re-importing can generally be considered a no-op, which I'm not sure is given. In that case it could make sense to have a separate rule (with a lower severity)
