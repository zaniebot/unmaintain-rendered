[red-knot] Support callable types and typing.Callable

#15382 opened by @carljm on 2025-01-09 18:54
@carljm

Our "callable type" should be able to represent a callable of arbitrary signature; typing.Callable can only spell a limited subset of those types.

This also includes implementing subtyping/assignability/equivalence for this type.

  • [x] Add a new CallableType variant (astral-sh/ruff#16493)
  • [x] Use the new variant for typing.Callable (astral-sh/ruff#16493)
  • [x] Use the new variant for lambda expressions (astral-sh/ruff#16547)
    • [x] Infer the return type of lambda expressions (astral-sh/ruff#16695)
  • [x] Implement support for calling a typing.Callable i.e., call method (astral-sh/ruff#16888)
  • [x] Implement various relationship methods for the new variant
    • [x] is_assignable_to (astral-sh/ruff#16845)
    • [x] is_gradual_equivalent_to (astral-sh/ruff#16634)
    • [x] is_subtype_of (astral-sh/ruff#16804)
    • [x] is_equivalent_to (astral-sh/ruff#16698)
    • [x] is_disjoint_form (#17094)
  • [x] Other methods
    • [x] is_fully_static (astral-sh/ruff#16633)
    • [x] is_single_valued (astral-sh/ruff#16493)
    • [x] is_singleton (astral-sh/ruff#16493)
    • [x] member_lookup_with_policy (astral-sh/ruff#16618)
    • [x] static_member (astral-sh/ruff#16618)
    • [x] to_meta_type (astral-sh/ruff#16618)
  • [x] Add property tests for relation methods between callable types (#17006)
