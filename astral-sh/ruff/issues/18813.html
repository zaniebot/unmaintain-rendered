Summary

The fix for unnecessary-dunder-call (PLC2801) can create a syntax error due to lack of padding if it is directly after a keyword. This has not been brought up yet in either #16053 or #16216. playground

three = 1 if 1 else(3.0).__str__()

PS D:\python_projects> uvx ruff check issue.py --isolated --select PLC --unsafe-fixes --fix --preview
