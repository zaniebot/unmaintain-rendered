False positive SLOT000 on Enum subclasses

#9890 opened by @jamesbraza on 2024-02-08 06:12
Issue opened by @jamesbraza on 2024-02-08 06:12

With ruff==0.2.1 and Python 3.12:

from enum import Enum

class MyEnum(Enum):
    pass

class CustomStrEnum(str, MyEnum):  # Throws SLOT000
    pass

I think this is a false positive for SLOT000 based on https://github.com/astral-sh/ruff/issues/5748. At the time, looks like Ruff decided not to check for this.

However, maybe nowadays it's possible, and it is a false positive SLOT000. Do you think we can fix this false positive
