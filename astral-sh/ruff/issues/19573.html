ruff --watch improvements: ignore dirs, debouncee

#19573 opened by @sathishvj on 2025-07-27 10:47
@sathishvj

Summary

Summary Improve the developer experience of ruff --watch by adding the ability to:

  1. Ignore specific directories or files

  2. Debounce file change events to reduce redundant action on rapid save operations or bulk changes

  3. Ignore Directories or Files Problem: Currently, it seems that ruff --watch triggers on all file changes. Again, it seems to be me that --exclude does not apply linting rules, but --watch still triggers.

  • Unnecessary linter runs
  • Jankiness/Flashing
  • Excessive CPU usage

Proposed Solution: Introduce a CLI flag (and/or config option in pyproject.toml) to ignore specific directories or glob patterns. Example:

ruff --watch --ignore-dirs node_modules,.venv,build,dist```

pyproject.toml:

``toml
[tool.ruff.watch]
ignore = ["node_modules", ".venv", "build", "dist"]
  1. Debounce File Events similar to the above, to reduce overwork on fast file changes.
