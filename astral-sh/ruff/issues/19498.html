start_of_file ignores a backslash, making I002 introduce a syntax error

#19498 opened by @dscorbett on 2025-07-22 20:35
@dscorbett
Summary

start_of_file ignores a backslash after a docstring, making the fix for missing-required-import (I002) insert the required import immediately after the docstring, on a new physical line but the same logical line, which is a syntax error. Example:

$ cat >i002.py <<'# EOF'
"docstring"\

print(f"{__doc__=}")
# EOF

$ ruff --isolated check i002.py --select I002 --config 'lint.isort.required-imports=["import sys"]' --diff 2>&1 | grep error:
error: Fix introduced a syntax error. Reverting all changes.

The erroring output in that example is:

"docstring"\
import sys

print(f"{__doc__=}")

The output should be:

"docstring"\

import sys
print(f"{__doc__=}")
