RUF037 does not support t-strings

#19951 opened by @dscorbett on 2025-08-18 02:02
@dscorbett

Summary

unnecessary-empty-iterable-within-deque-call (RUF037) does not support t-strings. This is similar to #18854. Example:

$ cat >ruf037.py <<'# EOF'
from collections import deque
queue = deque(t"")
# EOF

$ ruff --isolated check ruf037.py --select RUF037 --target-version py314 --preview
All checks passed!
