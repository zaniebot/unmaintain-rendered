Should eq-without-hash (PLW1641) not apply to stubs?

#15292 opened by @Avasam on 2025-01-06 03:29
@Avasam

I don't think that eq-without-hash (PLW1641) should apply to stubs (it currently does). Stubs authors should aim to faithfully represent the runtime implementation. And if the runtime doesn't implement __hash__, the stubs shouldn't define it either.

Third-party stub authors have no control on this. First-party stubs would get FURB189 in their source .py files.

Ruff: 0.8.5 (rule is currently in preview)
