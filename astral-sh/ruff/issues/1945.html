SIM115 should exempt exit_stack.enter_context(open(…))

#1945 opened by @andersk on 2023-01-18 03:02
@andersk

contextlib.ExitStack.enter_context uses the context manager, so it should not trip SIM115 Use context handler for opening files:

with contextlib.ExitStack() as exit_stack:
    f = exit_stack.enter_context(open("filename"))

This is necessary when you need to conditionally open a file, or open many files in a loop.

Similarly for contextlib.AsyncExitStack.enter_async_context, although I guess that’s not used for open() specifically.
