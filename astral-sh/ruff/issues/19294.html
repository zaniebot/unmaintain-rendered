[`pylint`] False negative on indirect `pathlib.Path`s (`PLW1514`)

Closed #19294 opened by @MeGaGiGaGon on 2025-07-11 21:45
Issue opened by @MeGaGiGaGon on 2025-07-11 21:45

Summary

unspecified-encoding (PLW1514) has false negatives if a pathlib.Path is opened through a variable: https://play.ruff.rs/9a30291c-642c-413f-be9a-1b82db550daf

from pathlib import Path

file = Path().open()  # True positive

path = Path()
file = path.open()  # False negative

This mainly applies to functions, since I ran into this case that would have been nice to be warned about:

def format_file(file: Path):
    with file.open() as f:
        contents = f.read()

Version

ruff 0.12.3 (5bc81f26c 2025-07-11) + playground
