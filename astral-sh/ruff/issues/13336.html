E731 Doesn't Ignore Dataclass Attributes

Closed #13336 opened by @max-muoto on 2024-09-12 05:24
Issue opened by @max-muoto on 2024-09-12 05:24

While E731 correctly flags class attributes that are assigned to lambdas:

class container:
    foo = lambda x: x

In the case of dataclases it incorrectly flags the assignment:

@dataclasses.dataclass
class Container:
    foo = lambda x: x

From my POV this should be functionally equivalent to setting a default in an __init__, which is allowed. Anything with dataclass_transform applied (Pydantic models, msgspec structs, etc) should likely additionally be ignored, however this would require more advanced introspection.
