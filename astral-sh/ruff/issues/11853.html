New Rule: Consider decorating method with @staticmethod PYL-R0201

Closed #11853 opened by @Zerotask on 2024-06-13 08:35
Issue opened by @Zerotask on 2024-06-13 08:35

Pycharm and Deepsource warns you, when you use an instance method which does not use self and therefore can be marked with @staticmethod.

The method doesn't use its bound instance. Decorate this method with @staticmethod decorator, so that Python does not have to instantiate a bound method for every instance of this class thereby saving memory and computation. Read more about staticmethods here.

It would be great to have that check in ruff as well.

Example:

class Calculator:
    def add(self, a, b):
        return a + b

could be:

class Calculator:
    @staticmethod
    def add(a, b):
        return a + b
