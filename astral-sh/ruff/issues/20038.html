Similar flag as `pylint --fail-under <score>`?

Question

Are there any plans on implementing a similar flag for ruff check like pylint --fail-under <score>? E.g. let the ruff check pre-commit hook pass if the score is over 90%. There are cases where we need to make changes to an old code base, and we can't fix all the linter issues, or add a # noqa: {} on all the failing issues. Having a threshold that could still pass the linting would be a great addition. Or is there a way to simulate this behavior with Ruff?

Version

ruff 0.12.9

Comment by @ntBre on 2025-08-22 18:39

Ah, that's interesting! I think the way we're likely to handle this is either with baselines (#1149) or a warning severity (#1256, not sure that would help here exactly).

The closest thing we have right now is probably the GitLab output format, which has a fingerprint for each diagnostic, intended for matching up diagnostics over time for GitLab's Code Quality reports. You might be able to hack something together with that.
