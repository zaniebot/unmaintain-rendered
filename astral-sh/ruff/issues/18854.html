Summary

unnecessary-empty-iterable-within-deque-call (RUF037) currently does not support empty strings. Based on my github searches, this is an uncommon but present pattern, so could be a potential improvement. playground

from collections import deque

queue = deque("")  # No error
queue = deque(b"")  # No error
