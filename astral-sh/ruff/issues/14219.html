`D102` permits missing docstrings in public methods if Python file name starts with `_`

Closed #14219 opened by @yury-fedotov on 2024-11-09 02:31
Issue opened by @yury-fedotov on 2024-11-09 02:31

Checks

List of keywords I searched for before creating this issue:

  • D102

Description

It seems like D102 permits missing docstrings in public methods if Python file name starts with _. This probably should not be the case, since the rule name suggests that it is concerned only with method names.

MRE

# File is called "_foo.py" - underscore is important.

class MyClass:

    def __init__(self) -> None:
        pass
    
    def important_method() -> None:
        pass

Example above passes D102. If you rename the file to foo.py - it would fail.

Run params

  • The command you invoked: ruff check
  • The current Ruff settings: D102 enabled.
  • The current Ruff version: 0.7.1.
