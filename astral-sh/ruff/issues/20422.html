UP008 should not apply when the class variable is shadowed

Closed #20422 opened by @dscorbett on 2025-09-15 17:54
Issue opened by @dscorbett on 2025-09-15 17:54

Summary

When the first argument to super in a method is a local variable, super-call-with-parameters (UP008) should not apply, even if the local variable has the same name as the class. Example:

$ cat >up008.py <<'# EOF'
class A:
    def f(self):
        return 1
class B(A):
    def f(self):
        return 2
class C(B):
    def f(self):
        C = B
        return super(C, self).f()
print(C().f())
# EOF

$ python up008.py
1

$ ruff --isolated check up008.py --select UP008 --unsafe-fixes --fix
Found 1 error (1 fixed, 0 remaining).

$ python up008.py
2
