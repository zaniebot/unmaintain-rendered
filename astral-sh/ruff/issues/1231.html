Comment by @edgarrmondragon on 2022-12-13 23:40

That's the behavior of flake8-annotations. The rationale is that inheritance patterns may depend on the concrete type of self.

class A:
    def public_method(self: "A", properties: Dict[Any, Any]) -> polars.DataFrame:
        """A Docstring."""
        _ = properties


class B:
    new_attr = True

    def public_method(self: "A", properties: Dict[Any, Any]) -> polars.DataFrame:
        """A Docstring."""
        super().public_method(self, properties)
        self.new_attr  # mypy would complain that `A` doesn't have a `new_attr` attribute.
