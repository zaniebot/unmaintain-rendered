Sorted `typing.Literal` Values Rule

Open #18279 opened by @TimothyWillard on 2025-05-23 15:26
Issue opened by @TimothyWillard on 2025-05-23 15:26

Summary

It would be helpful for consistency in function signatures to have sorted literals. For example:

# Okay because `sorted(["a", "b", "c"]) == ['a', 'b', 'c']`
def f(x: Literal["a", "b", "c"]) -> None:
  print(x)

# Warning because `sorted(["c", "b", "a"]) != ["c", "b", "a"]`
def g(x: Literal["c", "b", "a"]) -> int:
  return ["a", "b", "c"].index(x)

This is similar to RUF022, but in a different context. I think there are conceptual complications with literals of mixed types (i.e. Literal[0, 1, "a", "b"]) that I don't have great thoughts on how to handle. Maybe skip for now or only apply the rule to strings? As far as I know this is not implemented by any other formatters/linters.
