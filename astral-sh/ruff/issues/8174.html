Request: external also matching prefixes

#8174 opened by @jamesbraza on 2023-10-24 17:09
Issue opened by @jamesbraza on 2023-10-24 17:09

Coming from https://github.com/astral-sh/ruff/issues/8153 with ruff==0.1.1, there is the external setting for third party tools using noqa comments. It currently works solely on exact matching of the codes inside the list.

For tools like pydoclint, which has DOC error codes: jsh9/pydoclint#3-style-violation-codes, it would be useful if external worked using prefixes as well. That way, one could just have external = ["DOC"].

So the request is to have external also work using prefix matching (not just exact matching).
