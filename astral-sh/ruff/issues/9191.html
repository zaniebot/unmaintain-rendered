`FURB118` is incorrector for `operator.contains`. The operands are reversed!

#9191 opened by @Skylion007 on 2023-12-18 18:37
Issue opened by @Skylion007 on 2023-12-18 18:37

There is a bug in the fixit for the FURB118

f = lambda a, b: a in b

gets replaced with

f = operator.contains

but this is NOT semantically equivalent. Per the documentation, operator.contains is equivalent to b in a not a in b!

f = lambda a, b: b in a

is actually equivalent to operator.contains not what is actually getting flagged (or fixed) by refurb.

This is appearing when trying to apply this fix with ruff master.
