ERA001 false positives on CODE_INDICATORS

Open #18461 opened by @SXHRYU on 2025-06-04 13:47
Issue opened by @SXHRYU on 2025-06-04 13:47

Summary

If the comment or comment line consists of one word derived from "return"/"break"/"continue"/"import" (i.e. CODE_INDICATORS) it is considered valid python code.

Examples:

# returnable  noqa: ERA001
# imports  noqa: ERA001
# breaking  noqa: ERA001

It can also be detrimental if it triggers on normal comment that contains function name on a new line:

# this is a long comment explaining a bug that happens when calling
# on_return  noqa: ERA001
# (^ this can be a valid function name)

Perhaps it's viable to wrap if !CODE_INDICATORS.is_match(line) to some less-greedy function?

Version

0.11.12
