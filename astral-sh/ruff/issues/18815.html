[Pylint] PLE2510 fix changes behavior with preceding slash

#18815 opened by @MeGaGiGaGon on 2025-06-20 06:18
@MeGaGiGaGon
Summary

If the backspace is preceded by a slash, the fix for invalid-character-backspace (PLE2510) can change behavior. playground

PS D:\python_projects> Get-Content issue.py

x = "\␈"
print(repr(x))

^ note: I'm cheating here, PowerShell does not actually display the backspace character as ␈, it actually gets displayed as x = ""

PS D:\python_projects> py issue.py

D:\python_projects\issue.py:1: SyntaxWarning: invalid escape sequence ''
  x = ""
'\\\x08'

PS D:\python_projects> uvx ruff check issue.py --isolated --select PLE

issue.py:1:7: PLE2510 [*] Invalid unescaped character backspace, use "\b" instead
  |
1 | x = "\␈"
  |       ^ PLE2510
2 | print(repr(x))
  |
  = help: Replace with escape sequence

Found 1 error.
[*] 1 fixable with the `--fix` option.

PS D:\python_projects> uvx ruff check issue.py --isolated --select PLE --fix

Found 1 error (1 fixed, 0 remaining).

PS D:\python_projects> Get-Content issue.py

x = "\\b"
print(repr(x))

PS D:\python_projects> py issue.py

'\\b'

This also applies to PLE2512, PLE2513, and PLE2515 playground, as well as PLE2514, but the playground does not support null chars.

Version

ruff 0.12.0 (87f0feb 2025-06-17) + playground
