C409 fix omits comma for `tuple([1],)`

Closed #19568 opened by @dscorbett on 2025-07-26 13:17
Issue opened by @dscorbett on 2025-07-26 13:17

Summary

If a tuple call has a trailing comma and its argument is a single-element list display, the fix for unnecessary-literal-within-tuple-call (C409) doesn’t add a trailing comma, which changes the program’s behavior. Example:

$ cat >c409.py <<'# EOF'
print(tuple([1],))
# EOF

$ python c409.py
(1,)

$ ruff --isolated check c409.py --select C409 --unsafe-fixes --fix
Found 1 error (1 fixed, 0 remaining).

$ cat c409.py
print((1))

$ python c409.py
1
