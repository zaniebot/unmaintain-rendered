YTT201 diagnostic is wrong when the expression uses `!=`

Closed #18292 opened by @dscorbett on 2025-05-24 21:29
Issue opened by @dscorbett on 2025-05-24 21:29

Summary

The diagnostic for sys-version-info0-eq3 (YTT201) is misleading for expressions with != because it is worded as if the expression uses ==. If the expression uses !=, the diagnostic should recommend < instead of >=.

$ cat >ytt201.py <<'# EOF'
import sys
sys.version_info[0] != 3
# EOF

$ ruff --isolated check ytt201.py --select YTT201 --output-format concise -q
ytt201.py:2:1: YTT201 `sys.version_info[0] == 3` referenced (python4), use `>=`
