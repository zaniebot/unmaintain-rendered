RUF010 fix should parenthesize generator expressions

#19433 opened by @dscorbett on 2025-07-19 21:31
@dscorbett

Summary

The fix for explicit-f-string-type-conversion (RUF010) introduces a syntax error when the formatted expression is an unparenthesized generator. Example:

$ cat >ruf010.py <<'# EOF'
f"{ascii(x for x in [])}"
# EOF

$ ruff --isolated check ruf010.py --select RUF010 --diff 2>&1 | grep error:
error: Fix introduced a syntax error. Reverting all changes.

ruff 0.12.4 (ee2759b36 2025-07-17)
