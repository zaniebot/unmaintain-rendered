`banned-api` `TID251` - allow TYPE_CHECKING only imports

#10429 opened by @Kilo59 on 2024-03-16 02:19
Issue opened by @Kilo59 on 2024-03-16 02:19

Requesting a setting to enable if TYPE_CHECKING: imports of banned-api imports. Often times a module may be undesirable to import at runtime, but a TYPE_CHECKING import causes no issues.

from __future__ import annotations

from typing import TYPE_CHECKING

from banned_pkg import Foo # violates TID251

if TYPE_CHECKING:
   from banned_pkg import Bar # this is okay
