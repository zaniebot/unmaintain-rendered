Unexpected behavior with Google `pydocstyle` convention

Issue opened by @jbschiratti on 2025-10-15 12:25

Summary

I apologize in advance if this issue is a misunderstanding of the expected behavior of ruff.

I created a dummy.py Python file with the following content:

"""Dummy Python script."""


def function_with_numpy_style_docstring(value: int, name: str = "John") -> str:
    """Determines how much money a person has.

    Parameters
    ----------
    value : int
        The amount of money a person has.
    name : str
        The name of the person.

    Returns:
    -------
    str:
        A short sentence.
    """
    return f"{name} has {value:d} dollars."

The file was created in an empty folder. It's not part of any project (i.e. no .ruff.toml or pyproject.toml files).

I ran ruff check as follows:

ruff check --select="D" --config 'lint.pydocstyle.convention = "google"' dummy.py

which returned:

All checks passed!

To me, the docstring above does not comply with the Google style.

When running ruff check on this file, I would have expected an error. Am I missing something obvious here?

Version

ruff 0.14.0
