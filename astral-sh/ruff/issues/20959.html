DOC102 false negative for NumPy-style comma-separated parameters

Closed #20959 opened by @dscorbett on 2025-10-18 15:53
Issue opened by @dscorbett on 2025-10-18 15:53

Summary

The NumPy docstring style guide says “When two or more input parameters have exactly the same type, shape and description, they can be combined”, but docstring-extraneous-parameter (DOC102) doesn’t support that, which causes false negatives. Example:

$ cat >doc102.py <<'# EOF'
def leq(x: object, y: object) -> bool:
    """Compare two objects for loose equality.

    Parameters
    ----------
    x1, x2 : object
        Objects.

    Returns
    -------
    bool
        Whether the objects are identical or equal.
    """
    return x is y or x == y
# EOF

$ ruff --isolated check doc102.py --select DOC102 --preview
All checks passed!
