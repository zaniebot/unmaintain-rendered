Comment by @ntBre on 2025-06-04 18:19

@MichaReiser would know better than me, but I don't think there's such an option after going through our isort settings. The closest I found was the combination of force-wrap-aliases and combine-as-imports both set to true. That will wrap code like this:

from typing import Any, Optional as Opt

into

from typing import (
    Any,
    Optional as Opt,
)

but it requires at least one of them to have an as alias.

With the default settings, I think your second code block should be left alone (playground example), but I don't think there's a way to automatically convert the first to the second.
