[SIM110] Ruff enforces all instead of for loop inside async function.

#5098 opened by @real-yfprojects on 2023-06-14 19:29
@real-yfprojects

The following MRE violates the SIM110 rule.

async def coroutine():
    ...


async def test():
    for i in range(10):
        if await coroutine():
            return True
    return False

Ruff suggests rewriting this as

async def coroutine():
    ...


async def test():
    return any(await coroutine() for i in range(10))

However this code doesn't work since any doesn't support AsyncGenerators. The same applies for all which is also covered by this rule. Instead ruff should ignore for loops with await statements.
