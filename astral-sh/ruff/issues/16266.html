Playground counts each non-BMP code point as two code points

#16266 opened by @dscorbett on 2025-02-19 21:04
@dscorbett
Description

The playground counts each non-BMP code point as two code points when applying fixes, leaving characters there that should have been replaced as part of the fix. Here is an example. The input is:

dict(𠤪=1)

Applying the quick fix for unnecessary-collection-call (C408) incorrectly produces:

{"𠤪": 1})

On the command line, though, Ruff 0.9.6 correctly fixes it to:

{"𠤪": 1}
