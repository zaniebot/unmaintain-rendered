After release of 0.1.9 the issue with base class inheritance start appearing (same file)

The issue possibly related to this PR: https://github.com/astral-sh/ruff/pull/9151

DJ008 Model does not define `__str__` method

Example:

import typing_extensions
from django.db import models as django_models

class ParentModel(django_models.Model):
    class Meta:
        abstract = True

    @typing_extensions.override
    def __str__(self) -> str:
        return f'{self.id}'

class ChildModel(ParentModel):  # <-- DJ008 Model does not define `__str__` method
    pass

Current ruff version: 0.1.9 Configuration: --strict with some small exclusions
