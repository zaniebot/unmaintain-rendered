Issue opened by @HexDecimal on 2023-07-28 23:46

An issue similar to #4481, Ruff incorrectly marks a case of SIM118 as an error and as fixable when neither is the case.

from typing import KeysView


class Foo:
    def keys() -> KeysView[object]:
        ...

    def __contains__(self, key: object) -> bool:
        return key in self.keys()  # SIM118 replaces this with `return key in self`

The above is a likely pattern when writing a subclass of collections.abc.Mapping.

Command: ruff example.py --select SIM --fix Version: ruff 0.0.280
