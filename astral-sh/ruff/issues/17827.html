E722: not reported when raising original exception

Open #17827 opened by @matejsp on 2025-05-04 05:44
Issue opened by @matejsp on 2025-05-04 05:44

Summary

When raising the original exception the ruff does not report the use of bare except. However flake8 does.

try:
    pass
except:
    raise

ruff check test.py
All checks passed!

flake8 test.py
test.py:4:1: E722 do not use bare 'except'

https://play.ruff.rs/fb125639-d561-407e-afca-16b2fd1240cd

In our case it was error prone because in our except we were handling errors in except block:

except:
   if input == 1:
     return x
   raise

Version

ruff 0.11.6
