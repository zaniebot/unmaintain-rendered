Rule PYI019 should not flag metaclass methods

#16127 opened by @Geo5 on 2025-02-12 21:54
@Geo5
Description

This code which includes a metaclass and passes mypy without errors:

from typing import TypeVar

F = TypeVar("F")


class FooMeta(type):
    def bar(cls: F) -> F:
        return cls

is fixed to:

ruff check --isolated --select PYI --preview --fix --diff

--- test.py
+++ test.py
@@ -1,8 +1,9 @@
 from typing import TypeVar
+from typing_extensions import Self
 
 F = TypeVar("F")
 
 
 class FooMeta(type):
-    def bar(cls: F) -> F:
+    def bar(cls) -> Self:
         return cls

Would fix 1 error.

Which mypy does not like:

$ mypy --strict test.py 
test.py:8: error: Self type cannot be used in a metaclass  [misc]
Found 1 error in 1 file (checked 1 source file)

$ python --version
Python 3.11.10
$ ruff --version
ruff 0.9.6
$ mypy --version
mypy 1.15.0 (compiled: yes)
