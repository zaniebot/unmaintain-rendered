Please allow to address rules by names instead of codes, too

Closed #15315 opened by @fraunhofersociety on 2025-01-07 10:27
Issue opened by @fraunhofersociety on 2025-01-07 10:27

Hello, could you please support addressing warnings by rule names instead of rule codes? That would make my code more readable for humans.

A. For example I would like to use

# noqa: some-rule-name

instead of

# noqa: XY123

B. Similarly, in the configuration file, I would prefer names over codes, too:

[tool.ruff]
target-version = "py312"
select = ["ALL"]
ignore = [
    "some-rule-name",  
]

C. Please also include rule names in the output of ruff.

Related:

https://github.com/astral-sh/ruff/discussions/3442

https://docs.astral.sh/ruff/rules
