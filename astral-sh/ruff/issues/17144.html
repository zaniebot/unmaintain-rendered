Comment by @AlexWaygood on 2025-04-02 18:06

I had a thought -- I think the issue (or at least one issue) might be that we should probably understand Literal[""] as disjoint from all Callable types, but we currently don't?

We know that for all StringLiteral types, their __class__ is set to exactly str. Not "str or some subclass of str" -- exactly str. And instances of str are not callable; therefore all StringLiteral types are disjoint from all callable types. The same applies to all Type::IntLiteral, Type::BooleanLiteral, Type::BytesLiteral and Type::SliceLiteral types.

If we understand these variants as being disjoint from Type::Callable types, I think this will mean that we'll simplify Callable[[], None] & Literal[""] to Never, which should fix the property test.
