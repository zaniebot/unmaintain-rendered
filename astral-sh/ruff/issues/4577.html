Unused lints don't trigger for tuples.

#4577 opened by @vloddot on 2023-05-22 13:54
@vloddot
The bug

Tuples don't produce unused lints.

For example, this lints as unused:

a = input()

However, this doesn't:

a, b = input().split(' ')
Command 
ruff <filename> --isolated
Ruff settings

Default.

Ruff version

ruff 0.0.269

Comment by @charliermarsh on 2023-05-22 14:07

This is consistent with Flake8 and Pyflakes behavior, and I believe is intentional:

  • https://github.com/PyCQA/pyflakes/issues/143
  • https://github.com/PyCQA/pyflakes/issues/341
Comment by @charliermarsh on 2023-05-22 14:07

Note that Ruff and Pyflakes will flag if the RHS is a tuple:

def f():
    c, d = (2, 3)
Comment by @charliermarsh on 2023-05-22 14:12

We could perhaps try to flag a and b if all tuple members are unused, but it adds enough complexity that I'm going to mark this as "expected" for now.
