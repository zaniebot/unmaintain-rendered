PERF401 fix should parenthesize generator expressions

#19204 opened by @dscorbett on 2025-07-08 12:26

Summary

The fix for manual-list-comprehension (PERF401) changes the programâ€™s behavior when the argument to append is an unparenthesized generator expression. It should parenthesize the argument. Example:

$ cat >perf401.py <<'# EOF'
i = "xyz"
result = []
for i in range(3):
   result.append(x for x in [i])
print(list(map(list, result)))
# EOF

$ python perf401.py
[[0], [1], [2]]

$ ruff --isolated check perf401.py --select PERF401 --preview --unsafe-fixes --fix
Found 1 error (1 fixed, 0 remaining).

$ cat perf401.py
i = "xyz"
result = [x for x in [i] for i in range(3)]
print(list(map(list, result)))
