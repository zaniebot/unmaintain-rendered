`SIM105` should not autofix when not an exception (`B030`)

Closed #5977 opened by @sbrugman on 2023-07-22 13:24
Issue opened by @sbrugman on 2023-07-22 13:24
try:
    ...
except ("not", "an", "exception"):
    pass

Detects the SIM105 and B030 violations. If the SIM105 fix is applied, then B030 is lost:

import contextlib

with contextlib.suppress(Exception):
    print()

I think we should not autofix SIM105 here, similar to refurb.
