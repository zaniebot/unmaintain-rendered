Support the builtins configuration variable from flake8

#1745 opened by @kovidgoyal on 2023-01-09 13:03
@kovidgoyal

It would be useful to me to support the builtins flake8 configuration variable. It allows telling flake8 that some names have been added to the python global namespace and so should not be reported as Undefined name errors.

A common use case for this is python translations. The stdlib module recommends adding the underscore as the gettext() function globally via builtins.dict. ruff currently complains that _ is an undefined name with no way to globally override that.
