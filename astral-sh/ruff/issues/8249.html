Docs: Redirect to rule page from only rule code

#8249 opened by @Mogost on 2023-10-26 11:03
@Mogost
Issue Description Problem

The Ruff Linter documentation provides URLs for specific rules, such as https://docs.astral.sh/ruff/rules/collapsible-if/. While these URLs are useful for referencing rules, they can be cumbersome to include in code comments or documentation. It would be more convenient if we could have simplified and consistent URLs for each rule, making them easier to reference.

Expected Behavior

It would be beneficial to have URL redirects in place, so that each rule can be referenced with a simplified URL structure, such as https://docs.astral.sh/ruff/rules/SIM102/, where 'SIM102' represents the specific rule. This would improve the usability of the documentation and streamline the process of linking to individual rules.

Additional Context

Developers frequently include references to specific rules in their code comments or documentation. Simplifying the URLs for these rules would enhance the overall experience of using the Ruff Linter and referring to its documentation.

In fact, it's much more of a comfort thing. It is very convenient when you know exactly how to form a reference to any rule. This is something similar to the links in Jira. At any time you put the appropriate code in the link part and you get the page you want.

Unfortunately, I do not see the same in PyCharm.

It's because this feature is part of ruff-lsp but the PyCharm Ruff plugin (which is what I assume you're using?) is implemented in a separate way. They're adding support for the Language Server Protocol (LSP) which you can try using:

Support ruff-lsp for only PyCharm Pro/IDEA Ultimate

You can enable it in Preferences/Settings -> Tools -> Ruff -> Use ruff-lsp (Experimental) for PyCharm Pro/IDEA Ultimate

The lsp integration applies only below features:

  • Errors/warnings highlighting (textDocument/publishDiagnostics)
  • Quick-fixes for these errors/warnings (textDocument/codeAction

This is experimental feature.
