feature: accept multiple allowed import aliases for unconventional-import-alias (ICN001)

#10598 opened by @bardiharborow on 2024-03-26 06:41
@bardiharborow

You can currently specify a single import alias to ICN001 for each module:

[tool.ruff.lint.flake8-import-conventions.aliases]
long_module_name = "l"

You should be able to specify multiple allowed import aliases:

[tool.ruff.lint.flake8-import-conventions.aliases]
long_module_name = ["l", "long"]

This would help with namespaced imports, for example where a module is imported as models inside the payments domain but payment_models outside that domain area.

When fixing the linting error it would be acceptable to choose e.g. the first from the list.
