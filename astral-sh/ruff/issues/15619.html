SIM201 and SIM202 fixes should be marked unsafe

#15619 opened by @dscorbett on 2025-01-20 15:33
@dscorbett

The fixes for negate-equal-op (SIM201) and negate-not-equal-op (SIM202) should generally be marked unsafe because __eq__ and __ne__ may return any type, not just bool. The fixes are still safe when the operands are of types whose __eq__ and __ne__ are known to return bool.

$ cat sim20.py
import numpy as np
a = np.array([0])
b = np.array([1])
print(not a == b)
print(not a != b)

$ python sim20.py
True
False

$ ruff --isolated check --select SIM201,SIM202 sim20.py --fix
Found 2 errors (2 fixed, 0 remaining).

$ python sim20.py
[ True]
[False]
