UP007 and UP045: typeshed considers some functions to be classes

#19144 opened by @dscorbett on 2025-07-04 13:48
Issue opened by @dscorbett on 2025-07-04 13:48

Summary

threading.Lock and multiprocessing.dummy.Lock became classes in Python 3.13, so the rules shouldn’t be suppressed for them with a recent enough target version. That is, the current behavior produces true positives for Python ≥ 3.13, but fixing the previous issue simplistically might make this produce false negatives. Example:

$ cat >up007_up045_py313.py <<'# EOF'
from multiprocessing.dummy import Lock as DummyLock
from threading import Lock
from typing import Optional, Union

o0: Optional[Lock]
o1: Optional[DummyLock]
u0: Union[Lock, None]
u1: Union[DummyLock, None]
# EOF

$ python3.13 up007_up045_py313.py; echo $?
0

Found 4 errors (4 fixed, 0 remaining).

$ python3.13 up007_up045_py313.py; echo $?
0
