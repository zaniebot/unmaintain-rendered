PLR6201 autofix is unsafe without type information

Closed #8096 opened by @Avasam on 2023-10-20 16:13
Issue opened by @Avasam on 2023-10-20 16:13

Non-hashable entries should not be put in a set. See the following minimal reproduction:

dict1 = {"foo": 1}
dict2 = {"foo": 2}
dicttest = {"foo": 3}
print(dicttest in (dict1, dict2))

It gets autofix to:

dict1 = {"foo": 1}
dict2 = {"foo": 2}
dicttest = {"foo": 3}
print(dicttest in {dict1, dict2})

which results in:

Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unhashable type: 'dict'

and pyright only reports as error: Set entry must be hashable since https://github.com/microsoft/pyright/releases/tag/1.1.323
