Comment by @charliermarsh on 2023-05-02 01:02

This is probably a result of the conventions defined in PEP 257, which recommends a blank line after the class docstring to mirror the blank line between methods in a class, but doesn't make any explicit recommendation for function docstrings. It looks like Black lets you do either.

Comment by @charliermarsh on 2023-05-02 01:10

I think I'm gonna close in the interest of minimizing configuration complexity. There isn't really a good home for this right now, since ideally it'd be part of the pydocstyle rule set, but then we'd be adding rules that don't exist upstream, and users with D enabled would have to explicitly ignore it. That can be fine, but there are tradeoffs involved, and here, I'm not sure that the upside is worth it.
