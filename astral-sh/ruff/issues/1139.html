I001: `ruff` and `black` disagree on the number of blank lines after imports in `.pyi`

Black's/PEP-484's style guidelines for pyi files differ slightly but ruff doesn't seem to follow those at least in the following case:

ruff --fix:

# a_stub.pyi
import decimal


class SomeType:
   var: decimal.Decimal

black (this results in a I001 from ruff):

# a_stub.pyi
import decimal

class SomeType:
   var: decimal.Decimal
