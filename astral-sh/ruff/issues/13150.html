Comment by @dhruvmanila on 2024-08-29 11:15

Hey, thanks for using Ruff. I'm assuming that you're using both the Ruff extension and the Pylance extension. Currently, Ruff only provides hover implementation for NoQA codes in the comments as seen at https://docs.astral.sh/ruff/editors/features/#hover and it doesn't support providing hover information for any symbols. I'm not exactly sure why you're getting two different content for the same symbol.

side question: is required to be rust expert to contribute to ruff?

I think basic knowledge of Rust would be required if you want to contribute to the Rust code, I wouldn't necessarily say that one needs to be an expert. You can refer to the contributing guidelines for more information.

or this repo is for purely vscode ext only?

This repository (https://github.com/astral-sh/ruff) contains the code for the ruff binary and the documentation that's hosted at https://docs.astral.sh/ruff while the VS Code extension is in https://github.com/astral-sh/ruff-vscode. Additionally, the implementation of the language server that's used to power any editor using the language server protocol is present under the ruff_server crate.

Hope this is helpful, I'll close this issue as resolved but feel free to ask any other questions that you might have.
