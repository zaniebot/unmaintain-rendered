`dir_fd=None` blocks PTH106, PTH107, PTH108, and PTH115

Open #21342 opened by @dscorbett on 2025-11-08 17:51
Issue opened by @dscorbett on 2025-11-08 17:51

Summary

os-rmdir (PTH106), os-remove (PTH107), os-unlink (PTH108), and os-readlink (PTH115) have false negatives when dir_fd is explicitly set to its default value of None. All other PTH rules for functions that support dir_fd handle dir_fd=None.

$ cat >pth1.py <<'# EOF'
import os
os.rmdir("path", dir_fd=None)
os.remove("path", dir_fd=None)
os.unlink("path", dir_fd=None)
os.readlink("path", dir_fd=None)
# EOF

$ ruff --isolated check pth1.py --select PTH106,PTH107,PTH108,PTH115
All checks passed!
