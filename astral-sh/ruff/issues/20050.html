RUF037 false negative on `deque(f"{""}")`

Closed #20050 opened by @dscorbett on 2025-08-22 19:02
Issue opened by @dscorbett on 2025-08-22 19:02

Summary

unnecessary-empty-iterable-within-deque-call (RUF037) checks whether an f-string is empty in the sense of being the empty literal f"", whereas it should should check whether it evaluates to an empty iterable. That is, it should be implemented in terms of is_empty_f_string instead of is_empty_literal. Example:

$ cat >ruf037.py <<'# EOF'
from collections import deque
deque(f"{""}")
# EOF

$ ruff --isolated check ruf037.py --select RUF037 --preview
All checks passed!
