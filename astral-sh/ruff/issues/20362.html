`RUF022` does not follow `lint.isort.classes` option

Open #20362 opened by @velsinki on 2025-09-12 10:05
Issue opened by @velsinki on 2025-09-12 10:05

Summary

The following example

"""Example __init__.py problem."""
from .defs import AnotherClass, CLASS

__all__ = [
    "AnotherClass",
    "CLASS",
]

with this option in ruff.toml

[lint.isort]
classes = ["CLASS"]

should result in no errors because of the overridden PEP8 type inference with classes, but RUF022 still triggers.

This is a small issue which I only noticed because the order of __all__ can be used in sphinx-autodoc. Still, this seems unexpected behavior in my opinion.
