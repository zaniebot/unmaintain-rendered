Fixes for PERF401 and PERF403 should parenthesize lambdas and ternary expressions

#18405 opened by @dscorbett on 2025-05-31 15:17
Issue opened by @dscorbett on 2025-05-31 15:17

Summary

The fixes for manual-list-comprehension (PERF401) and manual-dict-comprehension (PERF403) should insert parentheses around the condition when it is a lambda expression or a ternary conditional expression. See #15050, which fixed the same issue for assignment expressions.

$ cat >perf401.py <<'# EOF'
src = [1]
dst = []
for i in src:
    if True if True else False:
        dst.append(i)
for i in src:
    if lambda: 0:
        dst.append(i)
# EOF

$ ruff --isolated check perf401.py --select PERF401 --preview --unsafe-fixes --diff 2>&1 | grep error:
error: Fix introduced a syntax error. Reverting all changes.

$ cat >perf403.py <<'# EOF'
src = (("x", 1),)
dst = {}
for k, v in src:
    if True if True else False:
        dst[k] = v
for k, v in src:
    if lambda: 0:
        dst[k] = v
# EOF

$ ruff --isolated check perf403.py --select PERF403 --preview --unsafe-fixes --diff 2>&1 | grep error:
error: Fix introduced a syntax error. Reverting all changes.
