Comment by @amyreese on 2025-08-28 18:37

I second the recommendation for using LibCST. It has a parser written in Rust, so it's currently the fastest full CST usable from Python, and it includes its own "codemod" mechanisms for scripting transformations across entire repositories. The Python API is quite nice, with good documentation and an online playground if you want to explore the CST for a bit of code.
