B009 and B010 should ignore non-NFKC attribute names

Closed #21126 opened by @dscorbett on 2025-10-29 17:22
Issue opened by @dscorbett on 2025-10-29 17:22

Summary

get-attr-with-constant (B009) and set-attr-with-constant (B010) should ignore non-NFKC attribute names because the usual attribute syntax would change the program’s behavior. Example:

$ cat >b009_b010.py <<'# EOF'
from types import SimpleNamespace
ns = SimpleNamespace(s=0)
setattr(ns, "ſ", 1)
print(ns.s, getattr(ns, "ſ"))
# EOF

$ python b009_b010.py
0 1

$ ruff check --isolated b009_b010.py --select B009,B010 --fix
Found 2 errors (2 fixed, 0 remaining).

$ cat b009_b010.py
from types import SimpleNamespace
ns = SimpleNamespace(s=0)
ns.ſ = 1
print(ns.s, ns.ſ)

$ python b009_b010.py
1 1
