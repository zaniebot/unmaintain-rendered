UP015 fix should be unsafe when it removes "U"

#18319 opened by @dscorbett on 2025-05-26 14:15
Issue opened by @dscorbett on 2025-05-26 14:15

Summary

The fix for redundant-open-modes (UP015) should be marked unsafe when it removes "U" from the mode because doing so changes behavior in Python 3.11 and later.

$ cat >up015.py <<'# EOF'
open("up015.py", "U")
# EOF

$ python3.11 up015.py 2>&1 | tail -n 1
ValueError: invalid mode: 'U'

$ ruff --isolated check up015.py --select UP015 --fix
Found 1 error (1 fixed, 0 remaining).

$ python3.11 up015.py; echo $?
0

Version

ruff 0.11.11 (0397682f1 2025-05-22)
