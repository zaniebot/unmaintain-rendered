FURB157 misses `decimal.Decimal("_-1")`

Closed #21186 opened by @dscorbett on 2025-11-01 17:03
Issue opened by @dscorbett on 2025-11-01 17:03

Summary

verbose-decimal-constructor (FURB157) has a false negative when the sign character follows an underscore. Example:

$ cat >furb157.py <<'# EOF'
from decimal import Decimal
Decimal("_-1")
# EOF

$ ruff --isolated check furb157.py --select FURB157
All checks passed!

This was a true positive in version 0.14.2.

Version

ruff 0.14.3 (8737a2d5f 2025-10-30)
