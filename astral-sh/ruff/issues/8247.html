False positive for SIM300 (yoda condition)?

Closed #8247 opened by @estyrke on 2023-10-26 09:01
Issue opened by @estyrke on 2023-10-26 09:01

sim300.py:

class Foo:
    NAME = "Foo"

class Bar:
    NAME = "Bar"

def check(name: str) -> None:
    for item in [Foo(), Bar()]:
        if item.NAME == name:
            print(item)

$ ruff --version
ruff 0.0.291
$ ruff check --isolated --select SIM300 sim300.py

Found 1 error.

Applying the suggested fix results in more Yoda-ish order in my opinion.
