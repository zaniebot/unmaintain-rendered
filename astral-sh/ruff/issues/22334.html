SIM105 fix imports builtins to access BaseException

#22334 opened by @dscorbett on 2026-01-01 22:48
Issue opened by @dscorbett on 2026-01-01 22:48

Summary

When the fix for suppressible-exception (SIM105) inserts a reference to BaseException, it always imports builtins, which is only necessary if BaseException has been shadowed, which is rare. Example:

$ cat >sim105.py <<'# EOF'
import sys
try:
    sys.exit(2)
except:
    pass
# EOF

$ ruff --isolated check sim105.py --select SIM105 --unsafe-fixes --fix
Found 1 error (1 fixed, 0 remaining).

$ cat sim105.py
import sys
import contextlib
import builtins
with contextlib.suppress(builtins.BaseException):
    sys.exit(2)
