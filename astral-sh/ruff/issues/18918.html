rule request: ghost param in docstring

#18918 opened by @DaniBodor on 2025-06-24 15:09
@DaniBodor
Summary

rule D417 detects undocumented parameters in docstrings. It would be very useful to have the inverse rule as well, which checks if there are parameters listed in the docstring that do not exist (anymore).

It can happen that while refactoring a function an argument gets removed, and it would be useful if ruff automatically detected that the docstring needs to be adjusted accordingly, as these can easily be left behind.
