SIM113 only applies when the index variable is initialized to 0

#21393 opened by @dscorbett on 2025-11-12 01:03
@dscorbett
Summary

enumerate-for-loop (SIM113) applies only when the index variable is initialized to 0, but enumerate takes an optional argument specifying its starting point, so SIM113 should only require that the variable start as a strict instance of int. For reference, unnecessary_cast_to_int::call_applicability and ResolvedPythonType::from include various ways to determine whether an expression is an int. Example:

$ cat >sim113.py <<'# EOF'
fruits = ["apple", "banana", "cherry"]
i = 1
for fruit in fruits:
    print(f"{i}. {fruit}")
    i += 1
# EOF

$ python sim113.py
1. apple
2. banana
3. cherry

$ ruff --isolated check sim113.py --select SIM113
All checks passed!

In the spirit of SIM113, that code should be:

fruits = ["apple", "banana", "cherry"]
for i, fruit in enumerate(fruits, 1):
    print(f"{i}. {fruit}")
