SIM105 fix changes behavior for except (): and except:

#18209 opened by @dscorbett on 2025-05-19 23:59
@dscorbett
Summary

The fix for suppressible-exception (SIM105) changes behavior when the except tuple is empty. Instead of contextlib.suppress(Exception), the replacement should be contextlib.suppress(). Alternatively, the rule (or just the fix) could be suppressed, letting except-with-empty-tuple (B029) handle it.

$ cat >sim105_1.py <<'# EOF'
try:
    1 / 0
except ():
    pass
# EOF

$ python sim105_1.py; echo $?
Traceback (most recent call last):
  File "sim105_1.py", line 2, in <module>
    1 / 0
    ~~^~~
ZeroDivisionError: division by zero
1

$ ruff --isolated check sim105_1.py --select SIM105 --unsafe-fixes --fix
Found 1 error (1 fixed, 0 remaining).

$ cat sim105_1.py
import contextlib
with contextlib.suppress(Exception):
    1 / 0

$ python sim105_1.py; echo $?
0

The fix also changes behavior when the except clause has no expression. Instead of contextlib.suppress(Exception), the replacement should be contextlib.suppress(BaseException). Alternatively, the rule (or just the fix) could be suppressed, letting bare-except (E722) handle it.

$ cat >sim105_2.py <<'# EOF'
import sys
try:
    sys.exit(2)
except:
    pass
# EOF

$ python sim105_2.py; echo $?
0

$ ruff --isolated check sim105_2.py --select SIM105 --unsafe-fixes --fix
Found 1 error (1 fixed, 0 remaining).

$ cat sim105_2.py                                                       
import sys
import contextlib
with contextlib.suppress(Exception):
    sys.exit(2)

$ python sim105_2.py; echo $?
2
