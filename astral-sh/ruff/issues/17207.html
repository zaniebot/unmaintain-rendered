False positive in pattern attribute assignment: [syntax-errors] Multiple assignments in case pattern

Closed #17207 opened by @sasanjac on 2025-04-04 13:55
Issue opened by @sasanjac on 2025-04-04 13:55

Summary

The current release 0.11.3 in preview mode throws a false positive syntax error in the following case:

ruff check file

match foo:
    case Bar(baz=baz): # [syntax-errors] Multiple assignments in case pattern
        foobar(baz)

As far as I know, this is valid syntax (see PEP 622) and necessary to match against class attributes of classes that don't explicitly specify match attributes.

Version

ruff 0.11.3
