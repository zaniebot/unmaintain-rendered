warn on invocations of `globals()` / `locals()`

#12845 opened by @kiaradlf on 2024-08-12 12:40
Issue opened by @kiaradlf on 2024-08-12 12:40

python has a number of introspection functions, among which globals() and locals().

these can currently be used both for reading of variables as well as for assignment purposes, say:

for foo in foos:
  globals()[foo] = ...

while it seems that ruff currently lets these slide, i believe that for both legibility as well as to facilitate static analysis, use of such invocations is considered an anti-pattern. it would be nice to see ruff suggest users to reconsider using the likes of these.
