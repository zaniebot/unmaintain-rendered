F821: false positive if variable is defined in a loop

#12451 opened by @JaRoSchm on 2024-07-22 11:29
Issue opened by @JaRoSchm on 2024-07-22 11:29

Hi,

this examples raises F821 in ruff 0.5.4 although it clearly is working:

for i in range(3):
    if i == 0:
        x = 0
    else:
        x = y

    y = x + 2

This happens if the variable is defined inside of the loop body. Searching through the many issues where F821 is mentioned I couldn't find this issue yet. Output of ruff check --select F821 --preview:

test.py:5:13: F821 Undefined name `y`
  |
3 |         x = 0
4 |     else:
5 |         x = y
  |             ^ F821
6 | 
7 |     y = x + 2
  |

Found 1 error.
