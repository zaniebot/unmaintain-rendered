Rule for missing members typing annotations

Open #13914 opened by @yairp03 on 2024-10-24 16:00
Issue opened by @yairp03 on 2024-10-24 16:00

Consider the following class:

class Person:
    name: str

    def __init__(self, name: str, age: int) -> None:
        self.name = name
        self.age = age

The rule would suggests adding a typing annotation for Person.age so the code would look like this:

class Person:
    name: str
    age: int

    def __init__(self, name: str, age: int) -> None:
        self.name = name
        self.age = age
