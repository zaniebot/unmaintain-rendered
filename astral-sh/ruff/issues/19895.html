RUF010 fix ignores t-strings

Closed #19895 opened by @dscorbett on 2025-08-13 13:33
Issue opened by @dscorbett on 2025-08-13 13:33

Summary

The fix for explicit-f-string-type-conversion (RUF010) ignores expressions containing t-strings. Example:

$ cat >ruf010.py <<'# EOF'
f"{ascii(t"")}"
# EOF

$ ruff --isolated check ruf010.py --select RUF010 --target-version py314 --preview --fix --output-format concise
ruf010.py:1:4: RUF010 Use explicit conversion flag
Found 1 error.

$ cat ruf010.py
f"{ascii(t"")}"

Ideal fixed output:

f"{t""!a}"

Version

ruff 0.12.8 (f51a228f0 2025-08-07)
