RUF100: have option to require codes, but for codes to be unknown to ruff

#1882 opened by @eriknw on 2023-01-15 03:03
@eriknw

This would allow ruff to "play nicely" with other linters that projects are still using as they're transitioning to ruff.

For example:

    string = "quoteme"
    quoted = f'"{string}"'  # noqa: B028

when using https://github.com/PyCQA/flake8-bugbear

I've had this come up with other error codes that ruff does not yet know about.

I still want the RUF100 check (or similar) to require that noqa pragmas have a code, but not require that ruff know about the codes.
