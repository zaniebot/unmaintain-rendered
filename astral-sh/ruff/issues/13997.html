I'm currently writing a CI pipeline in gitlab. I tried to make ruff generate reports in different format for sonarqube, gitlab, and grouped (for stdout). However, it seems like to do that I must call the ruff check command for every file format I want.

I think it would be beneficial to add the possibility to specify an array for output-format in the same way pytest-cov pluging works. For instance, in my case, I would do:
