Spurious logging-too-many-args (PLE1205) if using the msg kwarg

Closed #6454 opened by @adamtheturtle on 2023-08-09 18:10
Issue opened by @adamtheturtle on 2023-08-09 18:10

logging.info, logging.warning, logging.error, etc. each take (msg, *args, **kwargs). If msg is given as a keyword argument, ruff 0.0.283 gives an error (https://beta.ruff.rs/docs/rules/logging-too-many-args/), which is incorrect.

# example.py

import logging

logging.info(msg="X")

ruff --select=PLE --isolated example.py

example.py:3:1: PLE1205 Too many arguments for `logging` format string
