Include newly added rules in changelog

Closed #5266 opened by @namurphy on 2023-06-21 19:19
Issue opened by @namurphy on 2023-06-21 19:19

Proposed change

Add a section in the changelog of each release on which rules have been added since the previous release.

If there already exists such a list that I missed, it would be helpful to increase its visibility.

Motivation

Right now, the changelog contained within BREAKING_CHANGES.md currently only includes backward incompatible changes. As a user of ruff who is actively following its development, I also want to know which rules have been added in each release so that I know which rules to consider adding to the ruff configuration.

Alternatives

At present, I usually end up doing a GitHub search of merged pull requests with the label of rule, though this misses PRs that are not labeled. (The labels really do help!)

It's also possible to look at the releases page on GitHub. However, this shows pull request titles, which often do not include the rule codes being added.

As always, thank you so much for a fantastic tool!
