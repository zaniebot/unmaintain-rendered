PTH119 and PTH120 should be marked unsafe

#20112 opened by @dscorbett on 2025-08-27 13:06
@dscorbett
Summary

The fixes for os-path-basename (PTH119) and os-path-dirname (PTH120) should be marked unsafe because the replacements are not always equivalent. In particular, pathlib normalizes paths by collapsing adjacent slashes and removing trailing slashes, but os.path.basename and os.path.dirname donâ€™t. Example:

$ cat >pth119_pth120.py <<'# EOF'
import os
path = "a//b/c/"
print(os.path.basename(path))
print(os.path.dirname(path))
# EOF

$ python pth119_pth120.py

a//b/c

$ ruff --isolated check pth119_pth120.py --select PTH119,PTH120 --preview --fix
Found 2 errors (2 fixed, 0 remaining).

$ cat pth119_pth120.py
import os
import pathlib
path = "a//b/c/"
print(pathlib.Path(path).name)
print(pathlib.Path(path).parent)

$ python pth119_pth120.py
c
a/b
