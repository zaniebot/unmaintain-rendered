Request: fixable = ["ALL"]

#4104 opened by @jamesbraza on 2023-04-25 22:02
@jamesbraza

Currently per https://beta.ruff.rs/docs/configuration/#using-pyprojecttoml with ruff==0.0.263, to fix all, one needs to pass all codes:

# Allow autofix for all enabled rules (when `--fix`) is provided.
fixable = ["A", "B", "C", "D", "E", "F", "G", "I", "N", "Q", "S", "T", "W", "ANN", "ARG", "BLE", "COM", "DJ", "DTZ", "EM", "ERA", "EXE", "FBT", "ICN", "INP", "ISC", "NPY", "PD", "PGH", "PIE", "PL", "PT", "PTH", "PYI", "RET", "RSE", "RUF", "SIM", "SLF", "TCH", "TID", "TRY", "UP", "YTT"]

In a config, if one has select = ["ALL"], it would be nice to have an accompanying fixable = ["ALL"]:

[tool.ruff]
fix = true
fixable = ["ALL"]
select = ["ALL"]

This is simpler, future compatible, and more readable.

Comment by @charliermarsh on 2023-04-25 23:57

What you have in the example should actually work, we respect the same set of selectors between fixable and select. (Note, though, that fixable = ["ALL"] doesn't meant that we're able to fix all rules; rather, it means that Ruff is allowed to fix all fixable rules.)

If you see otherwise, happy to re-open :)
