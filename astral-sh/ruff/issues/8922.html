Comment by @charliermarsh on 2023-11-30 14:27

Thanks for the clear issue. I think this is roughly the same as https://github.com/astral-sh/ruff/issues/8383 and there's some more commentary in that issue + a few ideas for improvement. The gist of it is that if a line doesn't contain any splittable whitespace, we avoid flagging E501, which is more often helpful than hurtful, since a line without splittable whitespace tends to be, e.g., a single long URL or other long token. In this case, though, we should probably be flagging E501, since we can split on the bracket at least.
