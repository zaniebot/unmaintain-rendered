Consistent accessibility

#22728 opened by @AndreuCodina on 2026-01-19 17:23
@AndreuCodina

Summary

A rule should prevent public functions that return private classes.

For example, the following code has a public class with a public method returning a private class.

class _Array:
    pass


class Numpy:
    def create_array(self) -> _Array:
        return _Array()

That forces you to import a private class.

In C#:

Private modules and folders should also be covered. For example, in my libraries, I have lots of files/folders prefixed with an underscore (_async_concurrent_dictionary.py, _service_lookup/_service_provider_engine.py...), containing classes without an underscore. This can be considered the equivalent of internal of C#. This is important because I import those classes from /src and /tests, but I don't want user libraries to see or import the code. This issue is related to this topic: https://github.com/astral-sh/ruff/issues/16892

Aliases should be covered too (https://github.com/astral-sh/ruff/issues/20916).
