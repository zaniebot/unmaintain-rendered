`*` unpacking should make PLC0207 fix unsafe

Closed #19660 opened by @dscorbett on 2025-07-31 11:44
Issue opened by @dscorbett on 2025-07-31 11:44

Summary

The fix for missing-maxsplit-arg (PLC0207) should be marked unsafe when the call contains a * unpacking, as it already is for a ** unpacking, because it can induce an error. Example:

$ cat >plc0207.py <<'# EOF'
"www.example.com".split(".", *[-1])[0]
# EOF

$ ruff --isolated check plc0207.py --select PLC0207 --preview --fix
Found 1 error (1 fixed, 0 remaining).

$ cat plc0207.py
"www.example.com".split(".", *[-1], maxsplit=1)[0]

$ python plc0207.py 2>&1 | tail -n 1
TypeError: split() takes at most 2 arguments (3 given)
