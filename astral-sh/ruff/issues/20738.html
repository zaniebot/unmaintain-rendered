new rule - detect accidental trailing newlines at the start and end of docstrings

#20738 opened by @DetachHead on 2025-10-07 06:47
@DetachHead

Summary

def foo():
    """

    foo bar baz

    bar

    """

in this example, there is an unnecessary additional newline at the start and end of the docstring. it would be nice if ruff had a rule to detect this.

note that this is partially already covered by multi-line-summary-first-line (D212), but it only detects the newline at the start of the docstring but not the end. i also don't like to have that rule enabled because i don't like how it formats multiline docstrings (why is there a newline at the end of the docstring but not at the start?):

def foo():
    """foo bar baz

    bar
    """
