outdated-version-block (UP036): Flag assignments (or any outdated sys.version_info)

#16487 opened by @Avasam on 2025-03-04 00:23
Issue opened by @Avasam on 2025-03-04 00:23

Summary

I found the following code snippet in pypa/distutils:

def _pypy_hack(name):
    PY37 = sys.version_info < (3, 8)
    old_pypy = hasattr(sys, 'pypy_version_info') and PY37
    prefix = not name.endswith(('_user', '_home'))
    pypy_name = 'pypy' + '_nt' * (os.name == 'nt')
    return pypy_name if old_pypy and prefix else name

And I'd like outdated-version-block (UP036) (or a similar rule) to flag usage of sys.version_info < (3, 8). When targetting python 3.9+, in any context this is the same as just writing False. Whilst Ruff can't necessarily autofix it correctly, it should be flagged for a human to update / remove the legacy code.
