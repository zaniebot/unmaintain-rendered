Bug: not detecting SIM105 when loop is present

#10495 opened by @jamesbraza on 2024-03-20 18:51
Issue opened by @jamesbraza on 2024-03-20 18:51

With ruff==0.3.3, I think SIM105 should be triggered here, but it's not:

try:
    for _ in range(1):
        pass
except (NotImplementedError, ValueError):
    pass

In other words, I think the above code snippet should be autofixed to:

import contextlib


with contextlib.suppress(NotImplementedError, ValueError):
    for _ in range(1):
        pass
