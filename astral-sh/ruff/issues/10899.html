This is about the too-many-arguments (PLR0913) rule and its documentation. I would suggest to improve the documentation about that rule because there is a difference in its treatment between ruff and pylint.

Maybe something like this?

Differing from PyLint, ruff indeed considers every argument. In contrast, PyLint disregards certain names by default, as dictated by the --ignored-argument-names option.

Background: As discussed with pylint-devs at #9547 the behavior of ruff and pylint are different. It seems to me that ruff is accurate and count every argument without any exception. But PyLint does have something like --ignored-argument-names with arguable default values (e.g. ignored_*).

Here is an example triggering the error in ruff but not pylint.

def weight_by_icd_group(data: pandas.DataFrame,
                        index_column: str,
                        icd_column: str,
                        weight_name: str = 'weight',
                        ignored_icds: tuple = ('UUU'),
                        exclude_appendix_artifact: bool = True
                        ) -> pandas.DataFrame:

Generally minimal differences between ruff and other linters in rule treatment should mentioned in the rules documentation.
