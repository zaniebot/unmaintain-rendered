I think a few more examples would be helpful. I'm also kind of wondering if you'll get the behavior you're looking for with:

[tool.ruff.isort]
split-on-trailing-comma = false

Because we do try to fit the import on a single line if it fits, but by default, we don't remove "magic" trailing commas, which would prevent us from squeezing the import in the first case.
