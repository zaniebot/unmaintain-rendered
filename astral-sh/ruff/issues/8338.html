The formatter's ParseError indicates the offset within the file but not the line number

#8338 opened by @rgeronimi on 2023-10-30 07:06
@rgeronimi

When ruff's formatter encounters an invalid Python file, it prints out the offset position of the offending Python statement rather than the line number. E.g.,: error: Failed to format commands/liberrors.py: source contains syntax errors: ParseError { error: UnrecognizedToken(Name { name: "THE_INVALID_FILE" }, None), offset: 208, source_path: "<filename>" } 157 files left unchanged It should return the line number, as Black does.
