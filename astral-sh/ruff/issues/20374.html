New rule: require class attributes to go after docstring and before all method definitions

#20374 opened by @DeflateAwning on 2025-09-12 22:08
@DeflateAwning

Summary

Example of valid-and-intended format:

class SimpleTestJob(EtlJob):
    """Test the `entry_target_runner.py` script and EtlJob class."""

    config_should_be_scheduled = False

    def extract(self):
        logger.info("This is the extract stage.")

    def transform(self):
        pass

Example of bad ordering:

class SimpleTestJob(EtlJob):
    """Test the `entry_target_runner.py` script and EtlJob class."""

    def extract(self):
        logger.info("This is the extract stage.")

    config_should_be_scheduled = False

    def transform(self):
        pass
