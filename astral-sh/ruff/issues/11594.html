Request: prefer %r/!r marker instead of calling repr in string formatting

#11594 opened by @Avasam on 2024-05-29 05:27
@Avasam

See this PR for the kind of changes I'd like to see automated: mhammond/pywin32#2272

From:

"%s %s" % (repr(foo), bar)
"{} {}".format(repr(foo), bar)
f"{repr(foo)} {bar}"

To:

"%r %s" % (foo, bar)
"{!r} {}".format(foo, bar)
f"{foo!r} {bar}"

I don't think this is supported by any linter atm. This should synergize well with UP030-UP032
