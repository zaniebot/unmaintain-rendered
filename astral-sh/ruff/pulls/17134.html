[syntax-errors] Starred expressions in return, yield, and for

#17134 opened by @ntBre on 2025-04-01 20:21
Pull request opened by @ntBre on 2025-04-01 20:21

Summary

Fixes https://github.com/astral-sh/ruff/issues/16520 by flagging single, starred expressions in return, yield, and for statements.

I thought yield from would also be included here, but that error is emitted by the CPython parser:

>>> ast.parse("def f(): yield from *x")
Traceback (most recent call last):
  File "<python-input-214>", line 1, in <module>
    ast.parse("def f(): yield from *x")
    ~~~~~~~~~^^^^^^^^^^^^^^^^^^^^^^^^^^
  File "/usr/lib/python3.13/ast.py", line 54, in parse
    return compile(source, filename, mode, flags,
                   _feature_version=feature_version, optimize=optimize)
  File "<unknown>", line 1
    def f(): yield from *x
                        ^
SyntaxError: invalid syntax

And we also already catch it in our parser.
