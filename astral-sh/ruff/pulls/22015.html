[ty] don't expand type aliases in implicit tuple aliases

#22015 opened by @mtshiba on 2025-12-17 05:05
@mtshiba
Summary

This PR fixes astral-sh/ty#1848.

T = tuple[int, 'U']

class C(set['U']):
    pass

type U = T | C

The reason why the fixed point iteration did not converge was because the types stored in the implicit tuple type alias Specialization changed each time.

1st: <class 'tuple[int, C]'>
2nd: <class 'tuple[int, tuple[int, C] | C]'>
3rd: <class 'tuple[int, tuple[int, tuple[int, C] | C] | C]'>
...

And this was because UnionType::from_elements was used when creating union types for tuple operations, which causes type aliases inside to be expanded. This PR replaces these with UnionType::from_elements_leave_aliases.
