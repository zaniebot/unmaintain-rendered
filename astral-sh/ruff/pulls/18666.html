Summary

Recently stumbled upon type checkers typically not being able to verify kwargs when they're unpacked from just constructed dictionary:

Snippet:

def foo(bar: int):
    return bar + 1


print(foo(**{"bar": 2}))
# No typing errors.
print(foo(**{"bar":2, "baz":22}))

pyright-play mypy-play

And I've found that PIE804 can be helpful to highlight those cases. Perhaps a note in PIE804 documentation can help advertise this rule as the one also enchancing type safety.
