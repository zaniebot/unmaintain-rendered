[pylint] Implement nested-min-max (W3301)

#4200 opened by @mccullocht on 2023-05-03 05:10
Pull request opened by @mccullocht on 2023-05-03 05:10

Detect nested calls to min() or max() and generate fixes for these, e.g. min(1, min(2, 3)) => min(1, 2, 3)

Behavior differs from pylint when keyword args are present -- we skip analysis in these cases as it may not actually be correct to to merge the nested statements. The following statements would not receive a diagnostic in this implementation, but do in pylint:

min(1, min(2, 3), key=test) => min(1, 2, 3, key=test)
min(1, min(2, 3, key=test)) => min(1, 2, 3)
