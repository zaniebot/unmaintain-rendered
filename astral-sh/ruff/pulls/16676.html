Summary

This PR stabilizes the behavior changes introduced by https://github.com/astral-sh/ruff/pull/13305 that were gated behind preview. The change is that __new__ methods are now no longer flagged by invalid-first-argument-name-for-class-method (N804) but instead by bad-staticmethod-argument (PLW0211)

Test Plan

There have been no new issues or PRs related to N804 or PLW0211 since the behavior change was released in Ruff 0.9.7 (about 3 weeks ago). This is a somewhat recent change but I don't think it's necessary to leave this in preview for another 2 months. The main reason why it was in preview is that it is breaking, not because it is a risky change.
