#16346 opened by @AlexWaygood on 2025-02-24 13:16

Summary

A followup to astral-sh/ruff#16337. Restricting visibility has the advantage that we get unused-code warnings if a method, function or type becomes unused. And it also means that it's harder for external modules to accidentally use methods that are meant to be implementation details, and aren't meant to be for external use.
