I noticed this while reviewing astral-sh/ruff#19100. The current example here is pretty confusing IMO: you'd never need to quote int in a type annotation, because it's always defined (it's a builtin!). Usually you quote annotations if they represent a forward reference to something defined later on. Also, the first "use instead" suggestion will only be valid if you also add from __future__ import annotations to the top of the file; this makes that explicit
