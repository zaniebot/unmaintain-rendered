Use a derive macro for Violations

#14557 opened by @MichaReiser on 2024-11-23 16:28
@MichaReiser

Summary

Use a derive macro instead of a custom proc macro to implement the auto generated Violation methods.

Derive macros are easier to understand because they implement a trait with the same name. It also helps Rustc reasoning about the code. E.g. it now detected that some fields are unused and that all violations only need to be pub(crate). Clippy now also lints our doc comments which it didn't do before.

I generated the documentation locally and verified that rules still have a documentation ;)
