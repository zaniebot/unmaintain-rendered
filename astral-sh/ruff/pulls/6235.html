Remove Parse trait

#6235 opened by @MichaReiser on 2023-08-01 10:39
@MichaReiser

Summary

This PR removes the Parse trait in favour of parse_program, parse_expression and parse_suite.

The motivation for this change is that Parse is a lot of boilerplate and the three functions above are suffucient for almost all use cases. The few other use cases should use parse_suite and then extract the node manually. This has the added benefit that it doesn't give the users the false safety that the operation cannot fail (e.g. IfStatement::parse paniced if you passed anything other than an if statement).
