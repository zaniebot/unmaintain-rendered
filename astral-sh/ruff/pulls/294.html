fix: Make assigns to dunder exception for E402.

#294 opened by @sgryjp on 2022-10-01 10:41
@sgryjp

This commit allow assignments to dunder names to be placed after a before normal imports. PEP 8 says:

Module level "dunders" (i.e. names with two leading and two trailing underscores) such as all, author, version, etc. should be placed after the module docstring but before any import statements except from future imports.

Note that the checking logic is a copy of that of pycodestyle.
