[ty] Improve unsupported-base and invalid-super-argument diagnostics to avoid extremely long lines when encountering verbose types

#22022 opened by @AlexWaygood on 2025-12-17 12:39
Pull request opened by @AlexWaygood on 2025-12-17 12:39

Summary

From the ecosystem report in https://github.com/astral-sh/ruff/pull/22019, it looks like it's not uncommon for these diagnostics to describe types where quite verbose disambiguation is required. This PR therefore improves our verbose diagnostics so that the type in question is printed on a separate line to the diagnostic summary message. This therefore avoids extremely long summary messages.
