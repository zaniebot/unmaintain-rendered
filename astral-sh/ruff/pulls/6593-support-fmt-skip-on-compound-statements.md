The pattern is always the same. Test if there's a trailing skip comment and, if so, call `SuppressedClauseHeader`. The enclosing node remains responsible for formatting the body and the trailing comments.

We could wrap this in a `ClauseHeader` struct but it would result in the same level of nesting because it would need to accept a `Format` object to write the content. It may be worth adding considering that @tjkuson needs it in https://github.com/astral-sh/ruff/pull/6592 to collapse `...` bodies.
