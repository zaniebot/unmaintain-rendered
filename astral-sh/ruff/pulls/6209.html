Implement Black's rules around newlines before and after class docstrings

#6209 opened by @charliermarsh on 2023-07-31 20:21
@charliermarsh
Summary

Black allows up to one blank line before a class docstring, and enforces one blank line after a class docstring. This PR implements that handling. The cases in crates/ruff_python_formatter/resources/test/fixtures/ruff/statement/class_definition.py match Black identically.
