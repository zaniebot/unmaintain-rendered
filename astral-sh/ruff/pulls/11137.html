ruff server: In 'publish diagnostics' mode, document diagnostics are cleared properly when a file is closed

#11137 opened by @snowsignal on 2024-04-25 01:51
Pull request opened by @snowsignal on 2024-04-25 01:51

Summary

Fixes #11114.

As part of the onClose handler, we publish an empty array of diagnostics for the document being closed, similar to ruff-lsp. This prevent phantom diagnostics from lingering after a document is closed. We'll only do this if the client doesn't support pull diagnostics, because otherwise clearing diagnostics is their responsibility.

Test Plan

Diagnostics should no longer appear for a document in the Problems tab after the document is closed.
