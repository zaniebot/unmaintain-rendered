# [ty] Add `respect-type-ignore-comments` configuration option

---

_@MichaReiser_

## Summary

Closes https://github.com/astral-sh/ty/issues/1422

As discussed on https://github.com/astral-sh/ty/issues/1422, this PR introduces a new `analysis.respect-type-ignore-comments` option that defaults to `true`. Setting it to false makes ty ignore any `type: ignore` comment, treating them the same as any other regular comment.

We may decide to split the `unused-ignore-comment` rule into two rules, one for `type: ignore` and another for `ty: ignore` comments but I recommend deferring this decision and waiting on user feedback on whether making this distinction is useful.

I also deferred adding the option to `overrides`. There's no technical reason why we couldn't support overriding analysis options per file. I don't know if there's a use case, and I'd rather make this possible based on user feedback.

One caveat of the `respect-type-ignore-comments` option is that it won't allow a user to specify that they prefer `type: ignore` comments over `ty: ignore`. This will become relevant when we add a `--add-ignore` command. I'm curious to hear more opinions on whether we should support this. I'm leaning towards exclusively recommending `ty: ignore`.  If so, I'd suggest renaming the option to `type-ignore-comments` and allowing one of: `disable`, `prefer`, or `enable`.
