Flag, but don't fix, unused imports in ModuleNotFoundError blocks

#3658 opened by @charliermarsh on 2023-03-21 23:14
@charliermarsh
Summary

This PR acts on the resolved solution from #3378. In charliermarsh/ruff#3288, we changed Ruff to avoid flagging unused import in ModuleNotFoundError blocks, like:

try:
    import orjson

    return True
except ModuleNotFoundError:
    return False

This is a common pattern used to detect module availability, and removing those imports tends to break code.

However, these imports are unused, and it's confusing that Ruff doesn't flag them. Further, this isn't the recommended way to test for module availability -- we have importlib APIs for that.

So this PR modifies the logic to (1) flag unused imports in blocks like the above, but (2) avoid autofixing them, and (3) using a dedicated message to point the user to those importlib APIs.

Closes #3378.
