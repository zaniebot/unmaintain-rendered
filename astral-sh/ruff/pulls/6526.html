Use top-level semantic detection for E402

#6526 opened by @charliermarsh on 2023-08-12 18:41
@charliermarsh
Summary

Noticed in astral-sh/ruff#6378. Given import h; import i, we don't consider import i to be a "top-level" import for E402 purposes, which is wrong. Similarly, we do consider import k to be a "top-level" import in:

if __name__ == "__main__":
    import j; \
import k

Using the semantic detection, rather than relying on newline position, fixes both cases.
