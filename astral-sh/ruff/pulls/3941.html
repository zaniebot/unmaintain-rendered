Support pyright: ignore comments

#3941 opened by @charliermarsh on 2023-04-12 02:51
Pull request opened by @charliermarsh on 2023-04-12 02:51

Summary

This PR enables us to flag Pyright's ignore comments (e.g., x: int = "str" # pyright: ignore [reportGeneralTypeIssues]). Previously, we only supported Mypy-style # type: ignore comments.

A couple observations from this work:

  1. Previously, we considered # type: ignore [specificCode] to (wrongly) be an error. Mypy and Pyright allow you to include whitespace between the ignore and the opening bracket. In Pyright, that actually seems to be the common convention, so it's important to support.
  2. On that front: I found the relevant regexes in Mypy and based the work on those.
  3. Interestingly, Pyright does respect # type: ignore comments, but it treats any # type: ignore as a blanket ignore (so # type: ignore[specificCode] is treated as equivalent to # type: ignore, as far as I can tell). I'm guessing that this is because # type: ignore codes are mapped to Mypy's registry, but Pyright still wants to respect the user intent). This doesn't really impact us, just an observation :)

Closes #3935.
