[red-knot] Remove Scope::name

#12137 opened by @MichaReiser on 2024-07-01 12:50
@MichaReiser

Summary

This PR removes Scope::name because it is currently unused outside tests and it requires cloning the names of every symbol and class which can get expensive.

I can see how having a name might be useful for debugging. I'm open to have a debug-only name stored on Scope but we should then design it that way and also avoid writing tests around it.
