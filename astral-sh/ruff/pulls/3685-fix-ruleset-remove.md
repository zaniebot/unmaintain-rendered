# Fix RuleSet.remove

---

_@MichaReiser_

This PR fixes `RuleSet.remove(rule)` where calling it with a `rule` that isn't in the set resulted in it being added to the set.

```rust
let mut set = RuleSet::default();
set.remove(Rule::AmbiguousFunctionName);

set.contains(AmbiguousFunctionName) // <--- This is now true, but should not
```

The reason for the bug is that the `remove` operation uses bitwise `XOR` rather than `and NOT`:

<table>
<tr>
	<td>
	<td>XOR
	<td>and NOT
<tr>
	<td><code>self</code>
	<td><code>00000001</code>
	<td><code>00000001</code>
<tr>
	<td>
	<td><code>XOR</code>
	<td><code>&</code>
<tr>
	<td><code>other</code> / <code>!other</code>
	<td><code>00100000</code>
	<td><code>11011111</code>
<tr>
	<td><code>result</code>
	<td><code>000100001</code>
	<td><code>000000001</code>
</table>

Fixes #3684
