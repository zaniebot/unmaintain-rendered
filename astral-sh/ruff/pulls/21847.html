[ty] Remove legacy concise_message fallback behavior

#21847 opened by @MichaReiser on 2025-12-08 15:01
@MichaReiser

Summary

This PR removes the legacy fallback behavior to use the primary annotation when the diagnostic's primary message is empty.

The old behavior was necessary during the transition from ty's very first diagnostic system to what we use now but, I think, all call sites now use the new system.

Test Plan

I did review all Diagnostic::new call sites and are fairly certain that they pass a non-empty message.
