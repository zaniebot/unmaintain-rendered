Summary

Another small cleanup I'm splitting out from the branch I have locally working towards astral-sh/ruff#13933. The traits HasScopedUseId and HasScopedAstId don't need to be generic: for both traits, the Id associated type is the same for all implementations of the trait. We can remove some complexity here by removing the associated types and specifying concrete types in the trait definitions rather than associated types.
