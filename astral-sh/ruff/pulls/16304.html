But... looking at this specific code in more detail makes me realise that we probably aren't handling descriptors on metaclasses correctly yet -- e.g. at runtime you can do this:

>>> class Descriptor:
...     def __get__(self, instance, owner=None):
...         if instance is None:
...             return self
...         else:
...             return 42
...             
>>> class Meta(type):
...     attr = Descriptor()
...     
>>> class UsesMeta(metaclass=Meta): ...
... 
>>> UsesMeta.attr
42

And in fact, it doesn't seem like we delegate attribute access from class-literal types to their metaclass at all right now üôÅ we emit a false positive even on this:
