Allow non-ruff.toml-named files for --config

#2463 opened by @charliermarsh on 2023-02-02 02:09
@charliermarsh

Previously, if you passed in a file on the command-line via --config, it had to be named either pyproject.toml or ruff.toml -- otherwise, we errored. I think this is too strict. pyproject.toml is a special name in the ecosystem, so we should require that; but otherwise, let's just assume it's in ruff.toml format.

As an alternative, we could add a --pyproject argument for pyproject.toml, and assume anything passed to --config is in ruff.toml format. But that would be a breaking change and is arguably more confusing. (This isn't a breaking change, since it only loosens the CLI.)

Closes #2462.
