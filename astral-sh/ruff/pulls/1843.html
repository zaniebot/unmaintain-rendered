Use absolute paths for --stdin-filename matching

#1843 opened by @charliermarsh on 2023-01-13 01:49
Pull request opened by @charliermarsh on 2023-01-13 01:49

Non-basename glob matches (e.g., for --per-file-ignores) assume that the path has been converted to an absolute path. (We do this for filenames as part of the directory traversal.) For filenames passed via stdin, though, we're missing this conversion. So --per-file-ignores that rely on the basename worked as expected, but directory paths did not.

Closes #1840.
