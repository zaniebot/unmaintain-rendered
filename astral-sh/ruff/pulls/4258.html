Respect insertion location when importing symbols

#4258 opened by @charliermarsh on 2023-05-06 20:15
@charliermarsh

Summary

Previously, Importer would only track the most recently-seen top-level import statement, and would add any new imports after that statement. However, because we visit function bodies and other scopes in a deferred manner, then if there are top-level imports below a given function, we'll visit those imports, then visit the function body, then insert the new import after the function.

This causes two problems: (1) it breaks the invariant, enforced in the fixer, that edits are ordered, since the import insertion can come after the edit that makes use of the symbol; and (2) in theory, this could lead to broken code, since a function could be called before we reach the import statement (since that statement comes later in the module).

The fix here is to track all import statements, and return the last import statement before a given edit location.

Closes #4208.
