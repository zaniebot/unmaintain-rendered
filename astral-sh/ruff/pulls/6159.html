Skip BOM when determining Locator's line starts

#6159 opened by @charliermarsh on 2023-07-28 17:52
@charliermarsh
Summary

If a file has a BOM, the import sorter always reports the imports as unsorted. The acute issue is that we detect that the line has leading content (before the imports), which we always consider a violation. Rather than fixing that one site, this PR instead makes .line_start BOM-aware.

Fixes astral-sh/ruff#6155.
