Fix comment formatting for yielded tuples

#6603 opened by @charliermarsh on 2023-08-15 21:17
@charliermarsh

Summary

Closes https://github.com/astral-sh/ruff/issues/6384, although I think the issue was fixed already on main, for the most part.

The linked issue is around formatting expressions like:

def test():
    (
        yield 
        #comment 1
        * # comment 2
        # comment 3
        test # comment 4
    )

On main, prior to this PR, we now format like:

def test():
    (
        yield (
            # comment 1
            # comment 2
            # comment 3
            *test
        )  # comment 4
    )

Which strikes me as reasonable. (We can't test this, since it's a syntax error after for our parser, despite being a syntax error in both cases from CPython's perspective.)

Meanwhile, Black does:

def test():
    (
        yield
        # comment 1
        *  # comment 2
        # comment 3
        test  # comment 4
    )

So our formatting differs in that we move comments between the star and the expression above the star.

As of this PR, we also support formatting this input, which is valid:

def test():
    (
        yield 
        #comment 1
        * # comment 2
        # comment 3
        test, # comment 4
        1
    )

Like:

def test():
    (
        yield (
            # comment 1
            (
                # comment 2
                # comment 3
                *test,  # comment 4
                1,
            )
        )
    )

There were two fixes here: (1) marking starred comments as dangling and formatting them properly; and (2) supporting parenthesized comments for tuples that don't contain their own parentheses, as is often the case for yielded tuples (previously, we hit a debug assert).

So, prior to https://github.com/astral-sh/ruff/pull/6485, this method was written as:

fn handle_trailing_expression_starred_star_end_of_line_comment<'a>(
    comment: DecoratedComment<'a>,
    starred: &'a ast::ExprStarred,
) -> CommentPlacement<'a> {
    if comment.line_position().is_own_line() {
        return CommentPlacement::Default(comment);
    }

    if comment.following_node().is_none() {
        return CommentPlacement::Default(comment);
    }

    CommentPlacement::leading(starred, comment)
}

In that PR, it was brought up that coverage didn't seem to hit those lines, so I removed them and tests were unchanged. However, with the new tests introduced here, the current implementation gave us this formatting (for the same input mentioned in the PR summary):

def test():
    (
        yield (
            # comment 1
            # comment 2
            # comment 3
            *test
        )  # comment 4
    )

This is reasonable but a bigger deviation from Black.

The new version attaches any comments between the * and the starred element as dangling, and formats them around the * similar to Black.
