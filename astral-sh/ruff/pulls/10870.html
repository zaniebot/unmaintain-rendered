Add display to TokenKind for error message

#10870 opened by @dhruvmanila on 2024-04-11 09:26
@dhruvmanila
Summary

This PR moves the Display implementation to a dedicated method display. I didn't choose as_str because it's not exactly a string representation.

Using a method allows us to tweak the display string slightly:

  1. For raw tokens such as !, ?, , and keyword, surround them with a single quote
  2. For certain tokens which are self-descriptive, use the lowercase version (int, float, comment)
  3. For all of the remaining tokens, keep the value same as the variant name

That said, I'm open to changing it if others have any opinions :)
