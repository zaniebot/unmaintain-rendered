# Add `MultiMap` implementation for storing comments

---

_@MichaReiser_

## Summary

This is the first PR in a series that extracts comments from the source code and stores them in a map per node. The implementation isn't used anywhere yet.

This PR copies over the `MultiMap` from rome. It is an optimized multi-map implementation that, in practice, stores all comments in a single vector.  It uses the fact that comments are inserted in-order because we want to maintain the same order as they appear in the source code. 

For more details see https://github.com/rome/tools/pull/3230 or read the doc-comment

## Test Plan

The implementation backs Rome's formatter (deployed to production) and comes with a few unit tests.
