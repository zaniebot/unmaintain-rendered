Replace is-macro with implementation in enums

@Glyphack

Summary

Replace usage of is_macro in generated AST models with implementation of those methods. Addresses this comment: https://github.com/astral-sh/ruff/pull/15544/files#r1919735841

The implementation is straightforward, there are just two decisions:

  1. For enums with one variant we emit the catch all cases in the match. Otherwise there is a clippy warning.
  2. All impl blocks have allow dead_code and match_wildcard_for_single_variants. Some of the methods are unused. Also I don't think we need to care about catch all case since the methods are obviously matching for one thing. I can also make this per function but I decided do it on the impl to generate smaller file.

Sadly we cannot remove the dependency yet, but we can if we replace more of the ast files with auto generated code.

Test Plan

No other file has changed. And the compilation and tests pass.
