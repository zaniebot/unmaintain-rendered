Summary

Implements bidirectional type inference using function return type annotations.

This PR was originally proposed to solve astral-sh/ty#1167, but this does not fully resolve it on its own. Additionally, I believe we need to allow dataclasses to generate their own __new__ methods, use constructor return types ​​for inference, and a mechanism to discard type narrowing like & ~AlwaysFalsy if necessary (at a more general level than this PR).
