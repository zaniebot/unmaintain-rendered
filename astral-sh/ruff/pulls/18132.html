[ty] Fix relative imports in stub packages

#18132 opened by @MichaReiser on 2025-05-16 13:19
@MichaReiser

Summary

Relative imports call file_to_module which always returned None for stub packages because a stub package path like pyyaml-stubs/__init__.py can't be converted into a valid ModuleName because -stubs isn't a valid identifier.

This PR fixes the path to module name conversion by stripping the -stubs suffix.

Fixes https://github.com/astral-sh/ty/issues/408
