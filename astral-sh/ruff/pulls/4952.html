Implement Binary expression without best_fitting

#4952 opened by @MichaReiser on 2023-06-08 07:31
Pull request opened by @MichaReiser on 2023-06-08 07:31

Summary

This PR implements the binary expression formatting without using best_fitting. This is good for performance because:

  • best_fitting requires memoizing its inner content because it includes the left and right in each variant and doing so requires an allocation for each memoized element
  • Printing memoized elements adds a small overhead to the Printer because memoized elements each have their own IR buffer that the printer now has to queue before the, otherwise flat, IR element buffer
  • Printing best fitting has exponential complexity when nesting because it has to try each variant.
