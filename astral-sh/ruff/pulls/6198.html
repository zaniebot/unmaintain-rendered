Avoid parenthesizing comprehension element

#6198 opened by @dhruvmanila on 2023-07-31 15:09
Pull request opened by @dhruvmanila on 2023-07-31 15:09

Summary

This PR adds a new precedence level for the comprehension element. This fixes the generator to not add parentheses around the comprehension element every time.

The new precedence level is COMPREHENSION_ELEMENT and it should occur after the NAMED_EXPR precedence level because named expressions are always parenthesized.

This matches the behavior of Python ast.unparse and tested with the following snippet:

import ast

code = ""
ast.unparse(ast.parse(code))
