Treat parameters-with-newline as empty in function formatting

#7550 opened by @charliermarsh on 2023-09-20 17:02
@charliermarsh
Summary

If a function has no parameters (and no comments within the parameters' ()), we're supposed to wrap the return annotation whenever it breaks. However, our empty_parameters test didn't properly account for the case in which the parameters include a newline (but no other content), like:

def get_dashboards_hierarchy(
) -> Dict[Type['BaseDashboard'], List[Type['BaseDashboard']]]:
    """Get hierarchy of dashboards classes.

    Returns:
        Dict of dashboards classes.
    """
    dashboards_hierarchy = {}

This PR fixes that detection. Instead of lexing, it now checks if the parameters itself is empty (or if it contains comments).

Closes astral-sh/ruff#7457.
