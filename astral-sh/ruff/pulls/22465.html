[ty] Rank top-level module symbols above most other symbols

#22465 opened by @BurntSushi on 2026-01-08 20:09
Pull request opened by @BurntSushi on 2026-01-08 20:09

This makes it so, e.g., os<CURSOR> will suggest the top-level stdlib os module even if there is an os symbol elsewhere in your project.

The way this is done is somewhat overwrought, but it's done to avoid suggesting top-level modules over other symbols already in scope.

Fixes https://github.com/astral-sh/ty/issues/1852
