@chanman3388 reviewed on 2024-04-25 14:00
Review comment by @chanman3388 on crates/ruff_linter/resources/test/fixtures/pylint/class_variable_slots_conflict.py:1 on 2024-04-25 14:00

This is fine according to pylint. Also: https://docs.python.org/3/library/dataclasses.html

So what you've written is fine, but:

@dataclass(slots=True)
class Test:
    __slots__ = ["a"]
    a: int

will raise a TypeError.
