Fix bracket spacing for single-element tuples in f-string expressions

#15537 opened by @MichaReiser on 2025-01-16 17:51
@MichaReiser
Summary

Fixes astral-sh/ruff#15535#issuecomment-2596286050

The formatter always parenthesizes single-element tuples even if they weren't in the source. This lead to an instability before where the formatter added spacing around the f-strings { {}, } that then were removed when reformatting the code because the tuple's now parenthesized {({},)}.

Test Plan

Added test. I reviewed our other expression formatting and I didn't find any other, except generators, that introduce parentheses. For generators, unparenthesized generators aren't allowed in an expression context.
