Default to isort's import sort logic

#691 opened by @charliermarsh on 2022-11-12 03:41
@charliermarsh

We now resolve to isort's sorting logic -- which is a little complicated, but in short:

  • Modules are sorted in a case-insensitive way.
  • Module members (the targets in an import from) are sorted based on a heuristic that tries to order constants, then classes, then all other variables.
