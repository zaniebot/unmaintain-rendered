Don't move type param opening parenthesis comment

#8163 opened by @dhruvmanila on 2023-10-24 11:52
Pull request opened by @dhruvmanila on 2023-10-24 11:52

Summary

This PR fixes the issue to avoid collapsing the type param declaration if there's a comment after the opening parenthesis. For example,

type foo[  # comment
    A,
    B
] = int

Here, we'll preserve the comment on the same line as is being done for other similar type of nodes.
