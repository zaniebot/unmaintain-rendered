Summary

Part of #18972

This PR makes pytest-warns-too-broad (PT030)'s example error out-of-the-box

Old example

import pytest


def test_foo():
    with pytest.warns(RuntimeWarning):
        ...

    # empty string is also an error
    with pytest.warns(RuntimeWarning, match=""):
        ...

New example

import pytest


def test_foo():
    with pytest.warns(Warning):
        ...

    # empty string is also an error
    with pytest.warns(Warning, match=""):
        ...

RuntimeWarning is not in the default warns-require-match-for list, while Warning is. The "Use instead" section was also updated similarly
