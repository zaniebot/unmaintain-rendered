### Release Notes

<details>
<summary>vitejs/vite-plugin-react-swc (@&#8203;vitejs/plugin-react-swc)</summary>

### [`v3.6.0`](https://togithub.com/vitejs/vite-plugin-react-swc/blob/HEAD/CHANGELOG.md#360)

[Compare Source](https://togithub.com/vitejs/vite-plugin-react-swc/compare/v3.5.0...v3.6.0)

##### Add parserConfig option

This will unlock to use the plugin in some use cases where the original source code is not in TS. Using this option to keep using JSX inside `.js` files is highly discouraged and can be removed in any future version.

### [`v3.5.0`](https://togithub.com/vitejs/vite-plugin-react-swc/blob/HEAD/CHANGELOG.md#350)

[Compare Source](https://togithub.com/vitejs/vite-plugin-react-swc/compare/v3.4.1...v3.5.0)

##### Update peer dependency range to target Vite 5

There were no breaking change that impacted this plugin, so any combination of React plugins and Vite core version will work.

##### Align jsx runtime for optimized dependencies

This will only affect people using internal libraries that contains untranspiled JSX. This change aligns the optimizer with the source code and avoid issues when the published source don't have `React` in the scope.

Reminder: While being partially supported in Vite, publishing TS & JSX outside of internal libraries is highly discouraged.

### [`v3.4.1`](https://togithub.com/vitejs/vite-plugin-react-swc/blob/HEAD/CHANGELOG.md#341)

[Compare Source](https://togithub.com/vitejs/vite-plugin-react-swc/compare/v3.4.0...v3.4.1)

##### Add support for `.mts` (fixes [#&#8203;161](https://togithub.com/vitejs/vite-plugin-react-swc/issues/161))

Using CJS in source code will not work in Vite (and will never be supported), so this is better to only use `.ts`.

But to better align with [Vite core defaults](https://vitejs.dev/config/shared-options.html#resolve-extensions), `.mts` extension will now be processed like `.ts`. This maybe reverted in a future major.

### [`v3.4.0`](https://togithub.com/vitejs/vite-plugin-react-swc/blob/HEAD/CHANGELOG.md#340)

[Compare Source](https://togithub.com/vitejs/vite-plugin-react-swc/compare/v3.3.2...v3.4.0)

-   Add `devTarget` option (fixes [#&#8203;141](https://togithub.com/vitejs/vite-plugin-react-swc/issues/141))
-   Disable Fast Refresh based on `config.server.hmr === false` instead of `process.env.TEST`
-   Warn when plugin is in WebContainers (see [#&#8203;118](https://togithub.com/vitejs/vite-plugin-react-swc/issues/118))
-   Better invalidation message when an export is added & fix HMR for export of nullish values ([#&#8203;143](https://togithub.com/vitejs/vite-plugin-react-swc/issues/143))
