Avoid attempting to fix .format(...) calls with too-few-arguments

#6401 opened by @charliermarsh on 2023-08-07 18:53
Pull request opened by @charliermarsh on 2023-08-07 18:53

Summary

We can anticipate earlier that this will error, so we should avoid flagging the error at all. Specifically, we're talking about cases like "{1} {0}".format(*args)", in which we'd need to reorder the arguments in order to remove the 1 and 0, but we can't reorder the arguments since they're not statically analyzable.

Closes https://github.com/astral-sh/ruff/issues/6388.
