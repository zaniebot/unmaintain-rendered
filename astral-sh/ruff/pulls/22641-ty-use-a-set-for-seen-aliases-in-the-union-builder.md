# [ty] Use a set for `seen_aliases` in the union builder

---

_@AlexWaygood_

## Summary

We only ever use `seen_aliases` to collect aliases that we've... seen... and we check whether we've seen an alias before by doing an `O(n)` `.contains()` check against the `Vec` of types. But if we use a hash-set, that `.contains()` check becomes `O(1)`.

I'm not sure if this will really improve performance measurably (most unions don't have that many aliases in them, let alone PEP-695 aliases), but it feels like it makes sense conceptually anyway.

## Test Plan

`cargo test`
