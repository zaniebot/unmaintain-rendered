perf(formatter): Improve is_expression_parenthesized performance

#5825 opened by @MichaReiser on 2023-07-17 08:18
Pull request opened by @MichaReiser on 2023-07-17 08:18

Summary

This PR improves the performance of is_expression_parenthesized by removing the unnecessary check whether the ( is part of a comment if the token is on the same line as the start character of the expression. Omitting the check for characters on the same line as the expression start is safe because, if there's a comment, then the expression would be commented out too.

This brings down the formatting time of this large file from 16s to 600ms
