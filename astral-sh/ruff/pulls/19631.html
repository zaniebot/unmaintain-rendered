Summary

Fixes https://github.com/astral-sh/ty/issues/911

Similar to https://github.com/astral-sh/ruff/pull/18252

The Drop relative drop order between the Database and Arc's that we call into_inner on (require exactly one reference) matter because Salsa's cancellation only waits for the Db to drop. That means, the db must come last!

I don't know how to avoid this more systematically but I'm also not convinced that we have to. There are only very few Arc's where this matters (just Index really) and there are very few places where it is used.

Test Plan

I typed like a mad man while the workspace diagnostic request was in flight and I haven't been able to reproduce the crash anymore
