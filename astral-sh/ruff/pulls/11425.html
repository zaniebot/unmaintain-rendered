Move string-prefix enumerations to a separate submodule

#11425 opened by @AlexWaygood on 2024-05-14 14:09
Pull request opened by @AlexWaygood on 2024-05-14 14:09

Summary

This moves the string-prefix enumerations in ruff_python_ast to a separate submodule. I think this helps clarify that these prefixes are purely abstract: they only depend on each other, and do not depend on any of the other code in nodes.rs in any way. Moreover, while various AST nodes use them, they're not really nodes themselves, so they feel slightly out of place in nodes.rs.

I considered moving all of them to str.rs, but it felt like enough code that it could be a separate submodule.
