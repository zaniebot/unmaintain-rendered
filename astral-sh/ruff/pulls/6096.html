Remove parser dependency from ruff-python-ast

#6096 opened by @MichaReiser on 2023-07-26 13:07
Pull request opened by @MichaReiser on 2023-07-26 13:07

Summary

The ultimate goal is to pull the RustPython fork into Ruff's monorepo and merging rustpython-ast and ruff_python_ast. However, this requires that rust_python_ast has no dependency on rustpython-parser because the parser depends on the AST crate.

The first step was to rewrite all references to rustpython_parser::ast to rustpython_ast. The second step was harder because I had to break up the ruff_python_ast crate

I'm not a 100% happy with the outcome because I needed to introduce a few new crates, merge and rename an existing crates:

  • Renamed ruff_rustpython to ruff_python_parser. I plan merging the two
  • Merge ruff_textwrap into ruff_python_trivia (it uses PythonWhitespace)
  • New ruff_source_file that defines SourceFile, Locator, SourcePosition and LineIndex. These structs aren't Python specific but used by many helper methods in the ast crate.
  • New ruff_python_codegen crate. Contains Stylist and Generator
  • New ruff_python_index crate. Contains Indexer and CommentRanges.

Review

This PR does not introduce any logical changes. I only moved code, mainly files, around.
