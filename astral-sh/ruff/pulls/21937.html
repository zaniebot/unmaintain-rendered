[ty] Use datatest instead of dirtest

#21937 opened by @MichaReiser on 2025-12-12 08:40
@MichaReiser
Summary

Same as astral-sh/ruff#21933 but for our mdtests.

datatest-stable uses a custom test-harness that mimics cargo test and supports cargo nextest to create a separate test for every file found in a given directory. Unlike dir-tests, datatest-stable doesn't require re-compilation after adding or removing a test file (because it uses a custom test harness). This should allow us (people like me who aren't using the mdtest Python script) to iterate faster on mdtests, since we no longer have to recompile ty_python_semantic whenever a markdown file changes.
