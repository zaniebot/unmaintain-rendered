[ruff][ext-lint] 5: per-external-linter toml config

#21786 opened by @pieterh-oai on 2025-12-04 06:08
@pieterh-oai

Summary

This came up in passing in #21415 (https://github.com/astral-sh/ruff/pull/21415#issuecomment-3526669293), so naturally I felt compelled to see what it'd look like.

This adds per-external-linter configuration. The config is just an optional TOML table (lint.ext-lint.name-of-linter.config) that lives in either the 'root' config (the one that defines the external linter) or in a supplementary config file in a subdirectory below that. On the Python side, the config is exposed as a dict via ctx.config.

Somewhat nice/unexpected orthogonality: by including the config in the ExternalLintRegistry and making it hashable, we automagically get different runtime environments for registries that have identical rules but distinct config settings.

(Renamed some of the TOML keys to consistently use ext-lint instead of a combination of external-linter, external-ast, etc.)

It's possible I've missed a plausible scenario where it makes sense to have configs for a linter that doesn't formally exist (e.g. if it's disabled).
