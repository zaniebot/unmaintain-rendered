[ty] Visit class arguments in source order for semantic tokens

#22063 opened by @RasmusNygren on 2025-12-18 21:53
@RasmusNygren

Summary

Before we would hit a debug assertion that tokens were inserted out-of-order for statements like class Foo(m=x, m). This was because arguments were always inserted before the keyword arguments in class definitions. This changes the ordering to use source order.

Test Plan

Added a small test and validated in the playground that this no longer panics.
