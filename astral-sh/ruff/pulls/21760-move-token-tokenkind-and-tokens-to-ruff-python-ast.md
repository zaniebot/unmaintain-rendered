# Move `Token`, `TokenKind` and `Tokens` to `ruff-python-ast`

---

_@MichaReiser_

## Summary

Initially, `Token`, `TokenKind` and `Tokens` were only used within the `Parser`. However,
this is no longer the case. We now use `Tokens` in many downstream crates as part of formatting,
linting, or when providing IDE features. 

Today, this requires that any of those crates have to depend on `ruff_python_parser` even if they're never calling the parser itself,
just so that they can get access to the `Token` types defined in `ruff_python_parser`. 

This PR extracts `Tokens` into `ruff_python_ast` which is our crate for: I only want to stuff with it but I don't care about how it's retrieved (e.g. by parsing or any other means).
