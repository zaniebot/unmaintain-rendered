[ty] Include all members on type in autocompletion suggestions for type[] types

#21670 opened by @AlexWaygood on 2025-11-27 18:56
@AlexWaygood

Summary

I noticed while reviewing https://github.com/astral-sh/ruff/pull/21650 that ty doesn't currently include members on type (such as __base__, __qualname__, __bases__ etc) when providing autocompletion suggestions on a type[] type, e.g.

def f(x: type[int]):
    x.<CURSOR>

This fixes that.
