[ty] Handle configuration errors in LSP more gracefully

#19262 opened by @MichaReiser on 2025-07-10 13:43
@MichaReiser
Summary

Up to now, ty crashed if the project configuration contained invalid syntax are was invalid otherwise.

This PR handles this case more gracefully by showing an error popup and falling back to the default settings. We probably want to mark the database as "in an error state" in the future when the LSP offers more destructive operations like formatting so that these can be disabled until the configuration errors are resolved.

Closes astral-sh/ty#76
