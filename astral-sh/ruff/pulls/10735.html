Build codspeed benchmarks by calling cargo directly

#10735 opened by @MichaReiser on 2024-04-02 12:19
@MichaReiser
Summary

This PR changes the benchmark CI workflow to call cargo build directly instead of calling codspeed build because codspeed uses a old version of Cargo (1.66) and it resolves feature flags differently, making astral-sh/ruff#10700 not build correctly.

I hope we can migrate back to codspeed build once codspeed upgrades its cargo dependency (I've opened a support thread in their discord), but until then lets use this to unblock us
