=================== This release fixes a couple bugs in Jiff's parser. In particular, a regression was introduced in jiff 0.2.11 where its RFC 2822 parser could panic on some inputs. Previous releases of Jiff are unaffected.

v0.2.11

Compare Source

=================== This release includes new APIs for customizing Jiff's strtime behavior along with a few minor bug fixes. Jiff's strtime formatting API has also been optimized. It's about twice as fast as it was.

This release also coincides with the publication of jiff-icu 0.2.0-beta.2, which has support for icu 2.0.0-beta.2.

Enhancements:

  • #‚Äã338: Add support for the %c, %r, %X and %x conversion specifiers.
  • #‚Äã341: Add support for %q in jiff::fmt::strtime (prints quarter of year).
  • #‚Äã342: Add support for %::z and %:::z in jiff::fmt::strtime.
  • #‚Äã344: Add support for %N in jiff::fmt::strtime (alias for %9f).
  • #‚Äã350: Add a "lenient" mode for strtime formatting APIs that ignores most errors.

Bug fixes:

  • #‚Äã328: Document default precision behavior of Display impls for datetime types.
  • #‚Äã340: Allow whitespace in more places in RFC 2822 parser (improves spec compliance).
  • #‚Äã346: TimeZone::get("UTC") should now always return TimeZone::UTC.

Performance:

  • #‚Äã338: Jiff's strftime APIs are now approximately twice as fast as they were. Performance should be comparable to chrono and time's prebuilt APIs.
