_Comment by @MichaReiser on 2026-01-04 19:00_

> FxOrderMap is a thin wrapper around FxIndexMap that also adds order-based Hash and Eq implementations

This is the theory :) A fixpoint cycle using `IndexMap` converges if the maps are equal except for their order but we would do an extra iteration when using `OrderMap`. Now, this can result in *too many iterations* errors as we've seen with `exported_names` but enforcing a consistent order might help determinism (maybe?)
