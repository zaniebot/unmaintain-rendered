[ty] improve TypedDict narrowing logic

#22786 opened by @mtshiba on 2026-01-21 11:32
@mtshiba
Summary

While working on astral-sh/ruff#22238, I noticed that the following narrowing wasn't working:

from typing import TypedDict, Literal

class Foo(TypedDict):
    tag: Literal["foo"]

class Bar(TypedDict):
    tag: Literal[42]

type TaggedUnion = Foo | Bar

def _(u: TaggedUnion):
    reveal_type(u["tag"])
    if u["tag"] == "foo":
        reveal_type(u)  # should be: Foo, but: Foo | Bar
    else:
        reveal_type(u)  # should be: Bar, but: Foo | Bar

The problem was that the narrowing logic for TypedDict didn't take into account type aliases that point to TypedDict. I fixed that and made the handling cleaner and less error-prone (I think).

Comment by @mtshiba on 2026-01-21 11:50

A bigger problem is that Type::as_{TypeVariant} doesn't expand TypeAlias to check that the value type is the target type variant. This is probably what blocks the narrowing shown above. I don't know the details of why this isn't done, but my guess is that it's to avoid infinitely expanding type aliases? I previously made a change to reject type aliases with meaningless infinite recursion and replace them with Divergent, so I think Type::as_{TypeVariant} can safely expand TypeAlias.
