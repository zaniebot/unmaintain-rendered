Avoid attributing runtime references to module-level imports

#4942 opened by @charliermarsh on 2023-06-07 21:48
Pull request opened by @charliermarsh on 2023-06-07 21:48

Summary

This PR fixes a subtle case in import-use attribution, illustrated here:

from __future__ import annotations

from typing import TYPE_CHECKING, cast

if TYPE_CHECKING:
    from threading import Thread


def fn(thread: Thread):
    from threading import Thread  # Flagged as F401

    casted_thread: Thread = cast(Thread, thread)
    return casted_thread

print(fn(1))

When we do cast(Thread, thread), we need Thread to be attributed to the import within fn. This is subtle, because similar to Pyright and other type checkers, the Thread on the left-hand side of the assignment (casted_thread: Thread) is resolved to the global import.

Closes #4939.
