Set non-empty range for indentation diagnostics

#4005 opened by @MichaReiser on 2023-04-18 06:19
@MichaReiser

I noticed in the byte offsets refactor that we set an empty range for all logical-lines indentation diagnostics. The downside of an empty range is that the TextEmitter can't draw a carret pointing to the violating code, which looks odd.

This PR sets location of the violation to the start of the violating line.
