This update contains breaking changes that remove the raw API with the hope of centralising on the HashTable API in the future. You can follow the discussion and progress in #‚Äã545 to discuss features you think should be added to this API that were previously only possible on the raw API.

Added
  • Added borsh feature with BorshSerialize and BorshDeserialize impls. (#‚Äã525)
  • Added Assign impls for HashSet operators. (#‚Äã529)
  • Added Default impls for iterator types. (#‚Äã542)
  • Added HashTable::iter_hash{,_mut} methods. (#‚Äã549)
  • Added Hash{Table,Map,Set}::allocation_size methods. (#‚Äã553)
  • Implemented Debug and FusedIterator for all HashTable iterators. (#‚Äã561)
  • Specialized Iterator::fold for all HashTable iterators. (#‚Äã561)
Changed
  • Changed hash_set::VacantEntry::insert to return OccupiedEntry. (#‚Äã495)
  • Improvedhash_set::Difference::size_hint lower-bound. (#‚Äã530)
  • Improved HashSet::is_disjoint performance. (#‚Äã531)
  • equivalent feature is now enabled by default. (#‚Äã532)
  • HashSet operators now return a set with the same allocator. (#‚Äã529)
  • Changed the default hasher to foldhash. (#‚Äã563)
  • ahash feature has been renamed to default-hasher. (#‚Äã533)
  • Entry API has been reworked and several methods have been renamed. (#‚Äã535)
  • Hash{Map,Set}::insert_unique_unchecked is now unsafe. (#‚Äã556)
  • The signature of get_many_mut and related methods was changed. (#‚Äã562)
Fixed
  • Fixed typos, stray backticks in docs. (#‚Äã558, #‚Äã560)
Removed
  • Raw entry API is now under raw-entry feature, to be eventually removed. (#‚Äã534, #‚Äã555)
  • Raw table API has been made private and the raw feature is removed; in the future, all code should be using the HashTable API instead. (#‚Äã531, #‚Äã546)
  • rykv feature was removed; this is now provided by the rykv crate instead. (#‚Äã554)
  • HashSet::get_or_insert_owned was removed in favor of get_or_insert_with. (#‚Äã555)
