[ty] Improve ability to solve TypeVars when they appear in unions

#19829 opened by @AlexWaygood on 2025-08-08 13:50
@AlexWaygood
Summary

This PR moves the Union branch higher up in the match expression in SpecializationBuilder::infer(). Doing this improves our ability to solve TypeVars when they appear in unions (see the primer report!)

Test Plan

Mdtests added. The first mdtest (TypeVars with an upper bound in a union) does not pass on main. The second one (constrained TypeVars in a union) does pass on main, but was accidentally broken in astral-sh/ruff#19811, and I only found out about the breakage from primer because we had some missing test coverage. So I'm adding that test coverage here!
