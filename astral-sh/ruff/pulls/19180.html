[ty] Add an instance of an Any subclass to the property tests

#19180 opened by @AlexWaygood on 2025-07-07 14:31
@AlexWaygood
Summary

Nominal instance types where the class has Any in its MRO "break the normal rules" of normal instance-type subtyping in lots of interesting ways. The most recent example of where this has caused an issue is astral-sh/ty#764. To help protect against issues like that, this PR adds the type unittest.mock.Mock (which has Any in its MRO) from the stdlib to the types generated by our property tests.
