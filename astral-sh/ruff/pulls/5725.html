Extend PEP 604 rewrites to support some quoted annotations

#5725 opened by @charliermarsh on 2023-07-13 04:11
@charliermarsh
Summary

Python doesn't allow "Foo" | None if the annotation will be evaluated at runtime (see the comments in the PR, or the semantic model documentation for more on what this means and when it is true), but it does allow it if the annotation is typing-only.

This, for example, is invalid, as Python will evaluate "Foo" | None at runtime in order to populate the function's __annotations__:

def f(x: "Foo" | None): ...

This, however, is valid:

def f():
    x: "Foo" | None

As is this:

from __future__ import annotations

def f(x: "Foo" | None): ...

Closes #5706.
