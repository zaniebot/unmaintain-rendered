Use Jupyter mode while parsing Notebook files

#5552 opened by @dhruvmanila on 2023-07-06 05:05
Pull request opened by @dhruvmanila on 2023-07-06 05:05

Summary

Enable using the new Mode::Jupyter for the tokenizer/parser to parse Jupyter line magic tokens.

The individual call to the lexer i.e., lex_starts_at done by various rules should consider the context of the source code (is this content from a Jupyter Notebook?). Thus, a new field source_type (of type PySourceType) is added to Checker which is being passed around as an argument to the relevant functions. This is then used to determine the Mode for the lexer.

Add new test cases to make sure that the magic statement is considered while generating the diagnostic and autofix:

I think introducing a new PyFileSource is much better and robust solution:

enum LanguageVariant {
	#[default]
	Standard,
	Pyi,
	Ipynb
}

impl LanguageVariant {
	fn as_mode(&self) -> Mode {
		match self {
			Self::Standard | Self::Pyi => Mode::Module,
			Self::Ipynb => Mode::Jupyter
		}
	}
}

struct PyFileSource {
	variant: LanguageVariant
}

impl PyFileSource {
	fn as_mode(&self) -> Mode {
		self.variant.as_mode()
	}
}
