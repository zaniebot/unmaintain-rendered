[pydocstyle-D405] Allow using parameters as a sub-section header

#9894 opened by @mikeleppane on 2024-02-08 12:55
@mikeleppane

Summary

This review contains a fix for D405 (capitalize-section-name) The problem is that Ruff considers the sub-section header as a normal section if it has the same name as some section name. For instance, a function/method has an argument named "parameters". This only applies if you use Numpy style docstring.

See: ISSUE

The following will not raise D405 after the fix:

def some_function(parameters: list[str]):
    """A function with a parameters parameter

    Parameters
    ----------

    parameters:
        A list of string parameters
    """
    ...
