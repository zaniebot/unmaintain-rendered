perf(pycodestyle): Introduce TokenKind

#3745 opened by @MichaReiser on 2023-03-26 18:30
@MichaReiser

This PR introduces a new TokenKind enum that only represents the token's kind without the parsed data (big int literal, string...). This is useful when you want a small and copyable representation over storing a reference to Tok.

Using TokenKind can further help with memory consumption and better cache locality because the kind is only 1byte large compared to the 40 bytes of a Tok.

This PR further changes the representation of Tokens from a vector of structs (Vec<(Location, Token, Location)>) to a struct of vectors (kinds: Vec<TokenKind>, starts: Vec<Location>, ends: Vec<Location>). This should reduce memory consumption because it avoids padding TokenKind to 4 bytes (3 wasted bytes for each token).
