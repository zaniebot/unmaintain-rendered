Allow the if_not_else Clippy lint

#20315 opened by @BurntSushi on 2025-09-09 12:17
@BurntSushi

Specifically, the if_not_else lint will sometimes flag code to change the order of if and else bodies if this would allow a ! to be removed. While perhaps tasteful in some cases, there are many cases in my experience where this bows to other competing concerns that impact readability. (Such as the relative sizes of the if and else bodies, or perhaps an ordering that just makes the code flow in a more natural way.)
