More stability checker options

#5299 opened by @konstin on 2023-06-22 12:52
@konstin
Summary

This contains three changes:

  • repos in check_ecosystem.py are stored as org:name instead of org/name to create a flat directory layout
  • check_ecosystem.py performs a maximum of 50 parallel jobs at the same time to avoid consuming to much RAM
  • check-formatter-stability gets a new option --multi-project so it's possible to do cargo run --bin ruff_dev -- check-formatter-stability --multi-project target/checkouts With these three changes it becomes easy to check the formatter stability over a larger number of repositories. This is part of the integration of integrating formatter regressions checks into the ecosystem checks.
