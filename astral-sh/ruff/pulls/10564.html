This PR selects more rules for typeshed in the ecosystem checks, bringing the configuration slightly closer to the (somewhat complex) ruff config typeshed now uses in CI for .pyi files: https://github.com/python/typeshed/blob/f337eb8a5149a05528d19605b7ddd0bc9b5125e4/pyproject.toml#L26-L149. In particular, having F selected would have prevented astral-sh/ruff#10509 from occuring, as the regression would have showed up in the ecosystem comment before I merged the PR that caused the regression (astral-sh/ruff#10341).
