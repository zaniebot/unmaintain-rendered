For users of pre-commit, it would be useful to add the typos pre-commit check that currently runs in CI. With this proposed change, CI would run the typos check twice (in the pre-commit stage and its own stage), which is not necessary.

Moving typos to pre-commit also identified a couple of typos that were not picked up with the original CI version of the tool. One that was fixed in blacks test cases in psf/black#3599 and one that is not considered a typo in black that has been added to _typos.toml.
