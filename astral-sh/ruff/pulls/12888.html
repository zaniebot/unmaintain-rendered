Select stable import name when multiple possible bindings are in scope

#12888 opened by @MichaReiser on 2024-08-14 09:41
@MichaReiser
Summary

Fixes astral-sh/ruff#12885

The issue with the FURB177 test is that there are two viable bindings for pathlib.Path: pathlib.Path or Path but bindings is a HashMap that has no guaranteed iteration order.

I'm reluctant to change bindings to a BTreeMap. I would suspect that it comes with quiet a performance penalty. That's why this PR fixes this issue locally by sorting the imports by their range and picking the last import if multiple options exist.

Alternatives
