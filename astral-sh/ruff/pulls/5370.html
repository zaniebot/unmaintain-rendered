Fix invalid syntax for binary expression in unary op

#5370 opened by @MichaReiser on 2023-06-26 15:15
Pull request opened by @MichaReiser on 2023-06-26 15:15

Summary

Fixes https://github.com/astral-sh/ruff/issues/5338

I added BinOp to can_break_expr for UnaryExpr but this can cause invalid syntax. It turned out, that I only had to add it because the grouping of the binary expression wasn't correct.

This PR changes the grouping to:

  • Group all binary expressions with the same precedence
  • Group the left
  • Group the right expression

This required changing the implementation significantly. It now "flattens" the left hand sides of the binary expression and formats all of them at once without recursing.
