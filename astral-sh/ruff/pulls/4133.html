Use non-empty ranges for logical-lines diagnostics

#4133 opened by @MichaReiser on 2023-04-27 14:47
Pull request opened by @MichaReiser on 2023-04-27 14:47

This PR sets non-empty ranges for the logical-line rules that then get correctly highlighted by --show-source.

I decided to align the ranges between all rules so that:

  • Extraneous whitespace: The range highlights the extraneous whitespace
  • Missing whitespace: The range highlights the token around which whitespace was expected. I found this easier to understand than pointing to the next token where the whitespace was expected because the message then reads: Whitespace expected around arithmetic token, but the source highlights the a identifier.
