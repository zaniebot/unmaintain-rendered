Add contents: write permission to release step

#4911 opened by @charliermarsh on 2023-06-07 01:03
@charliermarsh

Summary

In the most recent release, the "add the binaries to the GitHub release" step failed, due to a missing permission. I think that when we introduced id-token: write, it effectively turned off all other permissions (see docs). The action-gh-release docs state that they need contents: write, so this PR adds it.
