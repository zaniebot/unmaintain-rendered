Pull request opened by @cnpryer on 2023-07-16 17:07

Summary

Implements a potentially more re-usable naming convention for TupleParentheses.

Subscript case -> Preserve - "Preserve the tuple's parentheses if they are there" SkipInsideLambda case -> Never - "Parentheses are never needed with this parent" (#5806; comment) Comprehension case -> Never - "Parentheses are never needed with this parent" (#5790; comment) StripInsideForLoop case -> NeverPreserve - "Parentheses are never preserved in special cases" (I interpret "Preserve" as "not always necessary but keep them anyway")

The counter to this I'd think is that there may be benefit to having node-specific differences.
