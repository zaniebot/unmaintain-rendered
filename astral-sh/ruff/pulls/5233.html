Use package roots rather than package members for cache initialization

#5233 opened by @charliermarsh on 2023-06-21 01:07

Summary

This is a proper fix for the issue patched-over in astral-sh/ruff#5229, thanks to an extremely helpful repro from @tlambert03 in that thread. It looks like we were using the keys of package_roots rather than the values to initialize the cache -- but it's a map from package to package root.
