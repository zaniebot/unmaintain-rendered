Yeah, I think you're both right!

I was looking at it pretty narrowly through the lens of "what would be required to respect the gradual guarantee here" -- and it seemed obvious that, to respect the gradual guarantee, if we intersected with Any when the second argument to isinstance() was of type Any and we intersected with int when the second argument was of type type[int], we should therefore also intersect with int & Any when the second argument was of type[int] & Any.

But if we're making these changes to intersections, then it does definitely make sense to make equivalent changes to unions -- it's sound behaviour and consistent behaviour, even if it's not strictly "necessary" to support the gradual guarantee in the same way.
