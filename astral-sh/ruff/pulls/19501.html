[ty] Invert ty_ide and ty_project dependency

#19501 opened by @MichaReiser on 2025-07-23 07:15
@MichaReiser
Summary

This PR inverts the ty_project -> ty_ide dependency to ty_ide -> ty_project.

The reason why ty_project dependet on ty_ide was so that ty_ide can have its own Db trait. I currently don't forsee any IDE specific methods that Db would need to implement that can't be on the ty_project::Db.

Inverting the dependency has the advantage that ty_ide can query db.project() which seems fairly important (e.g. to get all files)

This helps with astral-sh/ruff#19475#discussion_r2224648880
