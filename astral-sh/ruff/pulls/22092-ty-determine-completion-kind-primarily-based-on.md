# [ty] Determine completion "kind" primarily based on how the symbol was defined

---

_@BurntSushi_

I'm not quite sure of the implementation here, so
I'm opening this up as a draft. If this seems like a
good direction, then I can:

* Add more tests.
* Fix the weirdness around `Member` and `MemberWithDef`.
  (The former now contains an optional `Definition` and
  the latter contains a required `Definition` but also
  embeds a `Member` which contains an otpional `Definition`.)

I think the alternative to this approach is to reuse the
goto definition stuff. That would help keep a single point
of truth. But:

* Main reason: For scope and attribute based completions, we are
  already hitting the use-def map where the `Definition` actually
  lives, so it seems a shame to not pull it out there.
* Secondary reason: The goto definition lookup for names really
  wants an AST to go along with the name. I think I could refactor
  `definitions_for_name` to break that, but it's a little annoying.

Fixes astral-sh/ty#2102
