Avoid extra parentheses in await expressions

Summary

This PR aligns the await parenthesizing with the unary case, which is: if the value is already parenthesized, avoid parenthesizing; otherwise, only parenthesize if the value needs parenthesizing.

Closes https://github.com/astral-sh/ruff/issues/7420.

This is actually a deviation from Black now.

Given:

return await self().http_client().fetch(
    f"http://127.0.0.1:{self.port}{path}", method=method, **kwargs,
)

We now format as:

return await (
    self()
    .http_client()
    .fetch(
        f"http://127.0.0.1:{self.port}{path}",
        method=method,
        **kwargs,
    )
)

Whereas Black formats as:

return (
    await self()
    .http_client()
    .fetch(
        f"http://127.0.0.1:{self.port}{path}",
        method=method,
        **kwargs,
    )
)

I need to explore this further.
