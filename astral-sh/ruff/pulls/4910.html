Remove subscript expressions from python_ast contains_effect helper

#4910 opened by @Sacrimento on 2023-06-06 22:12
@Sacrimento
Summary

Hello! This PR fixes a few rules where a fix was not possible because of the presence of square brackets in an expression. Previously, the Subscript expression was incorrectly considered to have runtime effects. However, I was unable to reproduce any scenarios where square brackets actually had any impact.

Here's an example for rule F841:

foo = {"hello": "world"}

def hello():
    bar = foo["hello"] # Unused variable, ruff won't fix because of [...] 

hello()

With this PR, the rules using ruff_python_ast.helpers.contains_effect will be able to fix errors containing subscript expressions.
