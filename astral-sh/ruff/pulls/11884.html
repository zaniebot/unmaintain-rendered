I don't think you can directly provide the settings key at the top level because the new server doesn't support workspace/didChangeConfiguration (yet). So, the correct code should be to specify it under init_options which sends this in the initialize request:

  init_options = {
    settings = {
      logLevel = "debug",
    }
  }
