[red-knot] Use type[Unknown] rather than Unknown as the fallback metaclass for invalid classes

#14961 opened by @AlexWaygood on 2024-12-13 19:44
Pull request opened by @AlexWaygood on 2024-12-13 19:44

Summary

This PR changes things so that we now infer type[Unknown] rather than Unknown as the fallback metaclass for an invalid class (either one that has a cyclic definition, or one where the metaclasses of the bases conflict). This better reflects the invariant that a class's metaclass must always be an instance of type. It follows the same principles established in https://github.com/astral-sh/ruff/pull/14942.

Test Plan

Assertions in existing mdtests updated. No new mdtests added, since the existing ones are reasonably comprehensive.
