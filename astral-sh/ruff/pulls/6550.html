Printer: Reserve buffer upfront

#6550 opened by @MichaReiser on 2023-08-14 07:08
@MichaReiser

Summary

I did some profiling on the formatter and noticed that we end up growing the Printer buffer multiple times. This PR tries to reduce the number of times that we need to grow the formatted string buffer by allocating a buffer of the same size as the unformatted document. This should be a good default because most files in a project are already formatted (we neither allocate too much nor too little).

Test Plan

This improved the formatter time by about 2%
