[ty] Never add elements to a UnionBuilder before setting recursively_defined

#22356 opened by @AlexWaygood on 2026-01-03 14:00
@AlexWaygood

Summary

In several places, we set UnionBuilder::recursively_defined() only after adding many elements to that union builder. But the recursively_defined field is only respected in UnionBuilder methods such as UnionBuilder::add_in_place_impl; setting recursively_defined after adding all the elements to the builder has no effect at all. There are similar issues with the various other union-builder settings such as order_elements, cycle_recovery, etc.

This PR fixes these bugs by adding a required settings parameter to UnionBuilder::new().
