Review comment by @AlexWaygood on CHANGELOG.md:15 on 2024-11-20 13:57
- **Use XDG (i.e. `~/.local/bin`) instead of the Cargo home directory in the standalone installer**
Review comment by @AlexWaygood on CHANGELOG.md:17 on 2024-11-20 13:59
    Previously, Ruff's installer used `$CARGO_HOME` or `~/.cargo/bin` for its target install directory. Now, Ruff will be installed into `$XDG_BIN_HOME`, `$XDG_DATA_HOME/../bin`, or `~/.local/bin` (in that order).
    
    This change is only relevant to users of the standalone Ruff installer (using the shell or PowerShell script). If you installed Ruff using `uv` or `pip`, you should be unaffected.
Review comment by @AlexWaygood on CHANGELOG.md:13 on 2024-11-20 14:00
    Ruff now defaults to Python 3.9 instead of 3.8 if no explicit Python version is configured using [`ruff.target-version`](https://docs.astral.sh/ruff/settings/#target-version) or [`project.requires-python`](https://packaging.python.org/en/latest/guides/writing-pyproject-toml/#python-requires) ([#13896](astral-sh/ruff#13896))
Review comment by @AlexWaygood on CHANGELOG.md:21 on 2024-11-20 14:00
    Ruff now uses a more recent version of the Unicode standard. In very rare cases, this can lead to formatting changes or new [`E501`](https://docs.astral.sh/ruff/rules/line-too-long/) violations if the computed line width differs.
Review comment by @AlexWaygood on CHANGELOG.md:38 on 2024-11-20 14:01
