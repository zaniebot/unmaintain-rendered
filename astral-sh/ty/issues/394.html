Following https://github.com/astral-sh/ruff/pull/21401 and https://github.com/astral-sh/ruff/pull/21456, we now support stub-file forward references in class bases, on the right-hand side of explicitly declared type aliases, and in all locations where a type expression is expected (including TypeVar bounds/constraints, etc.). If there are more places where we find that stub authors are expecting forward references to work and they currently don't, then we can consider changing how we treat stub files so that all name lookup in stub files is deferred. For now, however, let's see if the steps we've currently taken are sufficient.

Further bug reports are welcome, @jorenham, if you find anywhere where we're still emitting clear false positives on stub files!
