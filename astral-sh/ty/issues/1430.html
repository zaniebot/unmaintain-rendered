Goto-definition for binary operators only works if the relevant dunder has exactly the correct signature

#1430 opened by @AlexWaygood on 2025-10-24 09:32
@AlexWaygood
Summary

We're fixing this issue in astral-sh/ruff#21049 for goto-definition for unary operators, but the issue also exists for goto-definition for binary operators. Goto-definition allows you to jump from the first + in this snippet to Foo.__add__, but doesn't currently allow you to jump from the second + in this snippet to Bar.__add__ (because Bar.__add__ has an invalid definition):

class Foo:
    def __add__(self, other): ...

Foo() + Foo()

class Bar:
    def __add__(self): ...

Bar() + Bar()

Ideally goto-definition wouldn't care about such details, and would jump to Bar.__add__ even though it's invalid here. It's also probably not high-priority to fix this, though.
