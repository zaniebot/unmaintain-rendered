@carljm
Thanks for trying ty and reporting an issue!

First, please check whether your question or problem is already known.

  1. If you are seeing a type error on your code that you didn't expect, does the documentation reference page for that error code explain the problem?
  2. Have we documented it in our typing FAQ?
  3. Is it a typing feature we haven't (fully) implemented yet?
  4. Or is it one of these known bugs?
  • Sometimes ty fails to narrow types as it should when there is a call to a NoReturn or Never returning function (e.g. sys.exit()) in a nested control flow path. This is tracked in astral-sh/ty#690
  • ty may not infer a specialization for a type variable inside a generic protocol. This is tracked in astral-sh/ty#1714
