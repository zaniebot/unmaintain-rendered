  • https://github.com/python/mypy/issues/14970

Comment by @erictraut on 2025-07-23 21:37

I had a similar problem in pyright when dealing with very large tuple literal expressions.

My solution was to add a limit to the number of tuple entries. If there is no inference context (i.e. it's not assigned to a variable or parameter with a declared type) and the number of entries exceeds the limit (currently 256), then the type of the expression is inferred to be tuple[Unknown, ...]. That behavior isn't ideal, but I figured it was preferable to long hangs.
