Unittest.TestCase.fail() does not narrow variable type

#1395 opened by @Manezki on 2025-10-18 14:20
@Manezki
Summary

Variable type is not narrowed down by checking for None value and interrupting execution with unittest.TestCase.fail(). Substituting unittest.TestCase.fail() with return does exclude None from possible types.

Expected behavior

After checking for None with execution interruption with unittest.TestCase.fail() None is no longer consider a possible type for the variable.

That is, in the reproduce I would expect to not get Attribute `color` on type `Structure | None` may be missing (possibly-missing-attribute) [Ln 23, Col 9]

Reproduce

https://play.ty.dev/d6d62783-54e5-4ada-b1a4-210e56eb40f5

Version

ty 0.0.1-alpha.23
