Thanks for the report!

You haven't provided us with a minimal repro here, but I'm guessing your code looks something like this?

import unittest

def string_or_int() -> int | None:
    return 42

class Foo(unittest.TestCase):
    def test_foo(self) -> None:
        x = string_or_int()
        self.assertIsNotNone(x)
        reveal_type(x)  # revealed: int | None

If so, this is not a form of narrowing supported by any other type checker right now. I think the real solution to this is python/typing#930, which would allow typeshed to annotate the return type of methods like assertIsNotNone in such a way that all type checkers would understand their narrowing behaviour.
