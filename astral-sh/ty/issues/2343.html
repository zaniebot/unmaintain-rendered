Subscript fails only when using PEP-695 style union type aliases

#2343 opened by @mtshiba on 2026-01-05 14:12
Issue opened by @mtshiba on 2026-01-05 14:12

For example (https://play.ty.dev/47f4b905-76f0-4b56-8df4-1e8b484c073b):

class C:
    def __getitem__(self, index: int) -> int:
        return 1

    def __add__(self, other: int) -> int:
        return 1

class D:
    def __getitem__(self, index: int) -> int:
        return 1

    def __add__(self, other: int) -> int:
        return 1

type CD = C | D
ImplicitCD = C | D

def f(x1: C | D, x2: CD, x3: ImplicitCD):
    reveal_type(x1[1])  # ok
    reveal_type(x2[1])  # error
    reveal_type(x3[1])  # ok

    reveal_type(x1 + 1)  # ok
    reveal_type(x2 + 1)  # ok
    reveal_type(x3 + 1)  # ok
