Non deterministic diagnostics

#1670 opened by @MichaReiser on 2025-11-28 21:28
@MichaReiser
Summary

Ecosystem reports now often show flaky diffs, for example astral-sh/ruff#21595#issuecomment-3568122741. Sometimes it's flaky diagnostics, sometimes it's only the ordering of union elements that changes between runs.

This has become more prevalent since astral-sh/ruff#20566 landed.

We need deterministic diagnostic output (including messages) for GitLab output reports (or baselines)

A PR fixing the instability should also enable the CI job added in astral-sh/ruff#21864, demonstrating that the instability is fixed

The simplest solution is to sort the union types in the cycle recovery function. We can also do this for recursively defined types only, as in astral-sh/ruff#21683.

But honestly, I don't really understand why this happens. Is it a flaw in our implementation, as in astral-sh/ruff#20966, or is it a salsa specification or bug? It seems I need to look into this in more detail.

Comment by @MichaReiser on 2025-11-29 14:53

Some notes from our internal Discord channel on why this might be happening https://discord.com/channels/1039017663004942429/1443381998100938783/1443492249307582484

I don't think this is a Salsa bug. It's just that some queries are execution order dependent and there's no deterministic execution order for cycles (because they can be entered from many different queries, running on multiple threads at the same time)
