Overloaded non-stub function f must have an implementation - issue just on the last overload

#1316 opened by @darioinova on 2025-10-07 09:11
@darioinova

Summary

When defining an overloaded function, the LAST implementation is always labeled as "Overloaded non-stub function f must have an implementation"

from typing import overload

class A:
    @overload
    def f(v: float) -> float:
        return v * 2

    @overload
    def f(v: int) -> int: # issue here
        return v * 2

from typing import overload

class A:
    @overload
    def f(v: float) -> float:
        return v * 2

    @overload
    def f(v: int) -> int:  # no issue here !!!!
        return v * 2

    @overload
    def f(v: str) -> str:
        return v * 2

    @overload
    def f(v: list) -> list:     # issue here
        return v + v

Comment by @AlexWaygood on 2025-10-07 10:14

I see! I can't see any implementation in the example snippets you posted in this issue, however. An "overload implementation" is a function not decorated with @overload that immediately follows all the @overload-decorated functions. See the typing-module docs here -- there is a non-@overload-decorated process function immediately after all the overloads that are decorated with @overload, and it is the final non-@overload-decorated process() function that provides the function implementation that is actually used at runtime
