Confusing naming unbound for methods

#636 opened by @Tishka17 on 2025-06-11 14:30
@Tishka17

Summary

In python there are bound methods which is actually a function/classmethod bound to an object (<bound method A.foo of <class '__main__.A'>>). As on opposite, there are unbound methods - those, who were not bound.

You can find this term in that meaning here: https://docs.python.org/3/library/unittest.mock-examples.html#mocking-unbound-methods https://docs.python.org/3/reference/datamodel.html https://docs.python.org/3/library/types.html

Ty uses "unbound method" as a synonym of an absent/not assigned attribute. https://github.com/astral-sh/ty/blob/main/docs/reference/rules.md#possibly-unbound-implicit-call

Checks for implicit calls to possibly unbound methods.

This doesn't relate to unbound local variable.

Instances of C have an x attribute which is definitely "bound" (definitely exists), but is not "declared".
