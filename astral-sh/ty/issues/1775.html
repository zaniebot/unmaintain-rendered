In this situation, I would expect the first autocomplete suggestion to be total=, because you can specify total=False when inheriting from TypedDict to specify that the TypedDict keys in the type you're defining should not be required. Other keyword arguments accepted when subclassing TypedDict are closed and extra_items (see the spec).
