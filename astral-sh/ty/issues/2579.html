Narrowing uses lossy conversion for bytes

#2579 opened by @dscorbett on 2026-01-21 16:05
@dscorbett
Summary

Narrowing a non-UTF-8 bytes subscript also narrows a different bytes subscript if String::from_utf8_lossy converts them to the same string. Narrowing should not use lossy conversion. Example:

$ cat >bug.py <<'# EOF'
from typing import reveal_type
def _(x: dict[bytes, int | None]):
    if x[b"\xaa"] is not None:
        reveal_type(x[b"\xff"])
# EOF

$ ty check --output-format concise bug.py
bug.py:4:21: info[revealed-type] Revealed type: `int`
Found 1 diagnostic
