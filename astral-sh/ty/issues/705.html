The return type should not be detected as None

#705 opened by @tonka3000 on 2025-06-25 19:12
Issue opened by @tonka3000 on 2025-06-25 19:12

Summary

In the below scenario ty report that the return type is Version|None but the code can only be Version.

class Version:
    def __init__(self):
        self.foo = 1


_ver: Version | None = None


def get_version() -> Version:
    global _ver
    if _ver is not None:
        return _ver # <=== ty error here
    _ver = Version()
    return _ver

ty report error[invalid-return-type] : Return type does not match returned value: expected Version, found Version | None
