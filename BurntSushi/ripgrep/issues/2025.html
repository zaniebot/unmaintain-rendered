Comment by @BurntSushi on 2021-10-18 16:51

Probably the simplest thing here is to use $'...' syntax. So:

$ echo "fox" | rg -r $'${1}\n${1}es' "(fox)"

Your sed command works likely because it is interpreting \n as an escape sequence for a newline character. It's hard to find precise documentation supporting this behavior in either the POSIX standards or GNU's documentation for sed, but that does appear to be what's happening.

Conversely, ripgrep does no interpretation of escape sequences. The only supported syntax is $group/${group} as a variable for captured text, and $$ to insert a literal $.

See also: https://stackoverflow.com/questions/8467424/echo-newline-in-bash-prints-literal-n
