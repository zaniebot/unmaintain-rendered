Comment by @ltrzesniewski on 2023-04-27 10:01

I may be assuming a bit much here, but you don't seem to have any Rust repository, so maybe you're unaware that unsafe is a legitimate Rust keyword. It doesn't automatically make the code insecure, it only enables a few additional features which prevent the Rust compiler from automatically proving that the code is 100% safe. It indicates a region of code which needs careful manual review.

The Rust standard library contains unsafe code everywhere, so requiring ripgrep to use only safe code is a bit unrealisitc. ðŸ™‚
