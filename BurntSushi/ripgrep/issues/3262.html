Comment by @BurntSushi on 2026-01-09 14:50

This is intended and correct behavior. Even with -F/--fixed-strings, the capture group $0 is still available. Moreover, there is no documentation suggesting or implying that the interpretation of the -r/--replace string is dependent on whether -F/--fixed-strings is enabled. I don't think it would be a good idea to do so. It's quite subtle. The rule, "you always need to escape $ to write a literal $" is simpler and not conditional.
