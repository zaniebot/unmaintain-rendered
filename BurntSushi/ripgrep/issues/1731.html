Parallel Command Execution (as in fd)

#1731 opened by @dharrigan on 2020-11-16 13:01
@dharrigan

Hi,

A feature I personally find incredibly useful in fd, is the ability for it to pass the results of the run individually, or in batch, to an external program using parallel command execution, using these switches:

    -x, --exec <cmd>                   Execute a command for each search result
    -X, --exec-batch <cmd>             Execute a command with all search results at once

More details can be found here about fd's parallel command execution.

Whilst a similar thing can be achieved by doing this already in rg (for example):

vim `rg -l text`

It's a bit cumbersome and is only working in batch. It would be great if rg had a similar mechanism as in fd, i.e.,

rg text -X vim {} (for batch)

Thank you.

-=david=-
