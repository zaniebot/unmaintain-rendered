however, unless i'm mistaken, this doesn't happen if dir contains a .gitignore file with the contents * (common eg in .venv folders)

You aren't mistaken. This is expected behavior. By providing the directory explicitly, ripgrep will descend into that directory regardless of any ignores at the sibling or parent level. But ripgrep will still respect ignores within that directory.
