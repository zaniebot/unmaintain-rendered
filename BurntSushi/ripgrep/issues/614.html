Comment by @BurntSushi on 2017-09-28 00:04

I'm on mobile, but technically the command you are running will never be guaranteed to produce sorted output, and ripgrep can't do a damn thing about it. Namely, xargs is free to invoke multiple distinct rg processes, and from there the output order is anyone's guess.

Now in this case the number of arguments is small, so I would expect xargs to just invoke rg once with all of them. But I don't actually know.

I'd suggest trying to reproduce this issue with xargs. You might also be able to observe interesting things via xargs's -P and -n flags.
