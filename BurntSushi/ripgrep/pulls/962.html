Use cwd to resolve paths in Ignore::matched

#962 opened by @tmccombs on 2018-06-24 06:56
@tmccombs

Fixes #829 and #278.

This does change the Ignore struct to depend on the working directory at the time of creation, I think this is fine, since Ignore isn't publicly accessible and the Walk structs already depend on the current working directory implicitly.

I tried to make a minimal change to fix the issue. An alternative implementation could be to call current_dir in the matched_ignore method instead of in add_parents, but I'm not sure of the performance implications of doing that.

Another possible solution would be to change the places that we call Ignore::matched to change the path to be relative to the absolute_base of the Ignore.
