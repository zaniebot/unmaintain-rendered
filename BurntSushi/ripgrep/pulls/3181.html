Building it can consume resources. In particular, on Windows, the various binaries are eagerly resolved.

I think this originally wasn't done. The eager resolution was added later for security purposes. But the "eager" part isn't actually necessary.

It would probably be better to change the decompression reader to do lazy resolution only when the binary is needed. But this will at least avoid doing anything when the -z/--search-zip flag isn't used. But when it is, ripgrep will still eagerly resolve all possible binaries.
