Look for global git/ignore in ~/.config/git, not ~/git

#343 opened by @blueyed on 2017-01-29 15:33
@blueyed

The documentation says:

If $XDG_CONFIG_HOME is not set or is empty, then $HOME/.config/git/ignore is used instead.

This is the expected behavior, but the code looked at ~/git/ignore instead.

This is my first Rust code, so it might be wrong..!

I think you need env::home_dir().map(|p| p.join(".config")) since env::home_dir() returns an Option<PathBuf>.
