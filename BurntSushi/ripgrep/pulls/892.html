ignore: impl Clone for DirEntry

#892 opened by @BurntSushi on 2018-04-21 15:42
@BurntSushi

There is a small hiccup here in that a DirEntry can embed errors associated with reading an ignore file, which can be accessed and logged by consumers if desired. That error type can contain an io::Error, which isn't cloneable. We therefore implement Clone on our library's error type in a way that re-creates the I/O error as best as possible.
