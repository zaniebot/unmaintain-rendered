Add --include-zero option

#1405 opened by @cstyles on 2019-10-17 02:45
@cstyles

This PR addresses the suggestion in BurntSushi/ripgrep#1370.

If --count or --count-matches is enabled, ripgrep currently only reports the number of matches if it's non-zero. This is desired behavior because otherwise it can generate a lot of noise but sometimes users may want to print zeroes anyway. There's actually already a setting for outputting a zero if there were no matches but there's no way of turning it on from the command line interface. This PR adds the --include-zero option to fix that.
