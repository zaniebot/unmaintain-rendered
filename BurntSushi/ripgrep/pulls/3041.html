Fix the "No matches with empty patterns file and invert match" bug

#3041 opened by @xenoninja on 2025-05-08 07:02
@xenoninja

This PR addresses BurntSushi/ripgrep#1332 and BurntSushi/ripgrep#3001. The problem is when an empty pattern file is given, ripgrep sees the pattern as "an empty regex" not "zero regexes".

| pattern file | grep's output | rg's output | |:-|:-:|:-:| | empty | everything | nothing | | newline | nothing | nothing |

Now the fix works out as below

| pattern file | grep's output | rg's output | |:-|:-:|:-:| | empty | everything | everything | | newline | nothing | nothing |
