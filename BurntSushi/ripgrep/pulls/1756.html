globset: fix recursive suffix over matching

#1756 opened by @zertosh on 2020-12-08 05:03
@zertosh

The syntax documentation correctly states how recursive suffixes should behave:

** recursively matches directories [..] foo/** matches foo/a and foo/a/b, but not foo.

Previously ** would actually match foo. This patch makes it so that it doesn't. Also adds support to the "prefix" strategic matcher so that it handles common cases like foo/**.
