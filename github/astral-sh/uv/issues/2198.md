# `-e .` not respected in nested requirements file

---

_Issue opened by @wadevries on 2024-03-05 09:19_

I have the following minimal setup:

pyproject.toml:

```
[project]
name = "uv-test"
version = "0.0.0"
dependencies = [ "django" ]
```

requirements.txt:
```
-r requirements_inner.txt
```

requirements_inner.txt:
```
-e .

first
```

Then I install the top-level requirements:

```
uv pip install -r requirements.txt
```

## Expected result

`first`, `django` and `uv-test` (the project itself) are installed along with their dependencies.
