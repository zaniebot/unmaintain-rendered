# Allow bypassing SSL verification in uv installation script

---

_Issue opened by @eegli on 2025-05-27 07:50_

### Summary

When installing uv via the [standalone installer](https://docs.astral.sh/uv/getting-started/installation/#standalone-installer) behind a corporate proxy, curl might fail due to self-signed certificates.

The remote installation script can be fetched by adding the `-k` flag:

```sh
curl -kLsSf https://astral.sh/uv/install.sh | sh
```

However, the script itself has no option to disable SSL verification:
> $ curl -kLsSf https://astral.sh/uv/install.sh | sh
downloading uv 0.7.8 x86_64-pc-windows-msvc
curl: (60) SSL certificate problem: self signed certificate in certificate chain
More details here: https://curl.se/docs/sslcerts.html

Downloading `install.sh` and modifying `downloader()` sure works but I'd rather rely on the remote script for installation. [Configuration options for the installer](https://docs.astral.sh/uv/configuration/installer/) already exist. I'd be great to have the script accept another option to allow insecure connections. 

If this is something you agree to support, I'm happy to look into it.

_Comment by @geofft on 2025-05-27 19:15_

Not a direct answer, but does setting `CURL_CA_BUNDLE` to the a file containing your corporate proxy's cert or CA cert (in text / PEM format) work? e.g.

```
export CURL_CA_BUNDLE=~/Downloads/corporate-ca.pem 
curl -LsSf https://astral.sh/uv/install.sh | sh
```

The environment variable will get inherited by all the curl processes.

In general, I would sort of think it's good practice for your corporate systems to set up validation for your internal certs to work by default instead of needing you to use `--insecure` :)
