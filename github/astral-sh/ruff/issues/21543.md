# [format] multiline type_param_sequence in return type forces multiline in function parameters

---

_Issue opened by @anabelle2001 on 2025-11-20 17:45_

### Summary

For example, the function
```py
def my_function(self) -> tuple[
    type[module.submodule.some.really.long.type],
    type[module.submodule.some.really.long.type],
]: ...
```
is unchanged by black 25.1.0, but is changed by ruff 0.14.4 into

```py
def my_function(
    self,
) -> tuple[
    type[module.submodule.some.really.long.type],
    type[module.submodule.some.really.long.type],
]: ...
```

### Version
ruff 0.14.4
