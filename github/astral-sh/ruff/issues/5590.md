# [Feature request] - Allow ignoring line length of an entire dict/list definition instead of having to ignore each line individually

---

_Issue opened by @michaeloliverx on 2023-07-07 10:08_

Lets say you have a massive inline dictionary like so:

```python
random_dict = {
    "key1": "value1",
    "key2": "value2",
    "key3": "Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed euismod urna non tellus maximus hendrerit. Duis eu magna lobortis, luctus massa id, semper magna. Fusce iaculis, leo vel fermentum semper, sem libero vestibulum lectus, ac posuere lectus metus id est. Nam condimentum consequat purus, in vestibulum mi convallis in. Sed malesuada, metus vel vulputate dapibus, turpis mi elementum nunc, at feugiat nulla eros ut dolor. Etiam commodo metus id sem lobortis, ac dignissim ante facilisis. Vestibulum ante ipsum primis in faucibus orci luctus et ultrices posuere cubilia curae; Nam fermentum elit et neque ultrices, eget scelerisque mauris gravida.",
    "key4": {
        "nested_key1": "nested_value1",
        "nested_key2": "Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed euismod urna non tellus maximus hendrerit. Duis eu magna lobortis, luctus massa id, semper magna. Fusce iaculis, leo vel fermentum semper, sem libero vestibulum lectus, ac posuere lectus metus id est. Nam condimentum consequat purus, in vestibulum mi convallis in. Sed malesuada, metus vel vulputate dapibus, turpis mi elementum nunc, at feugiat nulla eros ut dolor. Etiam commodo metus id sem lobortis, ac dignissim ante facilisis. Vestibulum ante ipsum primis in faucibus orci luctus et ultrices posuere cubilia curae; Nam fermentum elit et neque ultrices, eget scelerisque mauris gravida."
    }
}
```

In this case I would have to stick a `# noqa: E501` in multiple places, instead I would like the ability to place a comment at the end of the dictionary or similar that tells ruff to ignore line length for this entire dictionary and or list or any collection really.

I have tried disabling line length for particular test files but that quickly makes it so the other code gets very long as well.
