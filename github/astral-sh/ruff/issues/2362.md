# Implement `flake8-no-implicit-concat`

---

_Issue opened by @ngnpope on 2023-01-30 20:33_

[GitHub](https://github.com/10sr/flake8-no-implicit-concat), [PyPI](https://pypi.org/project/flake8-no-implicit-concat/).

- [x] [`NIC001`](https://github.com/10sr/flake8-no-implicit-concat#violation-codes): Implicitly concatenated str literals on one line
  - Already implemented via `ISC001` from [`flake8-implicit-str-concat`](https://github.com/charliermarsh/ruff#flake8-implicit-str-concat-isc).
- [x] [`NIC002`](https://github.com/10sr/flake8-no-implicit-concat#violation-codes): Implicitly concatenated str literals over multiple lines
  - Already implemented via `ISC002` from [`flake8-implicit-str-concat`](https://github.com/charliermarsh/ruff#flake8-implicit-str-concat-isc) when using the configuration option provided by `ruff` (`allow-multiline = false`)
- [x] [`NIC101`](https://github.com/10sr/flake8-no-implicit-concat#violation-codes): Implicitly concatenated bytes literals on one line
  - Already implemented via `ISC001` from [`flake8-implicit-str-concat`](https://github.com/charliermarsh/ruff#flake8-implicit-str-concat-isc).
- [x] [`NIC102`](https://github.com/10sr/flake8-no-implicit-concat#violation-codes): Implicitly concatenated bytes literals over multiple lines
  - Already implemented via `ISC002` from [`flake8-implicit-str-concat`](https://github.com/charliermarsh/ruff#flake8-implicit-str-concat-isc) when using the configuration option provided by `ruff` (`allow-multiline = false`)

This is somewhat the inverse of [`flake8-implicit-str-concat`](https://github.com/charliermarsh/ruff#flake8-implicit-str-concat-isc). Despite this, they agree that implicit concatenation on a single line shouldn't happen. One also has a separate rule for `bytes` vs `str` where the other uses combined rules.
