# let some rules not cause an error exit code

---

_Issue opened by @spaceone on 2023-01-30 11:42_

`ruff` has many rules and i would like to let most of the rules be checked.
we have a repository wide configuration and a pre-commit hook.

As the code base is grown over the years we have many style violation for specific rules (e.g. `RET50{5,6,7,8}` or `SIM`, which are currently also not autofixable) and we don't have them time to do it manually currently.
but the rules would make sense for our code base.

So I would like to have them visible for the developers, so that they see that there is an issue (which they might fix then as well) but `ruff` should not exit with an exit code =! 0 for them, as this would e.g. block the pre-commit hook.

‚Üí A new config option could provide a list of rules which still get reported but don't cause an error.
