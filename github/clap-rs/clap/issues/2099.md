# Unexpected behaviour with missing args using required_unless

---

- [ x] Searched the [discussions](https://github.com/clap-rs/clap/discussions)

### Code

```rust
// You code goes here. Please try to write a complete program,
// including `fn main() { ... }`
  use clap::Clap;                                                                                                                                                                             
  #[derive(Clap, Clone, Debug)]                                                                                                                                                               
  #[clap(about = "test")]                                                                                                                         
  struct Args {                                                                                                                                                                               
      #[clap(                                                                                                                                                                                 
          short,

      #[clap(                                                                                                                                                                                 
          short,                                                                                                                                                                              
          long,                                                                                                                                                                               
          required_unless = "path"                                                                                                                                                            
          )]                                                                                                                                                                                  
      c: Option<String>,

          name = "path",                                                                                                                                                                      
          short = "p",                                                                                                                                                                        
          long = "path",                                                                                                                                                                      
          )]                                                                                                                                                                                  
      d: Option<String>                                                                                                                                                                       
  }                                                                                                                                                                                           
                                                                                                                                                                                              
  fn main() {                                                                                                                                                                                 
     let args = Args::parse();                                                                                                                                                               
  } 
```

### Steps to reproduce the issue

1. Run `cargo run` with no options specified, when multiple options are required. 

### Version

* **Rust**: 1.44.1
* **Clap**: 3.0.0-beta.1

### Actual Behavior Summary

When it is run without any arguments, it states that only the first argument is required.
```
error: The following required arguments were not provided:
    --a <a>

USAGE:
    project --a <a>
```
### Expected Behavior Summary

The message should state all the required arguments.
```
error: The following required arguments were not provided:
    --a <a>
    --b <b>
    --c <c>

USAGE:
    project [OPTIONS] --a <a> --b <b> --c <c>
```
### Additional context

Something to note is that when using StructOpt instead, it behaves as expected, likely add a result of StructOpt not using the beta version of clap.
